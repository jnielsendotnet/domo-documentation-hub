Hello everyone!

We're excited to announce that version 1.3.1 of the Domo Documentation Hub is now available! This release brings expanded React tutorials, a brand-new AI FAQ, important fixes to Domo.js documentation, and major improvements to our docs tooling and preview workflows.


🌟 What's New

🤖 New Domo AI FAQ
A new "Domo AI FAQ" article is now live, giving customers a single place to find answers to the most common questions about Domo's AI capabilities.
Thanks to Ken Boyer for authoring the article and Jared Peterson for polishing the title, intro, and navigation links.

⚛️ Refreshed React Tutorials
Our React tutorial library has been significantly overhauled for clarity and real-world applicability:
  • Todo App — now walks through AppDB integration, Redux Toolkit async state, and loading user identity with IdentityClient/UserClient
  • Mapbox World App — clearer step-by-step setup, data fetching, and map-style customization
  • AI Book Recommender — streamlined instructions and code examples
  • Todo App now recommends pnpm over yarn, aligning with Domo's tooling guidance
  • Retired the outdated Bar Code Scanner PWA tutorial
Thanks to Jon Tiritilli for the tutorial refresh.

🛠️ Domo.js Docs — CDN Fixes & Version Cleanup
Copy-paste examples across the Domo.js tutorials, guides, and reference pages now use the unpkg-hosted CDN (`https://unpkg.com/ryuu.js`), so the snippets run standalone without requiring a CLI scaffold. Version numbers and internal links across the v4/v5/main Domo.js pages have also been corrected.
Thanks to Jason Hansen for the CDN fix and Han Kim for the version/link cleanup.

🚀 Mintlify Preview Workflow
A new GitHub Action now automatically generates Mintlify preview deployments for pull requests and long-lived release branches, so reviewers can see rendered changes without needing an open PR.
Thanks to Jon Tiritilli for building the workflow.

🔄 Smarter OpenAPI Sync
Our OpenAPI sync workflow has been reworked for reliability:
  • Per-file PRs — each changed YAML spec now gets its own PR with its own nav edit, instead of one bundled PR
  • Content-hash diffs — switched from unreliable mtime comparisons to SHA-256 hashing
  • Stale-nav pruning — automatically strips orphaned OpenAPI entries from docs.json when upstream YAMLs are renamed or removed, fixing preview-deploy breakages from case-rename issues
Thanks to Jon Tiritilli for the rework.


🙏 Thank You

A heartfelt thank you to everyone who contributed to this release. Your work keeps our developer and customer-facing documentation accurate, useful, and a pleasure to read.

If you have questions or feedback, please reach out — we're always happy to help!
