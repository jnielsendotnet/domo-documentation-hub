Hello everyone!

We're excited to announce that version 2.0.0 of the Domo Documentation Hub is now available! This is our biggest release yet — a full programmatic migration of the Knowledge Base from Salesforce to MDX, a comprehensive refresh of our style guide, a major SCIM documentation rewrite, and a lot more.


🌟 What's New

📚 Full Programmatic Salesforce → MDX Migration
Our entire Knowledge Base has been programmatically re-synced from Salesforce into MDX — across English, German, Spanish, French, and Japanese. This included custom tooling to convert HTML to MDX, download and validate article images, strip Salesforce tables of contents, collapse orphaned dividers, fix nested tables and callouts, and normalize inline image handling. The result: hundreds of articles are now cleaner, more consistent, and fully aligned with our MDX conventions.
Thanks to Jared Peterson for architecting and executing the migration end-to-end, including building the synthesis, diff, and formatting scripts that made it possible.

📐 KB Style Guide & Template Overhaul
The Domo KB Style Guide and New Article Template received a major refresh: clearer rules for heading hierarchy, FAQ placement, horizontal rule usage, internal links, code blocks, table alignment and cell padding, em-dash exceptions, `<br/>` usage in tables, and imperative-mood exceptions for fixed best-practice headings. Future-tense language has been cleaned up, and section-title capitalization is now explicit.
Thanks to Jared Peterson and Dan Brinton for the deep collaboration on these standards.

🔐 SCIM Integration Documentation Rewrite
The SCIM Integration article has been significantly revised with improved role and user instructions, clearer attribute descriptions, better user-deletion notes, and tightened formatting for role/user setup steps.
Thanks to Dan Brinton for leading the rewrite and Jared Peterson for the formatting polish.

🔑 New Access Tokens Documentation
A new Access Tokens article is now published, giving admins clear guidance on creating and managing tokens.
Thanks to Dan Brinton for authoring the article.

📄 Page Ordering for Embed
New documentation covering page ordering behavior in Embed is now available.
Thanks to Mamta Bolaki for the contribution.

🤖 AI Services OpenAPI Refresh
The AI Services OpenAPI spec has been updated, and the legacy AI Service spec has been retired to eliminate duplication.
Thanks to Joe Clark for modernizing the AI Services documentation.

🔄 Smarter OpenAPI Sync Triggers
The OpenAPI sync workflow now fires on `repository_dispatch` from upstream API repos instead of a fixed cron, with a `workflow_dispatch` escape hatch (including a `force_sync` option) for manual runs. Changes to API specs show up in docs faster and with less drift.
Thanks to Jon Tiritilli for the workflow rework.

🧹 MDX Parse Error Sweep
89 articles across English, Japanese, and German were scrubbed of MDX parsing errors — closing tags, braces, brackets, and other subtle syntax issues — across five dedicated cleanup rounds. Migration artifacts (one-time scripts, CSVs, and temporary skills) have also been removed now that the migration is complete.
Thanks to Jared Peterson for the meticulous cleanup.

📝 Release Notes Skill
A new Claude Code skill now generates friendly, email-ready release notes by diffing git tags — the very skill that wrote the notes you're reading.
Thanks to Jon Tiritilli for building it.


🙏 Thank You

A heartfelt thank you to everyone who contributed to this release — especially for the sustained effort behind the 2.0 migration. Your work has made the Documentation Hub dramatically more consistent, maintainable, and useful for every reader who lands on it.

If you have questions or feedback, please reach out — we're always happy to help!
