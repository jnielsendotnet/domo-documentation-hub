### Nouvelles fonctionnalités et optimisations

# Mises à jour du journal d'activité

Nous avons converti le journal d'activité en réaction. Cette mise à jour offre les améliorations suivantes:

* Filtre des événements plus intuitif, avec le filtre Type d'événement et Événement divisé en filtres distincts accessibles par liste déroulante.
* Les filtres Date et Personne sont désormais intégrés dans les en-têtes de colonne au lieu d'être proposés en tant que filtres distincts accessibles par liste déroulante.
* Une expérience utilisateur plus compacte, qui utilise l'espace de manière plus efficace.

Consultez l'article [Journal d'activité](/s/article/360042934574) dans la base de connaissances.

# Mises à jour de Conversation par IA

Conversation par IA est désormais disponible dans l'application mobile et dans Embed.

Posez une question sur vos données et Conversation par IA vous donnera les réponses en détaillant la façon dont elle les a obtenues. Vous disposerez également de visualisations, de contexte de données et d'invites afin de poser d'autres questions. Allez plus loin que les requêtes en langage naturel et laissez la conversation par IA vous aider à accomplir des tâches telles que la configuration d'une alerte, l'appel d'un modèle d'IA, etc.

# DomoStats: Rapport Groupes

<Frame><img alt="rtaImage.jpeg" src="/images/kb/ka0Vq0000002hX7-00N5w00000Ri7BU-0EMVq000001VwNJ.jpg"/></Frame>

Le rapport DomoStats Groupes répertorie les groupes Domo et les attributs pertinents.

Ce rapport comporte des informations importantes, notamment:

* Nom du groupe
* Type de groupe
* Date de création
* ID du créateur

Le rapport Groupes peut être associé à d'autres rapports de groupe tels que le rapport Accès aux DataSets. En savoir plus sur [DomoStats](/s/article/360043433813).

<Note>
**Remarque:** un rapport DomoStats précédent fournissait des informations sur les adhésions aux groupes. Ce rapport portait par erreur le nom de Groupes; il a été renommé Membres de groupe.

</Note>

# Retrait de l'aperçu automatique dans Analyzer pour les DataSets Cloud Amplifier

L'aperçu automatique est désormais désactivé par défaut pour les DataSets Cloud Amplifier lorsque vous ouvrez Analyzer. Cette mise à jour réduit le nombre de requêtes produites et les coûts. Vous pouvez toujours activer l'aperçu automatique dans le menu Analyzer.

# Flux de travail: initialisation de variables

Vous pouvez désormais attribuer une valeur initiale à une variable pendant l'expérience de création de version dans les flux de travail. Cette fonctionnalité évite d'avoir à utiliser des paramètres de début ou des fonctions de moteur de code pour renseigner les variables exigeant toujours les mêmes valeurs au démarrage.

Cette fonctionnalité a également ces autres avantages:

* **Conception de flux de travail simplifiée**
  : attribuez des valeurs initiales aux variables directement pendant la conception.
* **Cohérence**
  : garantit que les variables contiennent toujours la même valeur initiale au début d'un flux de travail.
* **Efficacité améliorée**
  : réduit les étapes de flux de travail inutiles.
* **Expérience utilisateur améliorée**
  : configurez une valeur initiale en quelques clics.

<Frame><img alt="Screenshot 2025-01-15 at 8.09.52 AM.png" src="/images/kb/ka0Vq0000009NGj-00N5w00000Ri7BU-0EMVq000004pCmQ.jpg" style={{width: 896, height: 393}}/></Frame>

En savoir plus sur l' [ajout d'une variable à un flux de travail](/s/article/000005331#add_a_variable) et sur l'attribution d'une valeur initiale à cette variable.
