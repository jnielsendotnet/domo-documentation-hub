### Domo.AI

**Remarque concernant la sécurité:** Domo considère que l'IA doit être gérée correctement, conformément aux directives internes de nos clients. Les assistants d'IA n'envoient à OpenAI que des métadonnées Domo concernant les sources de données Domo.

À ce titre, les administrateurs qui souhaitent utiliser les assistants d'IA dans Domo doivent activer les modèles OpenAI à utiliser dans Paramètres d'administration de la couche de service d'IA. Par ailleurs, les utilisateurs non-administrateurs de Domo qui souhaitent exploiter les assistants d'IA doivent avoir un rôle dans lequel l'autorisation Utiliser les services d'intelligence artificielle est activée.

Pour l'exécution des assistants d'IA, Domo proposera dans les prochains mois une option LLM en interne qui pourra être exploitée sans envoi d'informations à OpenAI.

## Assistant d'IA Beast Mode

L'assistant d'IA Beast Mode de [Domo.AI](http://Domo.AI) peut aider à générer des calculs Beast Mode dans l'éditeur Beast Mode. Pour utiliser l'assistant d'IA Beast Mode, l'autorisation Services d'intelligence artificielle doit avoir été attribuée à votre rôle.

Voici quelques exemples de demandes:

* Renvoie la différence entre la date de commande et la date d'expédition en mois.
* Crée un champ calculé lorsque la priorité Critique ou Élevée de la commande devient Importante. Dans les autres cas, définis la commande comme Non importante.
* Ajoute une semaine à la colonne Données de commande.
* Calcule la somme glissante des ventes totales par date de commande.

Pour en savoir plus, voir la rubrique [Assistant d'IA Beast Mode](/s/article/000005304).

Pour en savoir plus sur les autorisations, voir la rubrique [Gestion des rôles personnalisés](/s/article/360043438973).

Pour plus d'informations sur [Domo.AI](https://ai.domo.com/), consultez la documentation [Domo.AI](/s/topic/0TO5w000000ZmGPGA0/domoai) dans la base de connaissances.

## Éditeur SQL avancé pour les vues de DataSet avec l'assistant d'IA

L'éditeur SQL avancé de l'Explorateur de vues vous permet de voir le code SQL qui alimente les vues de DataSet. Vous pouvez modifier le code SQL, l'appliquer et observer l'évolution de votre requête.

Par ailleurs, vous pouvez utiliser l'assistant d'IA pour créer une requête SQL pour les données que vous souhaitez inclure dans votre vue de DataSet. Écrivez une invite en langage naturel et l'assistant d'IA renvoie une requête SQL.

Voici quelques exemples de demandes:

* Montre-moi toutes les lignes dont l'ordre de priorité est élevé ou critique et où la valeur de contrat totale est supérieure à 10 000.
* Nettoie ma colonne Date de mise à jour et utilise le format AAAA-MM-JJ.
* Crée une nouvelle colonne calculée appelée Bénéfice qui représente le total des ventes moins le coût total.
* Associe le DataSet Détails du client dans le domaine du client et inclus toutes les colonnes des deux DataSets.

Pour en savoir plus, consultez la rubrique [Éditeur SQL avancé avec l'assistant d'IA](/s/article/360046074774#sql_editor_and_text-to-sql).

Pour plus d'informations sur [Domo.AI](https://ai.domo.com/), consultez la documentation [Domo.A](/s/topic/0TO5w000000ZmGPGA0/domoai) dans la base de connaissances.

### Flux de travail

Les flux de travail permettent d'exploiter d'autres fonctionnalités telles que le [moteur de code](#code_engine), les [files d'attente](#queues) et les [formulaires](#forms), et de concevoir et de mettre en œuvre des processus de flux de travail exécutables avec peu de code. Vous pouvez démarrer un flux de travail manuellement par l'intermédiaire d'une alerte, d'un flux d'approbation ou d'une minuterie. Pour un démarrage rapide, utilisez notre bibliothèque des modèles de flux de travail.

Pour en savoir plus, voir notre [Présentation des flux de travail](/s/article/000005108) dans la base de connaissances.

### Moteur de code

Le moteur de code est un environnement runtime natif de Domo qui peut exécuter du code JavaScript ou Python, et interagir avec n'importe quelle fonctionnalité à l'intérieur ou à l'extérieur de Domo. Vous pouvez utiliser la bibliothèque Domo de packages et de fonctions pour les intégrations et les services courants, ou créer vos propres fonctions à utiliser dans votre instance Domo. Les fonctions du moteur de code s'intègrent avec les flux de travail sans codage.

Pour en savoir plus, voir la rubrique [Moteur de code](/s/article/000005173).

### Files d'attente

Les files d'attente vous permettent d'organiser les tâches manuelles dans des files distinctes et de contrôler les utilisateurs qui peuvent y accéder. Vous pouvez configurer une file d'attente pour qu'elle affiche les tâches d'un individu ou d'un groupe. Chaque tâche est associée à un formulaire généré par un flux de travail. Vous pouvez filtrer les files d'attente pour afficher les tâches par personne assignée ou par statut, surveiller le volume des files d'attente et personnaliser les dispositions.

Pour en savoir plus, voir [Files d'attente](/s/article/000005172).

### Connecteurs JSON sans code et JSON sans code OAuth

Les connecteurs JSON sans code et JSON sans code OAuth vous permettent de connecter Domo à des API REST qui renvoient des données dans un format JSON standard. Ces deux connecteurs prennent en charge les méthodes d'autorisation standard telles que l'authentification de base, la clé d'API, le jeton Bearer et OAuth 2.0. Chaque connecteur comprend trois zones principales de configuration: Type de connexion, Type de demande et Paramètres avancés en option.<br/>
Ces deux connecteurs sont disponibles au téléchargement dans l' [Appstore Domo](https://www.domo.com/appstore/) [<InlineImage src="/images/kb/ka0Vq00000001FR-00N5w00000Ri7BU-0EMVq0000000XDh.jpg" />.](https://www.domo.com/appstore/)

Pour en savoir plus, consultez les rubriques [Connecteur JSON sans code](/s/article/360043433473) et [Connecteur JSON sans code OAuth](/s/article/360052105454).

## Jauges Entre ou Dans la plage

Les jauges Entre ou Dans la plage vous aident à visualiser les résultats compris entre les limites inférieure et supérieure. Elles indiquent si le résultat est compris dans la plage cible, ce qui vous permet d'identifier rapidement les exceptions par rapport aux valeurs attendues. Ce graphique est disponible dans la section Jauges d'Analyzer.<br/>
Les tailles de police, la largeur des barres et les couleurs sont toutes personnalisables.

## Studio d'applications

La version bêta ouverte du Studio d'applications est désormais disponible.

Le Studio d'applications est un générateur d'applications à programmation schématisée (low-code) qui rend la création, la distribution et la consommation des applications Domo accessibles à tous. Le Studio d'applications propose des thèmes préconçus et de nombreuses méthodes pour personnaliser l'apparence de vos applications. La création des tableaux de bord prend une nouvelle dimension, avec un niveau de précision et de navigation qui exigeait auparavant l'utilisation de la structure de développement d'applications. Le Studio d'applications est en passe de devenir la destination par excellence pour la présentation et la narration des données, le tout depuis une même application.
