# Graphique à barres horizontales haut-bas

Avec le nouveau graphique à barres horizontales haut-bas qui s'ajoute au graphique à barres verticales haut-bas, vous pouvez afficher des listes plus longues et fournir d'autres détails sur les mesures avec un volume plus important.

<Frame>![Screenshot 2024-04-25 at 11.05.20 AM.png](/images/kb/fr/0EMVq000000pNXF.jpg)</Frame>

Les deux types de graphique à barres verticales et horizontales haut-bas dans Analyzer comprennent les options de style suivantes :

- Par défaut
- Ligne
- Barre
- Barre arrondie
- Bâtons de sucette

Le paramètre de style de barre comprend une option permettant de configurer le pourcentage de hauteur des barres dans le graphique. Vous pouvez également utiliser les colonnes de votre DataSet afin de définir les valeurs haut et bas de votre graphique.

Pour plus d'informations sur les graphiques haut-bas, voir [Graphique haut-bas](/fr/s/article/360043429173).

### Améliorations de la plateforme

# Administration de la passerelle d'API

Les administrateurs peuvent désormais créer et gérer des clients de passerelle d'API dans Domo et les journaux d'activité incluent désormais plus de détails sur les actions des clients.

La nouvelle autorisation Créer des clients OAuth d'API contrôle la capacité de création de clients de passerelle d'API. Les administrateurs disposant de cette autorisation peuvent désactiver l'autorisation pour les autres utilisateurs de leur instance Domo.

Outre ces nouvelles fonctionnalités, les administrateurs peuvent également gérer les clients d'API par programme avec les options suivantes :

- Exploitation et contrôle des autorisations des rôles utilisateur. Pour plus d'informations sur la création de clients, consultez le [portail de développement](http://developer.domo.com/).

- Ajout d'horodatages pour la création de clients.

# Studio d'applications

#### Intégration de publications

L'intégration du Studio d'applications avec Domo Everywhere vous permet de créer une publication directement depuis vos applications de Studio d'applications et de publier le contenu pour les abonnés.

Un abonné peut s'inscrire aux publications individuellement, en tant qu'invité, ou s'abonner automatiquement pour recevoir toutes les publications d'une société de publication fiable. 

Pour en savoir plus sur le Studio d'applications, voir [Studio d'applications | Présentation](/fr/s/article/000005295). 

Pour plus d'informations sur la publication de contenu avec Domo Everywhere, voir [Publication de contenu dans une instance d'abonné](/fr/s/article/360045120554).

<Frame>![Publish App.png](/images/kb/fr/0EMVq000000rqdB.jpg)</Frame>



#### Intégration de la sandbox

La sandbox dans le Studio d'applications vous permet de concevoir et de tester des applications du Studio d'applications dans un environnement distinct et de promouvoir le contenu uniquement lorsqu'il est prêt pour une audience plus large. Lors de la création de référentiels dans la sandbox, vous pouvez choisir l'option Nouvelle application Studio d'applications et promouvoir une ou plusieurs applications spécifiques à des fins de promotion de contenu.

Pour en savoir plus sur le Studio d'applications, voir [Studio d'applications | Présentation](/fr/s/article/000005295). 

Pour plus d'informations, sur la sandbox, voir [Sandbox Domo](/fr/s/article/4403367344023).

<Frame>![c6e84622-7f04-4378-bbc0-8100ec799958.png](/images/kb/fr/0EMVq000000qXcb.jpg)</Frame>



# DomoStats : Rôles, Permissions des rôles et Propriétaires de groupe

Les nouveaux rapports Rôles et Permissions des rôles permettent aux administrateurs de voir les rôles et les permissions pour chaque utilisateur de leur organisation. 

Le rapport DomoStats Propriétaires de groupe permet aux administrateurs de voir les affectations de propriété de groupe, y compris la propriété d'individus et/ou de groupes.

Pour plus d'informations sur DomoStats, voir [Connecteur DomoStats](/fr/s/article/360043433813).

# Flux de travail

#### Type de données de compte

Les flux de travail, le moteur de code et les formulaires prennent désormais en charge un type de données de compte. Ce nouveau type de données vous permet de configurer et de mapper les données de compte dans les étapes de vos flux de travail, dans les fonctions du moteur de code et dans les formulaires. 

Une nouvelle fonctionnalité de recherche vous permet de trouver facilement les comptes accessibles et de les mapper à un paramètre d'une étape de flux de travail. Les paramètres configurés avec un type de données de compte peuvent être soit mappés à une variable de type comparaison (like) soit à une valeur codée en dur à l'aide d'un champ de recherche.

Pour plus d'informations sur les flux de travail, voir [Flux de travail](https://domo-support.domo.com/s/topic/0TO5w000000ZkAMGA0/workflows?language=en_US).

<Frame>![Screenshot 2024-05-01 at 10.47.30 AM.png](/images/kb/fr/0EMVq000000rpKY.jpg)</Frame>



#### Bibliothèque générale du moteur de code

Les packages et les fonctions du moteur de code dans notre bibliothèque générale comprennent désormais de nouveaux noms d'affichage, des titres et des descriptions conviviaux, ainsi que des valeurs exemples d'entrée et de sortie pour chaque type de fonction.

<Frame>![Global Code Engine Clean up — Domo Cards.png](/images/kb/fr/0EMVq000000rtG5.jpg)</Frame>



#### Notifications

Grâce aux nouveaux paramètres de notification disponibles pour les flux de travail, vous pouvez configurer des notifications pour des événements de flux de travail spécifiques.

En fonction de vos préférences, vous pouvez recevoir des notifications par e-mail et par SMS lorsque survient un événement de flux de travail tel qu'une opération terminée avec succès ou un échec pendant l'exécution.

Pour plus d'informations sur les notifications, voir [Configuration des paramètres de notification et d'alerte](/fr/s/article/360042925814).

<Frame>![workflows.png](/images/kb/fr/0EMVq000000rzN7.jpg)</Frame>



#### Accès du créateur de tâches

La fonctionnalité facultative Accès du créateur est désormais disponible dans Sécurité des cessionnaires. Si le paramètre Accès du créateur est activé, les utilisateurs sont en mesure de voir les tâches qu'ils ont créées et qui se trouvent dans la file d'attente.

# Améliorations d'Analyzer

#### Propriétés contextuelles des graphiques sensibles

Les propriétés contextuelles des graphiques sensibles vous permettent de cliquer avec le bouton droit sur les éléments de graphique dans Analyzer et de voir la liste des propriétés de graphique qui s'appliquent à cet élément. Dans l'image ci-dessous, un clic droit sur l'axe des Y affiche la propriété du graphique Échelle de catégorie (Y).

<Frame>![Screenshot 2024-05-01 at 2.45.36 PM.png](/images/kb/fr/0EMVq000000s4GL.jpg)</Frame>



#### Types de saisie de chiffres

Tous les types de graphiques dans Analyzer comprennent désormais une fonctionnalité de validation pour les attributs de graphique qui exigent la saisie de chiffres. Cette validation garantit que vous saisissez des valeurs numériques correctes dans le champ des attributs de graphique.

Dans l'exemple ci-dessous, l'attribut de pourcentage de hauteur exige une entrée numérique comprise entre zéro et cent. Si la valeur saisie est en dehors de cette place, l'attribut est indiqué en rouge et aucun rendu du graphique n'est effectué.

<Frame>![Screenshot 2024-05-01 at 2.55.52 PM.png](/images/kb/fr/0EMVq000000s4oD.jpg)</Frame>



#### Rendu rapide

La nouvelle fonctionnalité Rendu rapide d'Analyzer modifie les propriétés d'un graphique afin qu'elles ne nécessitent plus de nouvelles données à extraire à chaque chargement, ce qui diminue le temps de rendu global. Pour les graphiques qui utilisent des DataSets volumineux, la fonctionnalité Rendu rapide vous permet de mettre à jour plusieurs éléments du graphique, sans qu'il faille attendre les actualisations répétées du graphique.



# Masquage des colonnes

La fonctionnalité Masquage des colonnes dans les DataSets vous permet de masquer les informations sensibles telles que les informations personnelles identifiables et les données de santé protégées. Vous pouvez utiliser le masquage des colonnes pour expurger des valeurs intégralement ou pour masquer un certain nombre de caractères dans un champ donné.

Vous pouvez configurer le masquage des colonnes dans la section PDP de l'écran Détails du DataSet.

Pour plus d'informations sur le [masquage des colonnes](/fr/s/article/360042934614) et PDP, consultez la base de connaissances.

<Frame>![ColumnMasking (1).png](/images/kb/fr/0EMVq000000xyD3.jpg)</Frame>

Les tableaux de bord intégrés en privé prennent désormais en charge les rapports planifiés.
