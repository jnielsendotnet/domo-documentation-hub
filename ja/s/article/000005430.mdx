### Domo. AI

<Note>
  **セキュリティに関する検討事項についての注記 —** Domoは、AIがお客様の社内ガイドラインに沿って適切に管理されている必要があることを認識しています。AIアシスタントは、Domoデータソースに関するDomoメタデータのみをOpenAIに送信します。

  そのため、管理者がDomoでAIアシスタントを使用するには、管理者設定でAIサービスレイヤーにおけるOpenAIモデルの使用を有効にする必要があります。さらに、Domoの管理者以外のユーザーがAIアシスタントを活用するには、「AIサービスを利用する」の許可が有効になっている権限が必要です。

  今後数ヶ月以内に、DomoはOpenAIに情報を送信せずに活用できる、AIアシスタント実行用の内部LLMオプションを導入する予定です。

</Note>

# Beast Mode AIアシスタント

Domo. AIのBeast Mode AIアシスタントは、Beast Modeエディター内でBeast Mode計算を生成するために役立ちます。Beast Mode AIアシスタントを使用するには、AIサービスの許可が有効な権限を持っている必要があります。

例えば次のようなリクエストができます。

* 「Return the difference between the Order Date and the Shipped Date in months（注文日と出荷日の差を月単位で返して）」
* 「Create a calculated field where if the order priority is Critical or High it becomes Important, otherwise it's Not Important（計算フィールドを作成して、注文の優先度が『重要』または『高』の場合には『重要』、それ以外の場合は『重要でない』に設定して）」
* 「Add one week to the Order Data column（注文データ列に1週間を追加して）」
* 「Calculate the running sum of total sales by order date（注文日別の総売上高の累計を計算して）」

詳細については、「 [Beast Mode AIアシスタント](/s/article/000005304) 」を参照してください。

許可の詳細については、「 [カスタマイズされた権限を管理する](/s/article/360043438973) 」を参照してください。

詳細については、 [Domo. AI](https://ai.domo.com/jp) またはナレッジベースの [Domo. AIに関する記事](https://domo-support.domo.com/s/topic/0TO5w000000ZmGPGA0/domoai?language=ja) を参照してください。

# AIアシスタントを使用したDataSetビューの高度なSQLエディター

Views Explorerの高度なSQLエディターを使用すると、DataSetビューを作成しているSQLを表示できます。SQLを編集して適用し、クエリの更新を監視できます。

さらに、AIアシスタントを使用して、DataSetビューに含めるデータのSQLクエリを作成できます。自然言語でプロンプトを記述すると、AIアシスタントはSQLクエリを返します。

例えば次のようなリクエストができます。

* 「Show me all rows where Order Priority is High or Critical and TCV is greater than 10000（注文優先度が『高』または『重要』でTCVが10,000より大きいすべての行を表示して）」
* 「Clean up my Updated Date column so that it is in this format YYYY-MM-DD（更新日列をクリーンアップしてYYYY-MM-DDのフォーマットにして）」
* 「Create a new calculated column called Profit that takes Total Sales minus Total Cost（総売上高から総コストを差し引いた、『利益』という名前の新しい計算列を作成して）」
* 「Join with the Customer Details DataSet on Customer Domain and include all columns from both DataSets（顧客ドメインの顧客の詳細DataSetと結合して、両方のDataSetのすべての列を含めて）」

詳細については、 [AIアシスタントを使用した高度なSQLエディター](/ja/s/article/360046074774#sql_editor_and_text-to-sql) に関する記事を参照してください。

詳細については、 [Domo. AI](https://ai.domo.com/jp) またはナレッジベースの [Domo. AIに関する記事](https://domo-support.domo.com/s/topic/0TO5w000000ZmGPGA0/domoai?language=ja) を参照してください。

### ワークフロー

ワークフローでは、 [コードエンジン](#code_engine) 、 [キュー](#queues) 、 [フォーム](#forms) などのほかの新機能を強力に活用して、実行可能なワークフロープロセスをローコード機能で設計および実装することができます。ワークフローは、手動、アラート、承認フロー、またはタイマーで開始することができます。ワークフローテンプレートライブラリを使用すれば、すぐに作業を開始できます。

詳細については、ナレッジベースの「 [ワークフローの概要](/ja/s/article/000005108) 」を参照してください。

### コードエンジン

コードエンジンは、Domoネイティブのランタイム環境で、JavaScriptまたはPythonコードを実行し、Domo内外の任意の機能を操作することができます。一般的な統合とサービスのためのパッケージと関数のDomoライブラリを使用するか、Domoインスタンスで使用する独自の関数を作成することができます。コードエンジンの関数は、コーディングなしでワークフローと統合できます。

詳細については、「 [コードエンジン](/ja/s/article/000005173) 」を参照してください。

### フォーム

新しいフォームエディターを使用すると、カスタマイズされた手動操作用のフォームを作成して公開することができます。フォームはワークフローとスムーズに統合されるため、ワークフローのユーザータスクの一部として再利用可能なフォームを作成および管理したり、フォーム送信を使用してワークフローを開始したりできます。

詳細については、「 [フォーム](/ja/s/article/000005171) 」を参照してください。

### キュー

キューを使用すると、手動タスクを様々なキューに整理し、それらにアクセスできるユーザーを制御できます。キューを設定して、個人またはグループのタスクを表示できます。各タスクはワークフローによって生成されたフォームに関連付けられ、キューにフィルターをかけて、タスクを担当者別またはステータス別に表示したり、キューのボリュームを監視したり、レイアウトをカスタマイズしたりできます。

詳細については、「 [キュー](/ja/s/article/000005172) 」を参照してください。

### JSON No Code/JSON No Code OAuthコネクター

JSON No Code/JSON No Code OAuthコネクターを使用すると、標準JSONフォーマットでデータを返すREST APIにDomoを接続できます。どちらのコネクターも、基本認証、APIキー、Bearerトークン、OAuth 2.0などの標準認証方式をサポートしています。各コネクターの設定画面には、接続タイプ、リクエストタイプ、オプションの詳細設定の3つのメインエリアがあります。

どちらのコネクターも、 [Domo Appstore](https://www.domo.com/appstore/) [<img src="/images/kb/ja/ka0Vq0000000rCS-00N5w00000Ri7BU-0EMVq000000CWli.jpg" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/> からダウンロードできます。](https://www.domo.com/appstore/)

詳細については、「 [JSON No Codeコネクター](/s/article/360043433473) 」および「 [JSON No Code OAuthコネクター](/s/article/360052105454) 」を参照してください。

# 範囲間または範囲内パネル

範囲間または範囲内パネルを使用すると、下限と上限の間の結果を可視化できます。出力が目標範囲内にあるかどうかが示されるため、期待値に対する例外をすばやく特定できます。このチャートは、Analyzerの［パネル］セクションで利用できます。

フォントサイズ、バーの幅、色はすべてカスタマイズ可能です。

# ドーナツグラフのラベル

人気のドーナツグラフをよりプレゼンテーションに適したものにするために、インタラクティブな分析用のデータラベルとホバーテキストオプションを追加しました。データラベルを使用すると、静的フォーマットのドーナツグラフでも、強力なインサイトを得ることができます。

ドーナツグラフにデータラベルを追加するには、Analyzerの［チャートの設定］で［データラベルの設定］を探します。

# 豊富な表の境界線

Mega表とピボット表では、次のような境界線のカスタマイズが可能になりました。

# Domo. AIの機能拡張

Amazon Bedrck MLコネクターとDatabricks MLコネクターを使用すると、サードパーティのデータモデルをDomo. AIソリューションで使用できます。コネクターは、Domoでアカウントを設定するために元のプロバイダーとは異なるキーを必要とします。アカウント設定は、Domo管理者設定で行うことができます。既存のモデルとDomoの機能を組み合わせることで、データの可視化やAIプロジェクトの作成などを行うことができます。

# App Studio
