## はじめに

Microsoft Teamsは、持続的なワークプレイスチャット、ビデオ会議、ファイルストレージ（ファイル上でのコラボレーションを含む）、アプリケーション統合を組み合わせた、コミュニケーションとコラボレーションのためのプラットフォームです。このサービスは、Office 365サブスクリプションのOffice生産性スイートと統合され、Microsoft以外の製品と統合できる拡張機能を備えています。Microsoft TeamsはSlackなどのビジネスコミュニケーションサービスの競合製品であり、Microsoft Skype for Businessから進化およびアップグレードしたものです。DomoのMicrosoft Teamsコネクターを使用すると、組織内のチームやグループでのチャットに関する詳細を取得できます。Microsoft Teams APIの詳細については、 [https://docs.microsoft.com/ja-jp/graph/teams-concept-overview](https://docs.microsoft.com/ja-jp/graph/teams-concept-overview "https://docs.microsoft.com/ja-jp/graph/teams-concept-overview") を参照してください。

Microsoft Teamsアカウントには、Data Centerから接続します。このトピックでは、Microsoft Teamesコネクターのユーザーインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加や更新スケジュールの設定、DataSet情報の編集に関する一般的な情報は、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

## 必要条件

Microsoft Teamsアカウントに接続してDataSetを作成するには、以下が必要です。

* Microsoft Azure Appのインストール。
* Azure AppのクライアントID。
* Azure Appのクライアントシークレット。これは、Azure Appを作成するときに生成されます。

#### クライアントIDを調べるには、以下の手順を実行します。

1. [https://portal.azure.com](https://portal.azure.com/ "https://portal.azure.com")
   にアクセスし、Microsoft認証情報を使用してサインインします。
2. ********［Azure Active Directory］>［App registrations］********
   を選択します。
3. お使いのAppを探し、クリックします。
4. Web Appの「Application ID」を確認します（これがクライアントIDです）。

#### 新しいクライアントシークレットを生成するには、以下の手順を実行します。

1. [https://portal.azure.com](https://portal.azure.com/ "https://portal.azure.com")
   にアクセスし、Microsoft認証情報を使用してサインインします。
2. ********［Azure Active Directory］>［App registrations］********
   を選択します。
3. お使いのAppを探し、クリックします。
4. ********［Certificates and secrets］********
   を開きます。
5. ［Client Secret］セクションで
   **［+New Client Secret］** をクリックします。

### Enterprise OAuth用Azure Appを作成する

Microsoft Azure Appを作成するには、以下の手順を実行します。

1. [https://portal.azure.com](https://portal.azure.com/ "https://portal.azure.com") にログインします。

2. ********［App registrations］******** をクリックします。

<Frame><img alt="Capture1.png" src="/images/kb/ka05w000001236O-00N5w00000Ri7BU-0EM5w000005vNz7.png"/></Frame>

3. ********［New registration］******** をクリックします。

4. アプリケーション名を入力し、サポートされているアカウントタイプを選択します。ここでは、サポートされているアカウントはマルチテナントである必要があります。

<Frame><img alt="clipboard_eb9eab104154ca1097aaf631fb1227879.png" src="/images/kb/ka0Vq0000000NXh-00N5w00000Ri7BU-0EM5w000005vNzv.png"/></Frame>

5. リダイレクト URI
[https://oauth.domo.com/api/data/v1/oauth/providers/microsoft-teams/exchange](https://oauth.domo.com/api/data/v1/oauth/providers/microsoft-reports/exchange "https://oauth.domo.com/api/data/v1/oauth/providers/microsoft-teams/exchange") を入力します。

<Frame><img alt="clipboard_e3ca801b61e568ac88bbaaf31c0352398.png" src="/images/kb/ka0Vq0000000NXh-00N5w00000Ri7BU-0EM5w000005vO06.png"/></Frame>

6. ********［Register］******** をクリックします。

7. アプリケーションが登録されたら、必要なAPI権限をアプリケーションに割り当てます。

8. **［API Permissions］>［+Add a permission］** を選択します。

<Frame><img alt="API_Permission-Add_Permission.png" src="/images/kb/ka05w000001236O-00N5w00000Ri7BU-0EM5w000005vNyw.png"/></Frame>

9. API権限要求ページが表示され、よく使われるMicrosoft APIが表示されます。

<Frame><img alt="Request_API_permissions-2.png" src="/images/kb/ka05w000001236O-00N5w00000Ri7BU-0EM5w000005vNz1.png"/></Frame>

10. 目的のAPIを選択し、必要に応じてユーザーに委任されたアプリケーションの権限を追加します。

<Frame><img alt="clipboard_e3d304305b31d867dddacb886005e89dc.png" src="/images/kb/ka0Vq0000000NXh-00N5w00000Ri7BU-0EM5w000005vO00.png"/></Frame>

11. **［Grant admin consent］** をクリックし、 **［Yes］** をクリックすると、権限に対する管理者の同意が得られます。 追加された権限は、詳細とともに **［Configured permissions］** セクションに表示されます。

<Frame><img alt="clipboard_e4414e0905f035dd543503e9d06485c22.png" src="/images/kb/ka0Vq0000000NXh-00N5w00000Ri7BU-0EM5w000005vNzy.png"/></Frame>

12. 次に、
****［Certificates & Secrets］**** をクリックします。証明書は、トークンを要求するときにアプリケーションの識別情報を証明するためのシークレットとして使用できます。また、パブリックキーと呼ばれることもあります。

<Frame><img alt="Capture4.PNG" src="/images/kb/ka05w000001236O-00N5w00000Ri7BU-0EM5w000005vNyu.png"/></Frame>

13. ****［+ New Client Secret］**** ボタンをクリックします。

<Frame><img alt="Capture5.png" src="/images/kb/ka05w000001236O-00N5w00000Ri7BU-0EM5w000005vNz3.png"/></Frame>

14. クライアントシークレットの説明と有効期限を指定し、
********［Add］******** をクリックします。

<Frame><img alt="Capture6.PNG" src="/images/kb/ka05w00000123LH-00N5w00000Ri7BU-0EM5w000005vO62.png"/></Frame>

15. **値** をコピーして、Domoのコネクター認証情報セクションの ********［クライアントシークレット］******** フィールドに貼り付けます。

<Frame><img alt="Secret.png" src="/images/kb/ka0Vq0000000NXh-00N5w00000Ri7BU-0EM5w000005vO03.png"/></Frame>

## Microsoft Teamsアカウントに接続する

このセクションでは、 Microsoft Teams コネクターページの **［認証情報］** および **［詳細］** ペインのオプションについて説明します。このページのほかのペインの項目である **［スケジュール設定］** や **［DataSetに名前を付け、説明を入力］** に関しては、ほとんどのタイプのコネクターで共通しています。詳細については、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。
