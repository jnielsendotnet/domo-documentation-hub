## はじめに

Adobe Targetを使用すると、A/Bテストを作成したり、カスタマイズしたエクスペリエンスをアプリケーションに埋め込んだりすることができます。DomoのAdobe Targetコネクターにより、キャンペーン、コンテンツ、パフォーマンスに関するデータを表示することができます。ターゲットとするマーケット向けのパッケージを作成する方法を学習しながら、自信を持ってマーケティング業務に取り組むことができます。Adobe Target APIの詳細については、 [https://developers.adobetarget.com/](https://developers.adobetarget.com/ "https://developer.omniture.com") を参照してください。

Data Centerで、Adobe Tagetアカウントに接続することができます。このトピックでは、Adobe Targetコネクターのユーザーインターフェース固有のフィールドとメニューについて説明します。DataSetの追加や更新スケジュールの設定、DataSet情報の編集に関する一般的な情報は、「 [データコネクターを使用してDataSetを追加する](/ja/s/article/360042926274) 」を参照してください。

## 必要条件

Adobe Targetアカウントに接続してDataSetを作成するには、以下の情報が必要になります。

* クライアントID（APIキー）
* クライアントシークレット
* テクニカルアカウントID
* 組織ID
* テナント
* シークレットキー

#### クライアントIDの取得方法：

* [Adobeによる説明](https://www.adobe.io/authentication/auth-methods.html#!AdobeDocs/adobeio-auth/master/AuthenticationOverview/ServiceAccountIntegration.md "https://www.adobe.io/authentication/auth-methods.html#!AdobeDocs/adobeio-auth/master/AuthenticationOverview/ServiceAccountIntegration.md")
  に従い、[Adobe I/O Console](https://console.adobe.io/integrations "https://console.adobe.io/integrations") で新しい統合環境を作成します。
* パブリックキーとプライベートキーのペアを作成し、パブリックキーをAdobeにアップロードして、プライベートキーをDomoにダウンロードします。
* **［Adobe統合環境の概要］** ページで、**APIキー（クライアントID）、テクニカルアカウントID、組織ID、** および **クライアントシークレット** を確認することができます。
* Domoは、ユーザーが入力した認証情報を使用してJSONウェブトークンを作成し、プライベートキーを使用してJWTトークンをアクセストークンと交換することにより、Adobe Analytics APIにアクセスします。
* 詳細については、[Adobeのウェブサイト](https://www.adobe.io/authentication/auth-methods.html#!AdobeDocs/adobeio-auth/master/AuthenticationOverview/ServiceAccountIntegration.md "https://www.adobe.io/authentication/auth-methods.html#!AdobeDocs/adobeio-auth/master/AuthenticationOverview/ServiceAccountIntegration.md") を参照してください。

#### クライアントシークレット、テクニカルアカウントID、組織IDの取得方法：

[Adobeによる説明](https://www.adobe.io/authentication/auth-methods.html#!AdobeDocs/adobeio-auth/master/AuthenticationOverview/ServiceAccountIntegration.md "https://www.adobe.io/authentication/auth-methods.html#!AdobeDocs/adobeio-auth/master/AuthenticationOverview/ServiceAccountIntegration.md") に従って [Adobe I/O Console](https://console.adobe.io/integrations "https://console.adobe.io/integrations") で新しい統合環境を作成すると、クライアントシークレット、テクニカルアカウントID、および組織IDが

**［Adobe統合環境の概要］** ページに表示されます。

## Adobe Targetアカウントに接続する

このセクションでは、Adobe Targetコネクターページの **［認証情報］** ペインと **［詳細］** ペインに表示されるオプションについて説明します。このページの他のペインの項目にある **［スケジュール設定］** や **［DataSetに名前を付け、説明を入力］** に関しては、コネクタータイプのほとんどで共通です。詳細につきましては「 [データコネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "データコネクターを使用してDataSetを追加する") 」をご覧ください。

### ［認証情報］ ペイン

このペインには、Adobe Targetアカウントの認証情報を入力するためのフィールドが表示されます。下表では、各フィールドに必要な内容を説明しています。

| フィールド           | 説明                                                                                                                                                                                                      |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| クライアントID（APIキー） | クライアントIDを入力します。詳しくは、 上記のセクション を参照してください。                                                                                                                                                                |
| クライアントシークレット    | クライアントシークレットを入力します。詳しくは、 上記のセクション を参照してください。                                                                                                                                                            |
| テクニカルアカウントID    | テクニカルアカウントIDを入力します。詳しくは、 上記のセクション を参照してください。                                                                                                                                                            |
| 組織ID            | 組織IDを入力します。詳しくは、 上記のセクション を参照してください。                                                                                                                                                                    |
| テナント            | Adobeのテナント名を入力します。Adobe Experience CloudにログインするためのURLの最初の部分がテナント名になります。例えば「 `[https://tenant-name.marketing.adobe.com](https://tenant-name.marketing.adobe.com)` 」というURLの場合、「tenat-name」の部分がテナント名になります。 |
| シークレットキー        | Adobe統合環境のPEM形式のプライベートキーを貼り付けます。                                                                                                                                                                        |

Adobe Targetの正しい認証情報を入力すると、同じアカウントを使用してAdobe Taeget用の新しいDataSetをいつでも作成することができます。コネクターアカウントは、Data Centerの **［アカウント］** タブで管理できます。このタブの詳細については、「 [コネクターのユーザーアカウントを管理する](/ja/s/article/360042926054 "コネクターのユーザーアカウントを管理する") 」を参照してください。
