## はじめに

Azure Data Lake Gen2 OAuthコネクターは、ビッグデータ解析ワークロード用の全社規模のハイパースケールリポジトリです。これにより、Azure Storageが、Azure上にエンタープライズデータレイクを構築するための基礎になります。大量のデータを簡単に管理することによって、PoC（導入検証）から生産への移行を迅速に行うことができます。データファイルは、CSV、XLS、XLSX、JSONのいずれかのファイルタイプで取得できます。AzureのAPIの詳細については、 [https://docs.microsoft.com/en-us/azu...ta-lake-store/](https://docs.microsoft.com/en-us/azure/data-lake-store/ "https://docs.microsoft.com/en-us/azure/data-lake-store/") を参照してください。
  
Microsoft Azure Data Lake Gen2 OAuthアカウントには、Data Centerから接続します。このトピックでは、Microsoft Azure Data Lake Gen2 OAuthコネクターのユーザーインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加や更新スケジュールの設定、DataSet情報の編集に関する一般的な情報は、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

## 必要条件

Microsoft Azure Data Lake Gen2 OAuthアカウントに接続してDataSetを作成するには、以下が必要です。

* Microsoft Azure AppクライアントID
* Microsoft Azure Appクライアントシークレット
* テナントID

認証情報を取得する場合は、Microsoftのアカウント担当者にお問い合わせください。

### トラブルシューティング

#### コネクターを使用して認証しようとすると、次のエラーメッセージが表示されます。

<Frame><img alt="OAuthエラー" src="/images/kb/ka0Vq0000003Qaf-00N5w00000Ri7BU-0EM5w000006uERU.jpg" style={{width: 483, height: 423}}/></Frame>

この問題は権限に関連しています。Azure Storageにアクセスするには適切な権限が必要です。権限がないと、接続に失敗します。

必要な権限を追加するには、次の手順を実行します。
  
**［App］>［API Permissions］>［Add a permission］>［Select Azure Storage］>［Check user\_impersonation］>［Add permissions］** の順に選択します。
