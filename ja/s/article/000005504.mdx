# スケジュールレポートのアラート送信

アラートの設定で、カードとDataSetのスケジュールレポートを送信する時間と頻度を指定できるようになりました。

アラートの詳細については、「 [Alert Centerのレイアウト](/ja/s/article/360043430373) 」を参照してください。

# 水平方向の高低線チャート

垂直方向の高低線チャートに加えて、水平方向の高低線チャートも利用できるようになります。このチャートでは、より長いリストを表示して、量が増加したメトリクスに関する詳細情報を提供することができます。

<Frame><img alt="Screenshot 2024-04-25 at 11.05.20 AM.png" src="/images/kb/ka0Vq0000001Im5-00N5w00000Ri7BU-0EMVq000000pNXF.jpg" style={{width: 925, height: 433}}/></Frame>
Analyzerの垂直方向と水平方向の高低線チャートタイプには、以下のスタイルオプションがあります。

* デフォルト
* 折れ線グラフ
* 棒グラフ
* 丸い棒グラフ
* ロリポップチャート

バースタイルの設定には、チャート上のバーの高さをパーセンテージで設定するオプションがあります。DataSetの列を使用して、チャートの最大値と最小値を設定することもできます。

高低線チャートの詳細については、 [「高低線グラフ」を参照してください。](/ja/s/article/360043429173)

### プラットフォームの機能拡張

# APIゲートウェイ管理

管理者は、DomoでAPIゲートウェイのクライアントを作成および管理できるようになり、アクティビティログにクライアントのアクションに関する詳細が含まれるようになりました。

新しい「API OAuthクライアントを作成」権限で、APIゲートウェイのクライアントを作成する機能を制御することができます。この権限を持つ管理者は、Domoインスタンス内のほかのユーザーの権限を無効にすることができます。

これらの新機能に加え、管理者はAPIクライアントをプログラムで管理することもできます。これらのオプションには以下のものがあります。

* ユーザー権限の許可の活用および制御。クライアントの作成方法の詳細については、「
  [デベロッパーポータル](http://developer.domo.com/) 」を参照してください。

* クライアントの作成時に日付とタイムスタンプを追加。

# App Studio

#### パブリッシュ統合

App StudioとDomo Everywhereの統合 により、App Studio Appから直接パブリケーションを作成し、コンテンツをサブスクライバーに公開できるようになりました。

サブスクライバー は、招待に応じて個別にサブスクライブすることも、信頼できる公開組織からのすべてのパブリケーションを受信するように自動サブスクライブすることもできます。

App Studioの詳細については、「 [App Studio | 概要](/ja/s/article/000005295) 」を参照してください。

Domo Everywhereを使用してコンテンツを公開する方法については、「 [コンテンツをサブスクライバー組織に公開する](/ja/s/article/360045120554) 」を参照してください。

<Frame><img alt="Publish App.png" src="/images/kb/ka0Vq0000001Im5-00N5w00000Ri7BU-0EMVq000000rqdB.jpg" style={{width: 864, height: 498}}/></Frame>
[このページのトップへ](#top)

#### Sandbox統合

App StudioのSandbox では、App Studio Appを別の環境で構築してテストし、幅広いユーザーの準備が整っている場合にのみコンテンツをプロモートすることができます。Sandboxでリポジトリを作成する際に、新しいApp Studio Appオプションを選択して、コンテンツのプロモーションを目的として特定のAppをプロモートすることができます。

App Studioの詳細については、「 [App Studio | 概要](/ja/s/article/000005295) 」を参照してください。

Sandboxの詳細については、「 [Domo Sandbox](/ja/s/article/4403367344023) 」を参照してください。

新しいRolesおよびRole Grants DomoStatsレポートでは、管理者は組織内のすべてのユーザーの権限と許可を確認できます。

DomoStatsの詳細については、「 [DomoStatsコネクター](/ja/s/article/360043433813) 」を参照してください。

# セッション管理

新しいセッション管理では、セッションの長さの確認と設定を行うことができます。これには、ユーザーのブラウザーやモバイルAppのセッションを確認して終了する機能などが含まれます。ブラウザーセッションとモバイル App セッションのセッションタイムアウトの長さを設定します。

セッション設定の詳細については、「 [セッション管理](/ja/s/article/000005432) 」および「 [セッション設定](/ja/s/article/000005431) 」を参照してください。

# ワークフロー

#### アカウントデータタイプ

ワークフロー、コードエンジン、フォームでアカウントデータタイプがサポートされるようになりました。この新しいデータタイプでは、アカウントデータをワークフローステップ、コードエンジン関数、フォームに設定し、マッピングすることができます。

これには、アクセス可能なアカウントを簡単に検索してワークフローのパラメーターにマッピングできる、新しいルックアップ機能が含まれています。 アカウントデータタイプで構成されたパラメーターは、ルックアップフィールドで同種の変数またはハードコードされた値にマッピングできます。

ワークフローの詳細については、「 [ワークフロー](https://domo-support.domo.com/s/topic/0TO5w000000ZkAMGA0/workflows) 」を参照してください。

# Analyzerの改良

#### Context Sensitive Chart Properties

Context Sensitive Chart Propertiesでは、Analyzer内のチャート要素を右クリックすると、その要素に適用されるチャートのプロパティを一覧表示できます。以下の画像では、Y軸を右クリックしてカテゴリースケール（Y）のチャートのプロパティが表示されています。

<Frame><img alt="Screenshot 2024-05-01 at 2.45.36 PM.png" src="/images/kb/ka0Vq0000001Im5-00N5w00000Ri7BU-0EMVq000000s4GL.jpg" style={{width: 901, height: 423}}/></Frame>
[このページのトップへ](#top)

#### 数値入力タイプ

Analyzerのすべてのチャートタイプに、数値入力が必要なチャート属性を検証する機能が追加されました。この検証により、チャート属性フィールドにクリーンな数値が入力されていることを確認できます。

以下の例では、height percentage属性に0から100の範囲の数値を入力する必要があります。入力された値がその範囲外の場合、属性が赤く強調表示され、チャートはレンダリングされません。

<Frame><img alt="Screenshot 2024-05-01 at 2.55.52 PM.png" src="/images/kb/ka0Vq0000001Im5-00N5w00000Ri7BU-0EMVq000000s4oD.jpg" style={{width: 897, height: 435}}/></Frame>
[このページのトップへ](#top)

#### Quick Render

Analyzerの新しいQuick Render機能では、チャートのプロパティが変更されるため、読み込みごとに新しいデータを取得する必要がなくなり、全体的なレンダリング時間が短縮されます。大規模なDataSetを使用するチャートの場合、Quick Renderによって、チャートの更新が繰り返されるのを待つことなく、チャートの複数の要素を更新できます。

[このページのトップへ](#top)

# 列マスク

DataSetの列マスク機能を使用して、PIIやPHIなどの機密情報を非表示にすることができます。列マスクを使用して、値全体を編集したり、特定のフィールド内の特定の文字数を非表示にしたりすることができます。

列マスクは、DataSetの詳細画面にあるPDPセクションで設定できます。

[列マ](/ja/s/article/360042934614) [スクとPDP](/ja/s/article/360042934614) の詳細については、Knowledge Baseを参照してください。

# プライベートエンベッドのスケジュールレポート

プライベートにエンベッドしたダッシュボードで、スケジュールレポートがサポートされるようになりました。

作成したスケジュールレポートからのインサイトを、エンベッドしたダッシュボードから定期的に直接受け取れるようになりました。レポートを受け取る頻度を設定し、組織内のほかのユーザーとレポートを共有できます。
