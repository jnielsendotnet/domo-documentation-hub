## はじめに

Google BigQueryは、大量の読み出し専用DataSetを処理するクラウドベースのビッグデータ解析ウェブサービスです。DomoのGoogle BigQuery Serviceコネクターを使用すると、指定したプロジェクトからデータを取得することができます。BigQueryのクエリは、標準の SQL SELECT ステートメントのバリエーションを使って記述されています。BigQuery APIの詳細については、ウェブサイト（
https://cloud.google.com/bigquery/docs/reference/v2/
）を参照してください。

Google BigQuery Serviceコネクターは「クラウドApp」コネクターであるため、クラウドに保管されたデータを取得できます。Data Centerのウィンドウ上部のツールバーで **［クラウドApp］** をクリックすると、Google BigQueryを含むほかのクラウドAppコネクターのページにアクセスできます。

このトピックでは、Google BigQuery Serviceコネクターユーザーインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加や更新スケジュールの設定、DataSet情報の編集に関する一般的な情報は、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

## ベストプラクティス

* Domoに取得する *前に* データを事前集計するようにクエリ構造を最適化しておくと、パフォーマンスが大幅に向上します。
* 取り込む前に視覚化できるようにデータを厳密に構造化しておけば、DomoのETLプロセスが不要になる場合があり、時間を節約できます。

## 必要条件

BigQueryサービスアカウントに接続するには、 Google BigQueryサービスアカウントのJSONキーが必要です。キーを生成するには、次の手順を実行します。

1. Google Cloud Platform Consoleで、［IAM & Admin］ページを開きます。
2. 左側のナビゲーションペインで **［Service accounts］** をクリックします。
3. プロジェクトを選択して **［Open］** をクリックします。
4. **［Create Service Account］** をクリックします。
5. サービスアカウントの名前と説明を入力します。
6. **［Create］** をクリックします。

   <Note>
   **注記：** サービスアカウントの許可ダイアログで「BigQuery Admin」権限が必要になる場合があります。詳細についてはGoogle管理者にお問い合わせください。

   </Note>

7. **［Project］>［Owner］** の順に選択します。
8. **［Continue］** をクリックします。
9. **［Create key］** をクリックします。
10. キーのタイプに **［JSON］** を選択します。
11. **［Create］** をクリックします。

秘密キーがコンピューターに保存されます。

### Using the Query Parameter

クエリパラメーターは、クエリパラメーターの初期値を示します。初回実行後、クエリは2回目以降の実行時にプロバイダーから更新済みのデータのみを要求します。クエリパラメーターは、カンマで区切って複数指定することができます。

**注記：** 上記の［Query］フィールドには、ここで提供されたパラメーターを使用する必要があります。クエリは、ここで提供されたパラメーター値に従ってデータを取得します。

</Note>

## よくある質問

<AccordionGroup>
<Accordion title="ページサイズはどのようにして決まりますか？">


コネクターの **［詳細］** セクションで設定される **［Max Results］** パラメーターは、結果のページごとに返されるデータの最大行数です。 **［Max Results］** を1,000などの小さい値に設定し、結果を改ページすると、クエリ結果セットが大きい場合の信頼性が向上することがあります。

Domoの **［Max Results］** 行制限以外にも、Googleにも結果のページごとに10 MBのバイト制限があります。行の値に大量のデータが含まれる場合、バイト制限を超過しないよう、返されるページ当たりの行数が少なくなる場合があります。


</Accordion>
<Accordion title="特定のロケーションにあるデータを処理するにはどうすればよいですか？">


DataSetを作成するときに、BigQueryデータを保存するためのクエリレポートで処理を実行するロケーションを指定できます。DataSetを作成した後は、ロケーションを変更できません。特定の場所で実行されるクエリは、そのロケーションのデータのみを参照できます。場所には、リージョンのロケーションとマルチリージョンのロケーションの2種類があります。詳細については、「 [データセットのロケーション](https://cloud.google.com/bigquery/docs/locations "https://cloud.google.com/bigquery/docs/locations") 」を参照してください。


</Accordion>
<Accordion title="DataSetから「大き過ぎて実行できません」のエラーが返されます。クエリで大きな結果を取得するにはどうすればよいですか？">


クエリで大きな結果が予想される場合は、コネクターインターフェースの **［詳細］** セクションに移動し、 **［Allow large results?］** で **［Yes］** を選択し、 **［DataSet Name］** メニューでDataSetを選択します。 **［Allow large results］** を **［Yes］** に設定するには、Googleアカウントで表作成の権限を有効にする必要があります。


</Accordion>
<Accordion title="データはどれくらいの頻度で更新されますか？">


必要に応じて何回でも更新できます。


</Accordion>
<Accordion title="注意すべきAPI制限はありますか？">


ありません

はい
