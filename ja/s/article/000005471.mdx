### はじめに

Azure SQLユーザーは、Cloud Amplifierのネイティブ統合を通じて、自分のデータをDomoに接続できます。まず接続パラメーターを指定し、Domo DataSetとして表示するテーブルを選択します。

### Azure SQL統合を作成する

この記事の手順に従って、読み取り専用のAzure SQL–Domo統合を設定します。

1. Domo Data Centerで <InlineImage src="/images/kb/ka0Vq000000BCNd-00N5w00000Ri7BU-0EMVq000000uR3y.jpg" /> **［Data Warehouse］** に移動します。

2. <InlineImage src="/images/kb/ka0Vq000000BCNd-00N5w00000Ri7BU-0EMVq000000uSeO.jpg" /> **［新しいクラウドアカウントを追加］** を選択します。
     
   <Frame><img alt="add new cloud account.jpg" src="/images/kb/ka0Vq000000BCNd-00N5w00000Ri7BU-0EMVq000000uRaE.jpg" style={{width: 500, height: 452}}/></Frame>

   **［既存のクラウドウェアハウスを拡大する］** モーダルが表示されます。
3. **［ネイティブ統合］** セクションの **［詳細を見る］** を選択して、ほかのオプションを表示します。
     
   <Frame><img alt="see more.jpg" src="/images/kb/ka0Vq000000BCNd-00N5w00000Ri7BU-0EMVq000000uTIf.jpg" style={{width: 500, height: 340}}/></Frame>

4. **［Azure SQL］** を選択します。
     
   <Frame><img alt="choose azure sql.jpg" src="/images/kb/ka0Vq000000BCNd-00N5w00000Ri7BU-0EMVq000001JtzW.jpg" style={{width: 500, height: 375}}/></Frame>

5. **［+新しい統合を追加］** を選択します。
     
   <InlineImage src="/images/kb/ka0Vq000000BCNd-00N5w00000Ri7BU-0EMVq000001K1Yr.jpg" /> <InlineImage src="/images/kb/ka0Vq000000BCNd-00N5w00000Ri7BU-0EMVq000001JrL0.jpg" />
6. Azure SQLデータベースに接続するには、表示されるフィールドに以下の接続パラメーターを入力します。
   * **［統合名］ —** 将来統合を更新する必要が出てきた場合に、統合のリストで検索する名前です。
   * **［統合の説明（オプション）］ —** 統合に関する追加情報。
   * **［Azure SQL Databaseの接続URL］ —** 接続先のAzure SQLインスタンスのホストとポート。例： *< [myhost.com](http://myhost.com):3306>* 。
   * **［Azure SQL Databaseのデータベース名］ —** 接続先のテーブルを含むデータベースの名前。
   * **［Azure SQL Databaseユーザー名］ —** 接続先のデータにアクセスできるAzure SQLインスタンスのユーザー名。
   * **［Azure SQL Databaseパスワード］ —** Azure SQLユーザーのパスワード。
       
     <Frame><img alt="azure sql.jpg" src="/images/kb/ka0Vq000000BCNd-00N5w00000Ri7BU-0EMVq000001JwKg.jpg" style={{width: 599, height: 442}}/></Frame>

7. フォームの下部のキャッシュオプションを設定し、**［次へ］** を選択します。Cloud Amplifierのキャッシュのルールについては、 [こちら](/ja/s/article/4412849158167#caching_rules) を参照してください。
     
   <Frame><img alt="Screenshot 2024-05-07 at 3.24.52 PM.png" src="/images/kb/ka0Vq000000BCNd-00N5w00000Ri7BU-0EMVq000000uWur.jpg" style={{width: 549, height: 156}}/></Frame>

   成功メッセージが表示されます。
8. **［Choose Tables to Connect］** を選択して、テーブルピッカーを開きます。
     
   <Frame><img alt="choose tables.jpg" src="/images/kb/ka0Vq000000BCNd-00N5w00000Ri7BU-0EMVq000001K20H.jpg" style={{width: 600, height: 381}}/></Frame>

9. テーブルピッカーを使用して、スキーマとテーブルを選択します。準備ができたら、**［DataSetを作成］** を選択します。
     
   <Frame><img alt="table picker.jpg" src="/images/kb/ka0Vq000000BCNd-00N5w00000Ri7BU-0EMVq000001L0hB.jpg" style={{width: 600, height: 356}}/></Frame>

10. （オプション）確認モーダルでは、DataSetをほかのDomoユーザーとシェアできます。追加するユーザーを選択し、ドロップダウンでアクセスレベルを選択した後、**［シェア］** を選択します。
      
    または、 **［今はやめておく］** を選択し、アクセス権をシェアせずにモーダルを閉じます。
      
    <Frame><img alt="share modal.jpg" src="/images/kb/ka0Vq000000BCNd-00N5w00000Ri7BU-0EMVq000000uXZB.jpg" style={{width: 550, height: 313}}/></Frame>

DomoでAzure SQL DataSetとやり取りできるようになりました。
