## はじめに

### 使用事例

* クライアントは1つの部署です。もしくは、さらに大規模なグループの1つのカテゴリーを担当しています。
* クライアントは、特定の部署を強調し、ほかの部署と比較しながら「ウォレットのシェア」や「合計に対する割合」を表示したいと考えています。
* また、ほかの部署と比較しながら、自分たちの部署の実際のメトリクス値を表示したいと考えています。

### 現在の状況

* ほとんどの場合、「合計に対する割合」を1つの棒グラフで表示し、メトリクス値を別の棒グラフで表示しなければなりません。
* 特定の部署を強調するには、部署のランクを把握して、その系列色をほかの棒の色とは異なる色に設定する必要があります。

  <Frame><img alt="3metric1.png" src="/images/kb/ka05w00000123pa-00N5w00000Ri7BU-0EM5w000005vPHr.png"/></Frame>
### 理想的な解決策

* 理想的な解決策としては、棒グラフでクライアントの特定の部署に常に固有の色を使用する方法があります。
* ほかにも、1つの棒グラフにメトリクス値と「合計に対する割合」を分かりやすく表示するという解決策もあります。

  <Frame><img alt="3metric2.png" src="/images/kb/ka05w00000123pa-00N5w00000Ri7BU-0EM5w000005vPHq.png"/></Frame>
この記事では、3つのメトリクスの棒グラフを作成して2つ目の解決策を実行する方法を説明します。ここでは、合計4つの部門を持つデパートのキッチン部門を担当するクライアントの例を使用します。クライアントは、自分の部門がほかの部門と比較してどのような状態にあるかを把握する必要がありますが、キッチン部門は常に視覚的に目立たせたいと考えています。

## 設定手順

**3つのメトリクスの棒グラフを設定するには：**

1. 次の手順を実行して、Beast Modeを3つ作成します。

   <table class="mt-responsive-table"><thead><tr><th colspan="1" rowspan="1"><p> Beast Mode名 </p></th><th colspan="1" rowspan="1"><p> 説明 </p></th><th colspan="1" rowspan="1"><p> Beast Mode </p></th></tr></thead><tbody><tr><td colspan="1" rowspan="1"><p> KitchenSales </p></td><td colspan="1" rowspan="1"><p> キッチン部門のみの売上高を表します。 </p></td><td colspan="1" rowspan="1"><p><code> SUM(CASE WHEN `Department` = ‘Kitchen’ THEN `Sales` ELSE 0 END) </code></p></td></tr><tr><td colspan="1" rowspan="1"><p> NonKitchenSales </p></td><td colspan="1" rowspan="1"><p> デパートのキッチン部門以外の売上高を表します。 </p></td><td colspan="1" rowspan="1"><p><code> SUM(CASE WHEN `Department` = ‘Kitchen’ THEN 0 ELSE `Sales` END) </code></p></td></tr><tr><td colspan="1" rowspan="1"><p> ShareOfDepartments </p></td><td colspan="1" rowspan="1"><p> 総売上に対する各部門の割合を表します。 </p></td><td colspan="1" rowspan="1"><p><code> (SUM(Sales) / SUM(Sales) OVER ()) * 100 </code></p><ul><li><p> つまり、すべての部門の合計が表示されます。 </p></li><li><p> 値をパーセント形式で表示するには、100をかけます。 </p></li></ul></td></tr></tbody></table>

2. グラフタイプとして［積上げ横棒グラフ］を選択します。

   通常の棒グラフの代わりに積上げ棒グラフを使用することによって、棒の一部を0とし、棒のほかの部分で実測値を表すことができます。そのため、使用する系統の場所にもとづいて、棒の系統の色を変えることができます。
3. Y軸には、「Department」を選択します。
4. X軸には、上で作成したBeastModeから「KitchSales」を選択します。

   これにより、キッチン部門を除くすべての部門の最初の系統が0になります。
5. 系統には、上で作成したBeastModeから「NonKitchenSales」を選択します。

   これにより、キッチン部門を除くすべての部門の2番目の系統は値を持ちますが、キッチン部門の2番目の系統の値は0になります。
6. 次の系統には、上で作成したBeastModeから「ShareOfDepartments」を選択します。

   この値は最初はグラフに表示されませんが、後のステップでデータラベルを使用してアクセスできます。
7. 「ShareOfDepartments」系統の列ラベルの名前を「シェア」に変更します。
8. ［売上の合計］列を降順にソートします。

   <Frame><img alt="3metric3.png" src="/images/kb/ka05w00000123pa-00N5w00000Ri7BU-0EM5w000005vPHp.png"/></Frame>
   これにより、部門に関係なく、売上額が高い順にデータを表示することができます。
9. グラフの設定を次のように設定します。

   * 一般

     + フォントサイズ – 大きい
     + ポートレートビューでの凡例の位置 – 非表示
     + ランドスケープビューでの凡例の位置 – 非表示
   * データラベルの設定

     + テキスト - `#%_CATEGORY_NUMBER Sales: $%_CATEGORY_TOTAL \\n%_SERIES_NAME: %_VALUE%`

       - `%_{{item}}` は、グラフ内のデータから抽出された値を表します。複数のメトリクスを表示できます。
       - `%_CATEGORY_NUMBER` には、棒のランクが表示されます。
       - `%_CATEGORY_TOTAL` には、棒の合計値が表示されます。
       - `%_SERIES_NAME` には、ラベルの最後の系統の名前が表示されます。
       - `%_VALUE` には、最後の系統の値が表示されます。
       - `\\n` は、改行を表します。
     + 単一ラベル表示 – True

       - 系統ごとにラベルを表示するのではなく、ラベルを1回で表示できます。
     + 位置 – 内側右

       - グラフのラベルの位置を選択できます。
     + 位置調整 – 左

       - ラベルのテキストを左揃えにして、見やすくすることができます。
     + 目盛りの略称を使用 – True

       - 数値が短くなり、読みやすくなります。
     + 小数位 -.0

       - 各ラベルの値を小数位1桁で表示します。
   * 値の目盛り(X)

     + ラベルフォーマット – 通貨

       - ラベルのフォーマットを通貨フォーマットにすることができます。
   * 色

     + 系統1 – 薄緑

       - この色が、キッチン部門のみの売上高を表す棒に使用されます。
     + 系統2 – 明るいグレー

       - この色が、キッチン部門以外のすべての売上高を表す棒に使用されます。
