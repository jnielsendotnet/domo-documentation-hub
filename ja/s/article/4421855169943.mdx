## はじめに

Domoは、オープンソースのオブジェクトリレーショナルデータベースシステムであるPostgreSQLとシームレスに連動します。DomoのPostgreSQL Advancedコネクターを使用すると、PostgreSQLデータベースからデータを取得し、カスタムレポートを作成することができます。すべてのデータとユーザーを1ヶ所にまとめる直感的なプラットフォームにより、リアルタイムなビジネスインサイト間のコラボレーションが容易になり、一元化が進みます。
  
PostgreSQLデータとほかの多数のシステムのデータを容易に組み合わせることができるため、ユーザーはデータを変換して、そのすべてをセルフサービスで利用することができます。必要なメトリクスをどのデバイスからでも確認でき、レポートを自動化し、業務に重要な変更が発生した場合にはアラートを受け取れます。 取得するデータの指定は、SQLクエリの入力で行います。PostgreSQL APIの詳細については、PostgreSQLのウェブサイト（ [https://www.postgresql.org/docs/](https://www.postgresql.org/docs/ "https://www.postgresql.org/docs/") ）を参照してください。

PostgreSQL Advancedコネクターは「データベース」コネクターです。データベースからデータを取得するにはクエリを使用します。Data Centerでは、ウィンドウ上部のツールバーで **［データベース］** をクリックすることで、このコネクターやほかのデータベースコネクターのページにアクセスできます。

PostgreSQLデータベースには、 Data Center から接続します。このトピックでは、PostgreSQLコネクターのユーザーインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加や更新スケジュールの設定、DataSet情報の編集に関する一般的な情報については、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

<Note>

</Note>

## ベストプラクティス

* 読み取り専用クエリのみを使用します。
* 「Append」ではなく「Replace」を使用します。

## 必要条件

PostgreSQLデータベースに接続しDataSetを作成するには、以下が必要です。

* PostgreSQLデータベースのログインに使用するユーザー名とパスワード
* データベースのホスト名
* データベースのポート番号
* データベース名
* CA証明書
* SSL証明書
* SSLキー

PostgreSQLデータベースに接続する前に、データベースサーバーの接続ポートでIPアドレスをいくつかホワイトリストに登録する必要があります。IPアドレスの詳細なリストは、「 [コネクターのIPアドレスをホワイトリストに登録する](/ja/s/article/360043630093 "コネクターのIPアドレスをホワイトリストに登録する") 」を参照してください。

## PostgreSQLデータベースに接続する

このセクションでは、PostgreSQL Advancedコネクターページの **［認証情報］** および **［詳細］** ペインのオプションについて説明します。このページのほかのペインの項目である **［スケジュール設定］** や **［DataSetに名前を付け、説明を入力］** に関しては、ほとんどのコネクターで共通しています。詳細については「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

### ［認証情報］ペイン

このペインには、データベースに接続するための認証情報の入力フィールドが含まれています。以下の表で、各フィールドについて説明します。

| フィールド           | 説明                                                                |
| --------------- | ----------------------------------------------------------------- |
| Username        | PostgreSQLユーザー名を入力します。                                            |
| Password        | PostgreSQLパスワードを入力します。                                            |
| Host            | SQLデータベースのホスト名を入力します。例： `[db.company.com](http://db.company.com)` |
| Port            | データベースのポート番号を入力します。                                               |
| Database        | SQLデータベース名を入力します。                                                 |
| Certificate     | CA証明書のテキストを貼り付けます。                                                |
| SSL Certificate | SSL証明書を貼り付けます。                                                    |
| SSL Key         | SSLキーを貼り付けます。                                                     |

有効なPostgreSQL認証情報を一度入力しておくと、随時同じアカウントを使用して新しいPostgreSQL Advanced DataSetを作成することができます。コネクターのアカウントは Data Center の **［アカウント］** タブで管理できます。このタブの詳細については、「 [コネクターのユーザーアカウントを管理する](/ja/s/article/360042926054 "コネクターのユーザーアカウントを管理する") 」を参照してください。

### ［詳細］ペイン

このペインでは、SQLクエリを作成して、データベースからデータを取得します。

| メニュー                      | 説明                                                                                                                                                                                                                                                                                                                                                                                           |
| ------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Query Type                | <table border="1"><tbody><tr><th colspan="1" rowspan="1"> クエリタイプ </th><th colspan="1" rowspan="1"> 説明 </th></tr><tr><td colspan="1" rowspan="1"> Custom Query </td><td colspan="1" rowspan="1"> SQLステートメントを直接入力します。このオプションはクエリパラメーターもサポートします。 </td></tr><tr><td colspan="1" rowspan="1"> Query Builder </td><td colspan="1" rowspan="1"> 利用可能な表と列のリストからクエリを作成します。 </td></tr></tbody></table> |
| Query                     | 必要なデータを選択するための構造化照会言語（SQL）クエリを入力します。<br/>例： `select * from Employee`                                                                                                                                                                                                                                                                                                                         |
| Query Parameter（チェックボックス） | このチェックボックスを選択してクエリパラメーターを追加します。                                                                                                                                                                                                                                                                                                                                                              |
| Query Parameter           | クエリパラメーターとその初期値をカンマで区切って入力します。有効なクエリパラメータは、lastvalue、lastrundateです。クエリパラメーターの値を設定するには、次の構文を使用します。 !\{lastvalue:_id}!=1,\{lastrundate:start_date}!=02/01/1944                                                                                                                                                                                                                                 |
| Database Tables           | レポートに表示するデータベース表を選択します。                                                                                                                                                                                                                                                                                                                                                                      |
| Table Columns             | レポートに表示したい表の列をすべて選択します。                                                                                                                                                                                                                                                                                                                                                                      |
| Query Helper              | 選択したフィールドにもとづいてクエリが生成されます。このテキストをコピーして、クエリフィールドに貼り付けます。必要に応じてこのクエリを編集します。                                                                                                                                                                                                                                                                                                                    |
| Fetch Size                | メモリのパフォーマンスのフェッチサイズを入力します。このフィールドに値が指定されない場合、コネクターはデフォルト値400を使用します。                                                                                                                                                                                                                                                                                                                          |
| Bigint To Long            | Domoでbigint列を常に長い列に変換する際に選択します。 <br/> これは、bigint列が大きな整数値を格納していても、Domoはその列が長い値を持っていると見なします。                                                                                                                                                                                                                                                                                                   |

## トラブルシューティング

* 適切なIPアドレスがホワイトリストに登録されているか確認します。
* フェッチサイズの縮小を試みるか、空白のままにします。
