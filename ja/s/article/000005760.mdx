## はじめに

Jiraは、チームのすべてのメンバーが作業の計画、追跡、管理を行うために構築されています。Jira on Prem Writebackコネクターを使用すると、オンプレミスのJira環境で新しい課題を作成したり、既存の課題を更新したりできます。

Jira On Prem Writebackアカウントには、Data Centerから接続できます。このトピックでは、Jira On Prem Writebackコネクターユーザーインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加、更新スケジュールの設定およびDataSet情報の編集を行うには、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

## 必要条件

Jira On Prem Writebackアカウントに接続してDataSetを作成するには、以下が必要です。

* JiraドメインURL
* クライアントID
* クライアントシークレット

## Jira On Prem Writebackアカウントに接続する

このセクションでは、Jira On Prem Writebackコネクターページの **［認証情報］** および **［詳細］** ペインのオプションについて説明します。このページのほかのペインの項目である **［スケジュール設定］** や **［DataSetに名前を付け、説明を入力］** に関しては、ほとんどのタイプのコネクターで共通しています。詳細については、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

## ［認証情報］ペイン

このペインには、Jira On Prem Writebackアカウントに接続する認証情報の入力フィールドが含まれています。以下の表で、各フィールドに必要な内容を説明しています。

| フィールド           | 説明                                                                      |
| --------------- | ----------------------------------------------------------------------- |
| Jira Domain URL | JiraドメインURLを入力します。例： [https://onjira.domo.com](https://onjira.domo.com) |
| Client ID       | アプリケーションのクライアントIDを入力します。                                                |
| Client Secret   | アプリケーションのクライアントシークレットを入力します。                                            |

有効なJira On Prem Writeback認証情報を一度入力しておくと、いつでも同じアカウントを使用して新しいJira On Prem Writeback DataSetを作成できます。コネクターのアカウントはData Centerの **［アカウント］** タブで管理できます。このタブの詳細については、「 [コネクター/アダプターアカウントを管理する](/ja/s/article/360042926054 "コネクター/アダプターアカウントを管理する") 」を参照してください。

## ［詳細］ペイン

| **メニュー**          | **説明**                                                                                                                                                                                                                                      |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| DataSet ID        | オンプレミスJira環境に書き込むデータを含むDomo DataSetのIDを入力します。書き戻すDataSetのみ選択できます。<br/>このIDはDataSet URL内にあります。例： [https://customer.domo.com/datasources/](https://customer.domo.com/datasources/)**aaaaaaaa-bbbb-cccc-dddd-eeeeeeeeeeee/** details/settings. |
| Select Project    | プロジェクトを選択します。                                                                                                                                                                                                                               |
| Select Issue type | 課題のタイプを選択します。                                                                                                                                                                                                                               |
| Required Fields   | **［Required Fields］** セクションには、選択した課題のタイプとプロジェクトに必要なフィールドが一覧表示されます。各必須フィールドを、そのフィールドに書き戻すデータを含むDataSet内の列にマップします。                                                                                                                            |
| Optional Fields   | **［Optional Fields］** セクションでは、DataSetの列を、選択したプロジェクトの課題のタイプのオプションフィールドにマップできます。これらのフィールドをマップする必要があるのは、Jiraインスタンスで必須ではないフィールドにデータを書き戻す場合のみです。                                                                                                 |
