## はじめに

このコネクターを使用して、Domo DataSetからMySQLデータベースにデータをエクスポートすることができます。 MySQLの詳細については、MySQLのウェブサイト（ [https://www.mysql.com/](https://www.mysql.com/ "https://www.mysql.com/") ）を参照してください。

MySQLデータベースへのデータのエクスポートはData Centerで行います。このトピックでは、MySQL Advanced Writebackコネクターのユーザーインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加や更新スケジュールの設定、DataSet情報の編集に関する一般的な情報については、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274) 」を参照してください。

## 必要条件

このコネクターを設定するには、次の情報が必要です。

* MySQLデータベースまたはスキーマの名前。
* MySQLデータベースサーバーのホスト名またはIPアドレス（例：
  [db.mycompany.com](http://db.mycompany.com) ）。
* MySQLデータベースサーバーのポート番号。
* MySQLのユーザー名とパスワード。
* MySQLサーバーのポート番号。
* （オプション）
  SSLクライアントキー
* （オプション）SSLクライアント証明書
* （オプション）SSL認証局

<Warning>
**重要** ：MySQLでは以下の権限が必要です。

* FILE
* CREATE
* DROP
* INSERT
* SELECT
* UPDATE

</Warning>

## 接続を設定する

このセクションでは、MySQL Advanced Writebackコネクターページの **［認証情報］** および **［詳細］** ペインのオプションについて説明します。このページのほかのペインの項目である **［スケジュール設定］** や **［DataSetに名前を付け、説明を入力］** に関しては、ほとんどのタイプのコネクターで共通しています。詳細については「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

### ［認証情報］ペイン

このペインには、データのコピー先となるMySQLデータベースのテーブルに接続するための認証情報の入力フィールドがあります。以下の表では、各フィールドに必要な内容を示しています。

| フィールド              | 説明                       |
| ------------------ | ------------------------ |
| Database Name      | MySQLデータベース名を入力します。      |
| Host               | MySQLデータベースのホスト名を入力します。  |
| Username           | MySQLユーザー名を入力します。        |
| Password           | MySQLパスワードを入力します。        |
| ポート                | MySQLデータベースのポート番号を入力します。 |
| SSL Key            | SSLクライアントキーを入力します。       |
| SSL Certificate    | SSL向けのクライアント証明書を入力します。   |
| SSL CA Certificate | SSL向けのMySQL認証局を入力します。    |

これらの認証情報の入手方法については、前述の「必要条件」を参照してください。

有効な認証情報を一度入力しておくと、随時同じアカウントを使用して新しいDomo-MySQL接続を設定できます。コネクターアカウントは、Data Centerの **［アカウント］** タブで管理できます。このタブの詳細は、「 [コネクターのユーザーアカウントを管理する](/ja/s/article/360042926054 "コネクターのユーザーアカウントを管理する") 」を参照してください。

### ［詳細］ペイン

このペインには、データを指定してデータの送信先を示すための多数のフィールドがあります。

| メニュー                                                         | 説明                                                                                                                                                                                                                                                                                                                                                                                         |
| ------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| DataSet ID                                                   | MySQLにコピーするDataSetのDataSet ID（GUID）を入力します。Data CenterでDataSetの詳細ビューを開き、URLの `datasources/` に続く部分を見ると、IDを見つけることができます。例えば、URLが `[https://mycompany.domo.com/datasources/845305d8-da3d-4107-a9d6-13ef3f86d4a4/details/overview](https://mycompany.domo.com/datasources/845305d8-da3d-4107-a9d6-13ef3f86d4a4/details/overview)` であれば、DataSet IDは `845305d8-da3d-4107-a9d6-13ef3f86d4a4` になります。 |
| How Would You Like To Select Your Table?                     | 新しいテーブルを作成してDataSetデータをエクスポートするか、既存のテーブルを選択してDataSetデータをエクスポートするかを指定します。                                                                                                                                                                                                                                                                                                                   |
| Table Name                                                   | データがコピーされるテーブルの命名方法を選択します。<br/>DataSet ID 。テーブル名は ［DataSet ID］ に入力した番号になります。<br/>Custom Name 。 ［Custom Table Name］ フィールドにテーブルのカスタム名を入力します。                                                                                                                                                                                                                                                 |
| Custom Table Name                                            | DataSetデータをコピーするMySQLデータベースのテーブルの名前を入力します。                                                                                                                                                                                                                                                                                                                                                 |
| Delete Existing Table and Create New Table For The Same Name | 通常、既存のテーブルと同じ名前のテーブル名を入力すると、コネクターにエラーが発生します。その場合は、このチェックボックスをオンにすると、既存のテーブルを削除して同じ名前で新規テーブルが作成されます。<br/>**警告** ：この操作で削除された既存のテーブルは復元できません。                                                                                                                                                                                                                                                 |
