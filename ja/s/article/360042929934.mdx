## はじめに

Pinterestは、ユーザーが新しい関心事項を視覚的に共有して発見できるようにするソーシャルネットワークです。Pinterest Enterpriseコネクターを使用すると、フォロワー、フォロー中、ピンなどの多数のPinterestエンティティを含む、様々なPinterestデータを確認することができます。Pinterest APIについての詳細は、PinterestのWebサイト（ [https://developers.pinterest.com/](https://developers.pinterest.com/ "https://developers.pinterest.com/") ）を参照してください。

Pinterestアカウントには、Data Centerから接続します。このトピックでは、Pinterestコネクターユーザーインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加や更新スケジュールの設定、DataSet情報の編集に関する一般的な情報については、「 [データコネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "データコネクターを使用してDataSetを追加する") 」を参照してください。

## 必要条件

Pinterestアカウントに接続してDataSetを作成するには、以下が必要です。

* PinterestクライアントID
* Pinterestクライアントシークレット
* Pinterestアカウントの認証情報（Pinterestへのログインに使用するメールアドレスとPinterestのパスワード）、Facebook、Google認証情報のいずれか

クライアントIDとクライアントシークレットのリダイレクトURIについては、

https://oauth.domo.com/api/data/v1/oauth/providers/pinterest-enterprise/exchange
を参照してください。

クライアントIDとクライアントシークレットの生成方法については、 [https://www.codeholic.in/blog/how-to-generate-pinterest-client-id-and-secret-key/](https://www.codeholic.in/blog/how-to-generate-pinterest-client-id-and-secret-key/ "https://www.codeholic.in/how-to-generate-pinterest-client-id-and-secret-key/") を参照してください。

## Pinterestアカウントに接続する

このセクションでは、Pinterestコネクターページの **［認証情報］** および **［詳細］** ペインのオプションについて説明します。このページの他のペインの項目にある **［スケジュール設定］** や **［DataSetの名前と説明］** に関しては、ほとんどのタイプのコネクターで共通しています。詳細については「 [データコネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "データコネクターを使用してDataSetを追加する") 」を参照してください。

### ［認証情報］ペイン

このペインには、Pinterestアカウントに接続するための認証情報の入力フィールドが表示されます。下の表は、各フィールドに必要な内容を説明しています。

| フィールド         | 説明                           |
| ------------- | ---------------------------- |
| Client ID     | PinterestクライアントIDを入力します。     |
| Client Secret | Pinterestクライアントシークレットを入力します。 |

クライアントIDとクライアントシークレットの生成方法については、
https://www.codeholic.in/blog/how-to-generate-pinterest-client-id-and-secret-key/
を参照してください。

有効なPinterest認証情報を一度入力しておくと、いつでも同じアカウントを使用して新しいPinterest DataSetを作成することができます。コネクターアカウントは、Data Centerの **［アカウント］** タブで管理できます。このタブの詳細については、「 [コネクターのユーザーアカウントを管理する](/ja/s/article/360042926054 "コネクターのユーザーアカウントを管理する") 」を参照してください。

## よくある質問

<AccordionGroup>
<Accordion title="# Pinterestの認証情報が保護されていることをどのように知ることができますか？">


DomoはOAuthプロセスを使用してPinterestに接続するため、認証情報は安全であり、保存されることはありません。DomoのPinterestアカウントへのアクセスはいつでも取り消すことができます。


</Accordion>
<Accordion title="# ボードIDはどこにありますか？">


「Boards」レポートを実行し、「Data Id Str」という列を確認すると、ボードIDを見つけることができます。


</Accordion>
<Accordion title="# データはどれくらいの頻度で更新できますか？">


DataSetは1日に1回更新する設定にする必要があります。


</Accordion>
<Accordion title="# 注意すべきAPIレート制限はありますか？">


Pinterestでは、各ユーザーは1時間あたり最大1,000件のAPIを呼び出すことができます。1回のDataSet実行に複数のAPI呼び出しが必要になる場合がありますのでご注意ください。60分のウィンドウは、最初のリクエストの実行時にもとづくスライディングウィンドウです。レート制限に達した場合、さらにいくつかのリクエストを取得するには、最大1時間待つだけで済みます。


</Accordion>
<Accordion title="# いいね！したピンはどうなったのですか？">


2017年6月、Pinterestは、［いいね！］ボタンと［保存］ボタンの共存によってユーザーが混乱すると判断し、UIから［ いいね！］ボタンを削除しました。現在は、［保存］ボタンのみを利用できます。それ以前に「いいね！」されたすべてのアイテムは、アカウントの「いいね！したピン」というシークレットボードに移動されました。



</Accordion>
</AccordionGroup>
