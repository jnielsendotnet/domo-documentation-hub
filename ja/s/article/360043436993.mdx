## はじめに

Microsoft OneDriveは、ビジネス用オンラインファイル共有とパーソナルクラウドコンテンツ管理サービスです。このコネクターを使用すると、指定したOneDriveフォルダにDomo DataSetからデータをエクスポートできます。OneDrive APIの詳細については、 [https://msdn.microsoft.com/en-us/lib.../dn659752.aspx](https://msdn.microsoft.com/en-us/library/office/dn659752.aspx "https://msdn.microsoft.com/en-us/library/office/dn659752.aspx") を参照してください。

OneDriveへのデータのエクスポートはData Centerで行います。このトピックでは、Microsoft OneDrive Writebackコネクターユーザーインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加や更新スケジュールの設定、DataSet情報の編集に関する一般的な情報は、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

<Note>
**注記：** Writeback DataSetの所有者は、入力DataSetの所有者または共同所有者である必要があります。

</Note>

## 必要条件

このコネクターを設定するには、次の情報が必要です。

* Microsoft Azure Appのインストール。
* Domoデベロッパーアカウントに関連付けられたクライアントIDとクライアントシークレット
* Microsoft Azure Appに関連付けられているシークレットIDと値

#### DomoクライアントIDとクライアントシークレットを取得するには：

1. [Domoデベロッパーアカウント](https://developer.domo.com/login "https://developer.domo.com/login") にログインします。
2. 右上隅の **［My Account］** の下にある **［New Client］** をクリックします。
3. アプリケーション名と説明を入力します。
4. **［Data］** と **［User］** のチェックボックスを選択して、アプリケーションのスコープを指定します。
5. **［Create］** をクリックします。クライアントを作成したら、 **［Manage Client］** をクリックしてそのクライアントを管理できます。
6. **［Client Secret］** が **［Manage Client］** セクションに表示されます。

#### シークレット（クライアント）IDを取得するには：

1. [https://portal.azure.com](https://portal.azure.com/ "https://portal.azure.com") にアクセスします。

2. **［Azure Active Directory］>［App registrations］** を選択します。

3. Web Appを見つけ、クリックします。

4. 「application ID」が **クライアントID** です。

#### シークレットの値を取得するには：

1. [https://portal.azure.com](https://portal.azure.com/ "https://portal.azure.com") にアクセスします。

2. **［Azure Active Directory］>［App registrations］** を選択します。

3. Web Appを見つけ、クリックします。

4. **［Certificates and secrets］** を開きます。

5. **［+New client secret］** をクリックします。

6. クライアントシークレットの説明とシークレットの有効期限を入力します。**［Add］** をクリックします。

7. 新しいクライアントシークレットが表示されます。それをコピーして、Domoコネクターの［Client Secret］フィールドに貼り付けます。

### Microsoftのアクセス許可

Microsoftで正しいアクセス許可を持っていると、Microsoft OneDrive Writebackコネクターを使用して効果的にデータにアクセスできます。

#### 新しいAppのアクセス許可を設定する

これらの手順は、新しいAppにのみ関係します。

1. [https://apps.dev.microsoft.com/](https://apps.dev.microsoft.com/ "https://apps.dev.microsoft.com/") に移動してサインインします。
2. ****［Add an app］**** をクリックします。
     
   <Frame><img alt="onedrive_1.png" src="/images/kb/ka05w00000123QC-00N5w00000Ri7BU-0EM5w000005vO9h.png"/></Frame>

3. Appに名前を付けます。
4. ****［Create］**** をクリックします。
     
   <Frame><img alt="onedrive_2.png" src="/images/kb/ka05w00000123QC-00N5w00000Ri7BU-0EM5w000005vO9i.png"/></Frame>

   表示されるApp IDがクライアントIDです。
     
   <Frame><img alt="Application_ID.png" src="/images/kb/ka05w00000123QC-00N5w00000Ri7BU-0EM5w000005vO9b.png"/></Frame>

5. ****［Generate new password］**** をクリックします。
     
   今後再び表示されることはないため、この情報は保存しておいてください。これがシークレットです。
     
   <Frame><img alt="Password.png" src="/images/kb/ka05w00000123QC-00N5w00000Ri7BU-0EM5w000005vO9c.png"/></Frame>

6. ****［OK］**** をクリックします。
7. ****［Add platform］**** をクリックします。
8. ****［Web］**** を選択します。
9. ****［Redirect URLs］**** フィールドに、URL **https://oauth.domo.com/api/data/v1/oauth/providers/domo-to-onedrive/exchange** を入力します。
     
   <Frame><img alt="ms_onedrive_writeback_1.png" src="/images/kb/ka05w00000123QC-00N5w00000Ri7BU-0EM5w000005vO9e.png"/></Frame>

10. ****［Save］**** をクリックします。
11. Files.Read.All、offline\_access、およびSites.ReadWrite.Allの権限を追加します。
      
    <Frame><img alt="ms_onedrive_writeback_2.png" src="/images/kb/ka05w00000123QC-00N5w00000Ri7BU-0EM5w000005vO9d.png"/></Frame>

12. ****［Save］**** をクリックします。
13. **［Manage］** の下で **［Certificates & Secrets］** を選択します。証明書は、トークンを要求するときにアプリケーションの識別情報を証明するためのシークレットとして使用できます。これは、パブリックキーと呼ばれることもあります。
14. **［+New client secret］** ボタンを選択します。
15. **［Description］** に値を入力し、［Expires］に適したオプションを選択し、**［Add］** をクリックします。

    <Warning>
    **重要：** ページを終了する前に、クライアントシークレット値（シークレットIDではありません）をコピーしてください。このページから離れると、再び表示されません。

    </Warning>

    <Frame><img alt="Secret_Value_selection.png" src="/images/kb/ka05w00000123QC-00N5w00000Ri7BU-0EM5w000005vO9g.png"/></Frame>

16. Domoコネクターに移動してクライアントIDとシークレットを使って接続します。

## 接続を設定する

このセクションでは、Microsoft OneDrive Writebackコネクターページの **［認証情報］** および **［詳細］** ペインのオプションについて説明します。このページのほかのペインの項目である **［スケジュール設定］** や **［DataSetに名前を付け、説明を入力］** に関しては、ほとんどのタイプのコネクターで共通しています。詳細については、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。
