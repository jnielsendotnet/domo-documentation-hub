## はじめに

Xeroは、中小企業向けのクラウドベースの会計ソフトウェアを開発しています。このコネクターを使用すると、銀行口座、取引、経費請求、連絡先などに関するデータを取得できます。DomoとXeroの間で接続を確立するには、ユーザーがまずXeroで認証し、それからDomoにデータへのアクセスを許可する必要があります。Xero APIの詳細については、Xeroのページ（ [http://developer.xero.com/documentat.../overview](https://developer.xero.com/documentation/api/accounting/overview "https://developer.xero.com/documentation/api/accounting/overview") ）を参照してください。

Xeroアカウントには、Data Centerから接続します。このトピックでは、Xeroコネクターユーザーインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加や更新スケジュールの設定、DataSet情報の編集に関する一般的な情報は、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

## 必要条件

XeroアカウントにアクセスしてDataSetを作成するには、Xeroアカウントへのログイン時に使用するメールアドレスとパスワードが必要です。

## Xeroアカウントに接続する

このセクションでは、Xeroコネクターページの **［認証情報］** および **［詳細］** ペインのオプションについて説明します。このページのほかのペインの項目である **［スケジュール設定］** や **［DataSetに名前を付け、説明を入力］** に関しては、ほとんどのタイプのコネクターで共通しています。詳細については、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

### ［認証情報］ペイン

DomoのXeroコネクターは、接続にOAuthを使用します。そのため、Domo内での認証情報の入力は必要ありません。 **［接続］** をクリック（または、Domoに既存のXeroアカウントがある場合は **［アカウントを追加］** を選択）すると、Xero OAuth画面が開き、Xeroの認証情報を入力できます。有効なXero認証情報を一度入力しておくと、随時同じアカウントを使用して新しいXero DataSetを作成することができます。コネクターのアカウントは Data Center の **［アカウント］** タブで管理できます。このタブの詳細については、「 [コネクター/アダプターアカウントを管理する](/ja/s/article/360042926054 "コネクター/アダプターアカウントを管理する") 」を参照してください。

<Warning>
**注記：** 指定した認証情報で一度に認証できるXeroアカウントは1つだけです。別の場所でXeroでOAuthを使用しようとすると、DomoのXeroアカウントが無効になります。

</Warning>

---

<Note>
**注記：** Domoで接続する際、既にXeroにログインしている場合は、 **［アカウントを追加］** をクリックすると自動的に認証されます。ログインしているアカウントとは異なるアカウントに接続したい場合は、まずXeroからログアウトする必要があります。

</Note>

## コンサルティング/サポートのトラブルシューティング

### 列名の日付

Xeroレポートの中には、実行日が列名として使われているデータを返すものがあります。例えば、2017年2月17日に「損益」レポートを実行すると、「Row\_\_17\_Feb\_17」という金額列が表示されます。これにより、スケジュールされたDataSetの更新が行われるたびに列の名前が変更されるため、これらの列はDataFlowやカードで使用されなくなります。

この問題を解決するには、列名の日付を一般的な列名に置き換えることを推奨します。Domo SQLクエリを使用すると、不具合のある列を一般的な名前の列名に置き換えることができます。その後、この列はMagic DataFlowで参照できるようになります。

次の例では、列名「Row\_\_16\_Feb\_17」を「ProfitLossAmount」という名前に置き換えます。
  
<Frame><img alt="" src="/images/kb/ka0Vq000000068f-00N5w00000Ri7BU-0EM5w000005vO1n.png" style={{width: 941, height: 81}}/></Frame>

```sql 
select NULL AS Row_Id, NULL AS Row_ProviderName, NULL AS Row_DateTimeUTC, NULL AS Row_ReportName, NULL AS Row_ReportDate, NULL ​AS Row_UpdatedDateUTC, NULL AS Row_ReportTitle, NULL AS Row_column_0, NULL AS ProfitLossAmount, NULL AS _BATCH_ID_, NULL AS _BATCH_LAST_RUN_ from xero_profit_and_loss_report
  
where 1 = 0 --this row returns nothing but allows setting the column names for the query`

union all

select * from xero_profit_and_loss_report
```

### 不要なピボット表

多くの場合、上記の列の命名の問題があるデータはピボット表として表されます。スケジュールされた実行が行われるたびに列名が変更されるため、ETLで［列を折りたたむ］アクションを使用してこのデータをアンピボットすることはできません。例えば、「Budget」レポートがこれに該当します。

この問題を解決するには、日付で名前が付けられた列をアンピボットしますが、出力DataSetが1つの列につき1つの日付行を持つように、日付値は行として保持します。

次の例では、まずDomo SQLクエリを使用して列の名前を一般的な名前に変更します。次にMagic ETL DataFlowを使用して **［列を折りたたむ］** アクションを実行します。次に、一般的な列名（現在は行データ）を日付に変換します。

1. 次のようにDomo SQLクエリを作成します。

   Transform: Replace the column names "Row\_\_Feb-17," "Row\_\_Mar-17," etc. with "Month1," "Month2," etc. using the method described above.
