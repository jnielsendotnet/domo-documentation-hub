## はじめに

Azure Data Lake Storeは、ビッグデータ解析ワークロード用の全社規模のハイパースケールリポジトリです。このコネクターを使うと、Data Lake Storeからデータファイルを取得し、Domoで利用できます。データファイルは、CSV、XLS、XLSX、JSONのいずれかのファイルタイプで取得できます。AzureのAPIの詳細については、 [https://learn.microsoft.com/ja-jp/pre...e-store-overview](https://learn.microsoft.com/ja-jp/previous-versions/azure/data-lake-store/data-lake-store-overview "https://learn.microsoft.com/ja-jp/previous-versions/azure/data-lake-store/data-lake-store-overview") を参照してください。

Microsoft Azure Storage Data Lake Gen2アカウントには、Data Centerから接続します。このトピックでは、Microsoft Azure Storage Data Lake Gen2コネクターのユーザーインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加、更新スケジュールの設定およびDataSet情報の編集を行うには、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

## 必要条件

Microsoft Azure Data Lake Store Gen2アカウントに接続してDataSetを作成するには、以下が必要です。

* アカウントキー
* アカウント名

認証情報を取得する場合は、Microsoftのアカウント担当者にお問い合わせください。

このセクションでは、 Microsoft Azure Storage Data Lake Gen2 コネクターページの **［認証情報］** および **［詳細］** ペインのオプションについて説明します。このページのほかのペインの項目である **［スケジュール設定］** や **［DataSetに名前を付け、説明を入力］** に関しては、ほとんどのタイプのコネクターで共通しています。詳細については、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

## よくある質問

<AccordionGroup>
<Accordion title="アカウント名とアカウントキーはどこで確認できますか？">


アカウント名とアカウントキーを取得するには、ストレージアカウントを作成する必要があります。

アカウント名を取得するには、以下の手順に従います。
  
1. [Microsoft Azure Portal](https://portal.azure.com/ "Microsoft Azure Portal") にアクセスし、Microsoft認証情報を使用してサインインします。
  
2. **［Storage accounts］** を選択します。
  
3. ［Storage accounts］を見つけてクリックします。
  
4. タイトルの右側にあるメニューアイコンをクリックし、［Copy title to clipboard］を選択します。

アカウントキーを取得するには、以下の手順に従います。
  
1. [Microsoft Azure Portal](https://portal.azure.com/ "Microsoft Azure Portal") にアクセスし、Microsoft認証情報を使用してサインインします。
  
2. **［Storage accounts］** を選択します。
  
3. ［Storage accounts］を見つけてクリックします。
  
4. **［Security - networking］>［Access Keys］** を開きます。
  
5. key1またはkey2の［Show］ボタンをクリックし、値をコピーします。


</Accordion>
<Accordion title="アカウントを作成できたにもかかわらず、ファイルやディレクトリの情報を取得できないのはなぜですか？">

  
Azure Data Lake Storage Gen2では、ストレージアカウントの階層型名前空間を有効にする必要があります。これが無効の場合、コネクターはディレクトリ情報を取得できず、ファイルを取得できません。
  
<Frame><img alt="image-2025-01-24-18-09-45-061 (1).png" src="/images/kb/ka0Vq0000006x0b-00N5w00000Ri7BU-0EMVq0000051Bub.jpg" style={{width: 794, height: 137}}/></Frame>

有効にするには、［Hierarchical namespace］セクションの［Disabled］をクリックし、ガイダンスに従ってストレージアカウントをAzure Data Lake Storage Gen2にアップグレードします。



</Accordion>
