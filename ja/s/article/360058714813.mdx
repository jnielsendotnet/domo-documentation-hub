## はじめに

Microsoft Reportsコネクターは、組織内のMicrosoftメールデータへの簡単なアクセス方法を提供します。Microsoftのメールデータに関する詳細を簡単かつ安全にDomoに取得することができます。そのためには、まずMicrosoft AzureでAppを作成する必要があります。Microsoftアカウントに接続するには、管理者アカウントか管理者の承認が必要です。DomoのMicrosoft Reportsコネクターを使用すると、Microsoftアカウントからメールとユーザーアクティビティに関するデータをDomoの中に取得することができます。

Microsoft Reportsアカウントには、Data Centerから接続します。このトピックでは、Microsoft Reportsコネクターユーザインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加や更新スケジュールの設定、DataSet情報の編集に関する一般的な情報については、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

## 必要条件

Microsoftアカウントに接続してDataSetを作成するには、以下が必要です。

* Microsoft Azure Appのインストール。
* Azure AppのクライアントID。
* Azure Appのクライアントシークレット。これは、Azure Appを作成するときに生成されます。

##### クライアントIDを見つけるには、次の手順を実行します。

1. [https://portal.azure.com](https://portal.azure.com "https://portal.azure.com") にアクセスし、Microsoft認証情報を使用してサインインします。
2. ****［Azure Active Directory］ > ［アプリの登録］**** を選択します。
3. お使いのAppを探し、クリックします。
4. Web Appの「アプリケーションID」を確認します（これがクライアントIDです）。

##### 新しいクライアントシークレットを生成するには、次の手順を実行します。

1. [https://portal.azure.com](https://portal.azure.com "https://portal.azure.com")
   にアクセスし、Microsoft認証情報を使用してサインインします。
2. ****［Azure Active Directory］ > ［アプリの登録］**** を選択します。
3. お使いのAppを探し、クリックします。
4. ****［証明書とシークレット］**** を開きます。
5. クライアントシークレットを作成します。

### Enterprise OAuth用Azure Appを作成する

Microsoft Azure Appを作成するには、次の手順を実行してください。

1. [https://portal.azure.com](https://portal.azure.com/ "https://portal.azure.com") にログインします。

2. ****［アプリの登録］**** をクリックします。

<Frame><img alt="Capture1.PNG" src="/images/kb/ka05w000001236O-00N5w00000Ri7BU-0EM5w000005vNz7.png"/></Frame>

3. ****［新規登録］**** をクリックします。

4. アプリケーション名を入力し、下記でサポートされているアカウントタイプを選択します。WebのリダイレクトURI
https://oauth.domo.com/api/data/v1/oauth/providers/microsoft-reports/exchange
を入力します。

<Frame><img alt="Capture3.PNG" src="/images/kb/ka05w00000123LH-00N5w00000Ri7BU-0EM5w000005vO5p.png"/></Frame>

5. ****［登録］**** をクリックします。

6. アプリケーションが登録されたら **［証明書とシークレット］** をクリックします。証明書は、トークンを要求するときにアプリケーションの識別情報を証明するためのシークレットとして使用できます。また、パブリックキーと呼ばれることもあります。

<Frame><img alt="Capture4.PNG" src="/images/kb/ka05w000001236O-00N5w00000Ri7BU-0EM5w000005vNyu.png"/></Frame>

7. **［+新規クライアントシークレット］** ボタンをクリックします。

<Frame><img alt="Capture5.PNG" src="/images/kb/ka05w000001236O-00N5w00000Ri7BU-0EM5w000005vNz3.png"/></Frame>

8. クライアントシークレットの説明と有効期限を指定し、****［追加］**** をクリックします。

<Frame><img alt="Capture6.PNG" src="/images/kb/ka05w00000123LH-00N5w00000Ri7BU-0EM5w000005vO62.png"/></Frame>

9. 値をコピーして、Domoのコネクター認証情報セクションの ****［Client Secret］**** フィールドに貼り付けます。

10. ここで、**［APIのアクセス許可］** をクリックします。

<Frame><img alt="Capture7.PNG" src="/images/kb/ka05w00000123LH-00N5w00000Ri7BU-0EM5w000005vO5x.png"/></Frame>

11. **［+権限を追加する］** ボタンをクリックします。

<Frame><img alt="Capture8.PNG" src="/images/kb/ka05w00000123LH-00N5w00000Ri7BU-0EM5w000005vO61.png"/></Frame>

12. 許可を追加した後、設定された権限は以下のように表示されます。

## Microsoftアカウントに接続する

このセクションでは、 Microsoft Reports コネクターページの **［認証情報］** および **［詳細］** ペインのオプションについて説明します。このページのほかのペインの項目にある **［スケジュール設定］** や **［DataSetに名前を付け、説明を入力］** に関しては、コネクタータイプのほとんどで共通です。詳細については「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。
