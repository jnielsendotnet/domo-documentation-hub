## はじめに

Azure Data Lake Storage Gen2は世界最高水準の生産性を持つデータレイクです。ビッグデータ解析ワークロード用の全社規模のハイパースケールリポジトリとなっています。Azure Storageが、Azure上にエンタープライズデータレイクを構築するための基礎になります。大量のデータを簡単に管理することによって、PoC（導入検証）から生産への移行を迅速に行うことができます。DomoのMicrosoft Azure Data Lake Store Gen2 using AADコネクターを使うと、Data Lake Storeからデータファイルを取得し、Domoで利用できます。データファイルは、CSV、TSV、TXTのいずれかのファイルタイプで取得できます。Azure APIについて詳しくは、製品ドキュメント（ [https://docs.microsoft.com/en-us/azu...ta-lake-store/](https://docs.microsoft.com/en-us/azure/data-lake-store/ "https://docs.microsoft.com/en-us/azure/data-lake-store/") ）を参照してください。

Microsoft Azure Data Lake Store Gen2 AADアカウントには、Data Centerから接続します。このトピックでは、Microsoft Azure Data Lake Store Gen2 using AADコネクターのユーザーインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加や更新スケジュールの設定、DataSet情報の編集に関する一般的な情報は、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274) 」を参照してください。

## 必要条件

Microsoft Azure Data Lake Store Gen2 AADアカウントに接続してDataSetを作成するには、以下が必要です。

* Azure Active Directory設定から取得したクライアントWeb AppのクライアントID（GUID）とシークレットキー
* Microsoft Azure Data Lake Storageのアカウント名
* テナントID

### Azure Appを設定する

必要なクライアントIDとシークレットを取得するには、最初にMicrosoft Azure Web Appを作成する必要があります。

**Azure Appを設定するには** ：

1. [Azureポータル](https://portal.azure.com/#home "https://portal.azure.com/#home") にログインします。
2. **［Azure Active Directory］>［App Registrations］>［New Registration］** の順に選択します。

3. 登録フォームに次の通りに入力します。

* ［Name］：アプリケーションの名前を入力します。
* ［Supported account types］：オプションを選択して、このアプリケーションを使用できる、またはこのAPIにアクセスできるユーザーを指定します。
* ［Redirect URI］：リダイレクトURIを入力します。認証応答は、ユーザーの認証に成功した後にこのURIに送信されます。この段階でURIを指定することは任意で、後で変更することもできますが、ほとんどの認証シナリオでは値が必要です。

4. **［Register］** をクリックします。登録したAppの概要が表示されます。

5. ここに表示される **［Application ID］** が **クライアントID** であり、**［Directory ID］** はDomoの **［認証情報］** ペインに入力する **テナントID** になります。

6. 次に、**［Certificates and secrets］** をクリックします。証明書は、トークンを要求するときにアプリケーションの識別情報を証明するためのシークレットとして使用できます。これは、パブリックキーと呼ばれることもあります。

7. **［+New client secret］** をクリックします。

8. クライアントシークレットの説明と有効期限を指定し、**［Add］** をクリックします。

9. 値をコピーして、Domoのこの **コネクターの［認証情報］** セクションの **［Client Secret］** フィールドに貼り付けます。

<Accordion title="このコネクターを作成するにはどのような認証情報が必要ですか？">


Azure Active Directory設定で取得したクライアントWeb AppのクライアントID、クライアントシークレット、テナントIDに加えて、ストレージアカウント名が必要です。


</Accordion>
<Accordion title="DomoクライアントID、クライアントシークレット、テナントIDを見つけるにはどうしたらよいですか？">


クライアントIDを取得するには：

* Azure Portal（ https://protal.azure.com ）にログインします。
* ［Azure Active Directory］>［App registrations］を選択します。
* ［Web App］>［Application ID］をクリックします。
* アプリケーションIDがクライアントIDです。

シークレットキーを取得するには：

* Azure Portal（ https://protal.azure.com ）にログインします。
* ［Azure Active Directory］>［App registrations］を選択します。
* 証明書とシークレットをクリックしてから、［+New client secret］を選択します。
* シークレットの値をコピーします。シークレットは一度だけ生成されることに注意してください。

テナントIDを取得するには：

* Azure Portal（ https://protal.azure.com ）にログインします。
* ［Azure Active Directory］>［Properties］を選択します。
* ディレクトリIDをコピーします。ディレクトリIDがテナントIDです。


</Accordion>

ありません。

## トラブルシューティング

* 認証が有効であることを確認してください。
* 設定を見直し、すべての必須項目が選択されていることを確認してください。
* コネクターの履歴にエラーメッセージが記録されていないか確認してください。
* まれに、リクエストした情報が多すぎた結果、API制限またはタイムアウトに達する場合があります。記録されている場合は、コネクターの履歴でエラーメッセージと実行時間を確認することができます。また、取得するアカウント数やレポートのメトリクス数を減らしたり、取得するタイムフレームを短縮したりすることができます。
