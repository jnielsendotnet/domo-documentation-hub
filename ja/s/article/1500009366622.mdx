## はじめに

Cornerstone OnDemandは、組織が社員を採用、トレーニング、エンゲージ、そして管理する手助けをします。また、Cornerstoneは適切な開発とエクスペリエンスにより、社員が職場で能力を最大限に発揮する手助けもします。Cornerstone OnDemandコネクターを使うと、データをCornerston OnDemandシステムからDomoに引き込むことができます。Cornerstone OnDemand APIの詳細については、Cornerstone OnDemandのWebサイトをご覧ください（[https://apiexplorer.csod.com/apiconn.../apiexplorer#/](https://apiexplorer.csod.com/apiconnectorweb/apiexplorer#/)）。

Cornerstone OnDemandアカウントには、Data Centerから接続します。このトピックでは、Cornerstone OnDemandコネクターのユーザーインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加、更新スケジュールの設定およびDataSet情報の編集に関する一般的な情報は、「[データConnectorを使用してDataSet を追加する](/s/article/360042926274)」に記載があります。

## 必要条件

Cornerstone OnDemandアカウントに接続してDataSetを作成するには、以下が必要です。

- ポータルのアクティブなアカウントに関連付けられている有効なユーザ名
- Cornerstone OnDemandアプリケーションにクライアントが割り当てたホスト名（Corpname）
- APIキーとAPIシークレット。統合センターの［APIを管理］タブで取得できます。

## Cornerstone OnDemandアカウントに接続する

このセクションでは、Cornerstone OnDemandコネクターページの［認証情報］および［詳細］ペインのオプションについて説明します。このページのほかのペインの項目である**［スケジュール設定］**や **［DataSetの名前と説明］**に関しては、ほとんどのタイプのコネクターで共通しています。詳細については「[データコネクターを使用してDataSetを追加する](/s/article/360042926274)」を参照してください。

### ［認証情報］ペイン

このペインには、Cornerstone OnDemandアカウントに接続する認証情報の入力フィールドが含まれています。以下の表は、各フィールドに必要な内容を説明しています。

| フィールド       | 説明                                                             |
| ----------- | -------------------------------------------------------------- |
| Username    | ポータルのアクティブなアカウントに関連付けられている有効なユーザー名を入力します。                      |
| Coprname    | Cornerstone OnDemandアプリケーションにクライアントが割り当てたホスト名（Corpname）を入力します。 |
| API Key     | APIキーを入力します。統合センターの［APIを管理］タブで取得できます。                          |
| API Secret  | APIシークレットを入力します。統合センターの［APIを管理］タブで取得できます。                      |
| Environment | 環境（Pilot、Stage、またはProduction）を選択します。                           |

有効なCornerstone OnDemand認証情報を一度入力しておくと、随時同じアカウントを使用して新しいCornerstone OnDemand DataSetを作成できます。コネクターアカウントは、Data Centerの**［アカウント］**タブで管理できます。このタブの詳細については「[コネクターのユーザーアカウントを管理する](/s/article/360042926054)」を参照してください。

### ［詳細］ペイン

このペインには、メインとなる **[レポート]** メニューの他、選択したレポートタイプに応じ表示・非表示となる様々なメニューが準備されています。

| メニュー  | 説明                                            |
| ----- | --------------------------------------------- |
| Views | ドロップダウンからビューを選択します。アクセスできるビューは、アクセスレベルに依存します。 |

### その他のペイン

スケジュール設定、再試行、オプションの更新など、コネクターインターフェースのその他のセクションについては、「[データコネクターを使用してDataSetを追加する](/s/article/360042926274)」を参照してください。

## よくある質問

<AccordionGroup>

<Accordion title="このコネクターのベースURLはどのエンドポイントですか？">
Cornerstone OnDemandのベースURLは、環境の選択によって異なります。

| 環境         | ベースURL                                                     |
| ---------- | ---------------------------------------------------------- |
| Pilot      | https://{corpname}-pilot.csod.com/services/api/x/odata/api |
| Stage      | https://{corpname}-stg.csod.com/services/api/x/odata/api   |
| Production | https://{corpname}.csod.com/services/api/x/odata/api       |
</Accordion>

<Accordion title="このコネクターを作成するにはどのような認証情報が必要ですか？">
Cornerstone OnDemandアカウントに関連づけられている、ユーザー名、Corpname、APIキー、およびAPIシークレットが必要です。APIキーとAPIシークレットは、統合センターの［APIを管理］タブで取得できます。環境（Pilot、Stage、またはProduction）も選択する必要があります。
</Accordion>

<Accordion title="データはどれくらいの頻度で更新できますか？">
必要に応じて更新できます。
</Accordion>

<Accordion title="注意すべきAPI制限はありますか？">


なし

## トラブルシューティング

* 認証が有効であることを確認してください。
* 設定を見直し、すべての必須項目が選択されていることを確認してください。
* コネクターの履歴にエラーメッセージが記録されていないかどうかを確認してください。
* まれに、多過ぎる情報をリクエストした結果、API制限またはタイムアウトに達する場合があります。記録されている場合は、コネクターの実行履歴でエラーメッセージと実行時間を確認することができます。また、取得するアカウント数やレポートのメトリクス数を減らしたり、取得するタイムフレームを短縮したりすることができます。
