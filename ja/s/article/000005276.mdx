## はじめに

Xeroは、キャッシュフロー、請求書、経費などを監視するツールを使用して、中小企業が財務状態をしっかり把握できるようにします。XeroをDomoに接続すると、リアルタイムデータをコラボレーションできるプラットフォームが得られ、データをシンプルに把握することができます。Domo Xeroコネクターを使用すると、経費をDomoに取り込むことができ、財務状況を簡単に追跡して、正確に資金の流れを確認できます。Xero APIの詳細については、Xeroのページ（ [http://developer.xero.com/documentat.../overview](https://developer.xero.com/documentation/api/accounting/overview "https://developer.xero.com/documentation/api/accounting/overview") ）を参照してください。

データをDomoに追加したら、Xeroの財務データを、好きなように組み合わせ、整理、可視化することができます。財務メトリクスが定義したしきい値に達したときに、即時通知が行われます。

Xero Customアカウントには、Data Centerから接続します。このトピックでは、Xeroコネクターユーザーインターフェースに固有のフィールドとメニューについて説明します。DataSetの追加や更新スケジュールの設定、DataSet情報の編集に関する一般的な情報は、「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

## 必要条件

Xero CustomアカウントにアクセスしてDataSetを作成するには、Xeroアカウントへのログイン時に使用するメールアドレスとパスワードが必要です。

* クライアントID
* クライアントのシークレット
* スコープ

## Xero Customアカウントに接続する

このセクションでは、Xero Customコネクターページの **［認証情報］** および **［詳細］** ペインのオプションについて説明します。このページのほかのペインの項目である **［スケジュール設定］** や **［DataSetに名前を付け、説明を入力］** に関しては、ほとんどのタイプのコネクターで共通しています。詳細については「 [コネクターを使用してDataSetを追加する](/ja/s/article/360042926274 "コネクターを使用してDataSetを追加する") 」を参照してください。

### ［認証情報］ペイン

DomoのXero Customコネクターは、接続にOAuthを使用します。そのため、Domo内での認証情報の入力は必要ありません。 **［接続］** をクリック（または、Domoに既存のXeroアカウントがある場合は **［アカウントを追加］** を選択）すると、Xero OAuth画面が開き、Xero Customの認証情報を入力できます。

このペインには、Xero CustomアカウントとDomoデベロッパーアカウントに接続するための認証情報の入力フィールドがあります。下表では、各フィールドに必要な内容を説明しています。

| メニュー          | 説明                                                                                                                                            |
| ------------- | --------------------------------------------------------------------------------------------------------------------------------------------- |
| Client ID     | Xero Custom接続クライアントIDを入力します。                                                                                                                  |
| Client Secret | Xero Custom接続クライアントシークレットを入力します。                                                                                                              |
| Scope         | Xero Custom接続のセットアップ時に指定されたスコープのスペース区切りリストを入力します。利用できる [スコープ](https://developer.xero.com/documentation/guides/oauth2/scopes/) の完全なリストが表示されます。 |

有効なXero Custom認証情報を一度入力しておくと、随時同じアカウントを使用して新しいXero DataSetを作成することができます。コネクターのアカウントは Data Center の **［アカウント］** タブで管理できます。このタブの詳細については、「 [コネクター/アダプターアカウントを管理する](/ja/s/article/360042926054 "コネクター/アダプターアカウントを管理する") 」を参照してください。

<Warning>
**注記：** 指定した認証情報で一度に認証できるXero Customアカウントは1つだけです。別の場所でXeroでOAuthを使用しようとすると、DomoのXero Customアカウントが無効になります。

</Warning>

## コンサルティング/サポートのトラブルシューティング

### 列名の日付

Xeroレポートの中には、実行日が列名として使われているデータを返すものがあります。例えば、2017年2月17日に「損益」レポートを実行すると、「Row\_\_17\_Feb\_17」という金額列が表示されます。これにより、スケジュールされたDataSetの更新が行われるたびに列の名前が変更されるため、これらの列はDataFlowやカードで使用されなくなります。

この問題を解決するには、列名の日付を一般的な列名に置き換えることを推奨します。Domo SQLクエリを使用すると、不具合のある列を一般的な名前の列名に置き換えることができます。その後、この列はMagic DataFlowで参照できるようになります。

次の例では、列名「Row\_\_16\_Feb\_17」を「ProfitLossAmount」という名前に置き換えます。

<Frame><img alt="" src="/images/kb/ka0Vq000000068f-00N5w00000Ri7BU-0EM5w000005vO1n.png" style={{width: 941, height: 81}}/></Frame>

```sql
select NULL AS Row_Id, NULL AS Row_ProviderName, NULL AS Row_DateTimeUTC, NULL AS Row_ReportName, NULL AS Row_ReportDate, NULL ​AS Row_UpdatedDateUTC, NULL AS Row_ReportTitle, NULL AS Row_column_0, NULL AS ProfitLossAmount, NULL AS _BATCH_ID_, NULL AS _BATCH_LAST_RUN_ from xero_profit_and_loss_report

where 1 = 0 --this row returns nothing but allows setting the column names for the query`

union all

select * from xero_profit_and_loss_report
```

### 不要なピボット表

多くの場合、上記の列の命名の問題があるデータはピボット表として表されます。スケジュールされた実行が行われるたびに列名が変更されるため、ETLで［列を折りたたむ］アクションを使用してこのデータをアンピボットすることはできません。例えば、「Budget」レポートがこれに該当します。

この問題を解決するには、日付で名前が付けられた列をアンピボットしますが、出力DataSetが1つの列につき1つの日付行を持つように、日付値は行として保持します。
