### Required Grant

To create, edit, update, and delete reports in Report Builder, you must have the following grant enabled for your role:

* **Edit Report (Report Builder) —** Report Builder only appears as an option in App Studio if you have this grant, which is enabled for Admins and editors by default.

To learn more about grants, see [Manage User Roles and Grants](/s/article/360043438973).

Follow the steps below to access Report Builder in App Studio.

Follow these steps to create a report from your app:

1. In the **Create New Email Report** modal, select **Choose Content**.
     
   <Frame><img alt="" src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq000006damY.jpg" style={{width: 501, height: 350}}/></Frame>
2. Under **Choose app pages**, select the specific pages you want to include in your report, then select **Next.** **Note:** Buttons, tabs, and controls on app pages are excluded from the report.

   The Report Builder interface opens and displays the pages you selected.

   [below](#configure_your_report).

# Edit Report Name, Description, and Tags

Under **About** in the left panel of the Report Builder, edit the report name and description and add report tags.

<Frame><img alt="" src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq0000078wsP.jpg" style={{width: 900, height: 455}}/></Frame>
* By default, the report automatically inherits the name of your app.
* Report tags help you organize and categorize reports. Tags cannot include apostrophes, underscores, backslashes, percent signs, or emojis.

<Frame><img alt="" src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq000006da9q.jpg" style={{width: 265, height: 300}}/></Frame>
# Design Report Appearance

Under **Report Appearance** in the left panel of the Report Builder, you can choose a report background color and the amount of density (the space around objects in the report).

<Frame><img alt="" src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq000006dVgQ.jpg" style={{width: 260, height: 231}}/></Frame>
Selecting **Import Theme From App** opens the App Studio themes modal. Learn more about [App Studio themes](/s/article/000005543).

<Note>
  **Note:** Supported theme components include chart color palette, layout density (minor rendering differences), background colors, borders, and rounded corners (minor rendering differences). Spacing and density may vary.
</Note>

<Frame><img alt="" src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq000006oo9S.jpg" style={{width: 568, height: 504}}/></Frame>
<Warning>
  **Important:** If changes are made to the *theme* used in the report after the report is created, those changes are NOT automatically synced with the report. The report continues to use the theme as it was when the report was created. To update your report theme, use the **Import Theme From App** option to reassign the theme or choose another.
</Warning>

# Edit Report Configuration

Use the **Report Configuration** section in the left panel of the Report Builder to make changes to the report header and footer, including whether to display the Domo logo and text and background colors.

When you're finished, save your changes to add the new/updated report to your list of reports.

### Create Report Views

To help your report viewers see the content most appropriate for them, you can choose filters and variables for each report in the <InlineImage src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq0000073oPF.jpg" /> **Report Views** section of Report Builder. There is no limit on the number of report views you can create.

<Note>
  **Note:** Changes to the report configuration or other report settings apply to all report views and their schedules.
</Note>

<Frame><img alt="" src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq000006rWvx.jpg" style={{width: 905, height: 438}}/></Frame>
To create a new report view, select **+ New Report View**.

<Frame><img alt="" src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq0000073je6.jpg" style={{width: 301, height: 196}}/></Frame>
In the Report View editor, you can [add filters](#add_filters), [schedule delivery](#schedule_your_report) for the report view, and [choose recipients](#add_report_recipients) for the report view. These tasks are described below.

# Add Filters

Under **<InlineImage src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq0000073oPF-1.jpg" /> Report Views** > **Filters** (tab), add filters to your report using the data in your app report and date range filters.

<Frame><img alt="" src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq000006rX2P.jpg" style={{width: 245, height: 283}}/></Frame>
<Note>
  **Note:** Filters are not a substitute for PDP (Personalized Data Permissions). Filters help customize views, but do not enforce data security. Use PDP as you would for other scheduled reports.
</Note>

# Schedule Your Report

After configuring your report and adding any filters to a particular report view, you can schedule your report view and begin distributing it to others. Under **<InlineImage src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq0000073oPF-2.jpg" /> Report Views** > **Schedule** (tab), you can choose when that report view should be emailed to recipients.

Use the fields to give your report email a subject, choose the frequency for report delivery (daily, weekly, or monthly), and a start and end date.

<Frame><img alt="" src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq000006rXLl.jpg" style={{width: 236, height: 473}}/></Frame>
<Warning>
  **Enable schedule:** You must enable your configured report schedule by toggling the **Enabled** button on. If you do not enable the report, it will not be sent as scheduled.
</Warning>

# Add Report Recipients

Under **<InlineImage src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq0000073oPF-3.jpg" /> Report Views** > **Recipients** (tab), you can choose who should receive it. Recipients receive the report by email.

You can also choose who owns the report view.

<Frame><img alt="" src="/images/kb/ka0Vq000000CPKP-00N5w00000Ri7BU-0EMVq000006rtVy.jpg" style={{width: 232, height: 492}}/></Frame>
### FAQ

<AccordionGroup>
<Accordion title="Can I include content from other apps in my reports?">


Yes, you can add cards and content that you own or have access to, even if they are located outside of the current app you are editing. [Add a card element](#add_actions_elements) to your report and choose a card from another app.

<Note>
  **Note:** All reports are tied to the app where they are created and appear under the "Your email reports" tab in that app.
</Note>


</Accordion>
<Accordion title="Can I use app buttons in reports? Can I add new ones?">


App buttons do not carry over into Report Builder. However, you can add a custom button in the report header that links to outside content, including the original app, dashboards, and cards. Learn how to configure this button [above](#edit_report_configuration).


</Accordion>
<Accordion title="How do tabs appear in reports?">


Tabs from your app are stacked vertically in the report.


</Accordion>
<Accordion title="Are charts and tables clickable in reports?">


No, charts and tables are static in reports. However, you can add a button that links users to interactive content. Learn how to configure this button [above](#edit_report_configuration).


</Accordion>
<Accordion title="Are tables supported?">


Yes, reports display table snapshots similar to the existing scheduled reports functionality. Learn more about scheduled reports.


</Accordion>
<Accordion title="How does dark mode for email clients affect reports?">


Dark mode may alter background and header colors in email, but it won't affect your charts or data visuals.

</Accordion>
</AccordionGroup>
