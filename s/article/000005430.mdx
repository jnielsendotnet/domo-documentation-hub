### Domo. AI

<Note>
  **A note about security considerations —** Domo recognizes that AI needs to be well-governed per our customers' internal guidelines. AI Assistants send only Domo metadata about Domo data sources to OpenAI.

  As such, to use AI Assistants in Domo, Admins must enable OpenAI models for use under Admin Settings for the AI Service Layer. Furthermore, non-admin users in Domo require a role with the Use AI Services grant enabled to leverage AI Assistants.

  In the coming months, Domo will introduce an internal LLM option for running AI Assistants that can be leveraged without sending information to OpenAI.

</Note>

# Beast Mode AI Assistant

The Beast Mode AI Assistant from Domo. AI can help generate Beast Mode calculations inside the Beast Mode editor. To use the Beast Mode AI Assistant, you must have the AI Services grant enabled for your role.

Here are some example prompts to try:

* "Return the difference between the Order Date and the Shipped Date in months"
* "Create a calculated field where if the order priority is Critical or High it becomes Important, otherwise it's Not Important"
* "Add one week to the Order Data column"
* "Calculate the running sum of total sales by order date"

See [Beast Mode AI Assistant](/s/article/000005304) to learn more.

To learn more about grants, see [Managing Custom Roles](/s/article/360043438973).

Learn more about [Domo. AI](https://ai.domo.com/) or explore [Domo. AI documentation](https://domo-support.domo.com/s/topic/0TO5w000000ZmGPGA0/domoai?language=en_US) in the Knowledge Base.

# DataSet Views Advanced SQL Editor with AI Assistant

The Advanced SQL Editor in the Views Explorer allows you to see the SQL that is powering any DataSet View. You can edit the SQL, apply it, and watch your query update.

In addition, you can use the AI Assistant to create a SQL query for the data you want to include in your DataSet View. Write a prompt in natural language and the AI Assistant returns a SQL query.

Here are some example prompts to try:

* "Show me all rows where Order Priority is High or Critical and TCV is greater than 10000"
* "Clean up my Updated Date column so that it is in this format YYYY-MM-DD"
* "Create a new calculated column called Profit that takes Total Sales minus Total Cost"
* "Join with the Customer Details DataSet on Customer Domain and include all columns from both DataSets"

Learn more about the [Advanced SQL Editor with AI Assistant](https://domo-support.domo.com/s/article/360046074774?language=en_US#advanced_sql_editor).

Learn more about [Domo. AI](https://ai.domo.com/) or explore [Domo. AI documentation](https://domo-support.domo.com/s/topic/0TO5w000000ZmGPGA0/domoai?language=en_US) in the Knowledge Base.

### Workflows

Workflows powerfully leverages other features, including [Code Engine](#code_engine), [Queues](#queues), and [Forms](#domo_forms), to allow you to design and implement executable workflow processes with low code capabilities. You can start a workflow manually, through an Alert, through an approval flow, or on a timer. Use our workflow template library to get started quickly.

### Code Engine

Code Engine is a Domo-native runtime environment that can execute JavaScript or Python code and interact with any feature inside or outside of Domo. You can use the Domo library of packages and functions for common integrations and services or create your own functions for use in your Domo instance. Code Engine functions integrate with Workflows without coding.

To learn more, see [Code Engine](https://domo-support.domo.com/s/article/000005173).

### Queues

Queues allows you to organize manual tasks into different queues and control who can access them. You can configure a queue to display tasks for an individual or a group. Each task is associated with a form generated by a workflow, and you can filter queues to view tasks by assignee or status, monitor queue volume, and customize layouts.

To learn more, see [Queues](https://domo-support.domo.com/s/article/000005172).

### JSON No Code & JSON No Code OAuth Connectors

The JSON No Code and JSON No Code OAuth connectors allow you to connect Domo to REST APIs that return data in a standard JSON format. Both connectors support standard authorization methods such as Basic Authentication, API Key, Bearer Token, and OAuth 2.0. Each connector includes three main areas of configuration: Connection Type, Request Type, and optional Advanced Settings.
Both connectors are available for download in the [Domo Appstore](https://www.domo.com/appstore/) [<InlineImage src="/images/kb/ka0Vq00000001FR-00N5w00000Ri7BU-0EMVq0000000XDh.jpg" />.](https://www.domo.com/appstore/)

To learn more, see [JSON No Code Connector](/s/article/360043433473) and [JSON No Code OAuth Connector](/s/article/360052105454).

# Between or In Range Gauge

The Between or In Range gauge helps you visualize results that are between lower and upper limits—showing whether the output is within the target range so you can quickly identify exceptions to the expected values. This chart is available in the Gauges section of Analyzer.
  
Font sizes, bar width, and colors are all customizable. See [Between or In Range Gauge](/s/article/000005373).

# Rich Table Borders

Mega Tables and Pivot Tables now allow for more border customization, including:

* Vertical border thickness and color
* Horizontal border thickness and color
* Outside borders
* Header borders

These changes do not affect existing tables. The new options are supported in Excel export.

# App Studio

The open beta for App Studio is available now.

App Studio is a low-code app builder that makes creating, distributing, and consuming Domo apps accessible to everyone. With pre-built themes and countless ways to customize the look and feel of your apps, App Studio is the next level of dashboard building experience, bringing the level of precision and navigation that previously required the App Dev Framework to your fingertips. App Studio is poised to become the ultimate destination for data presentation and compelling data narratives, all within a single app.

To learn more about App Studio, see the [Overview](https://domo-support.domo.com/s/article/000005295?language=en_US) in the Knowledge Base.
