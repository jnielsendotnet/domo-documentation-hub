title: "Elevation for Sensitive Changes"
---


### Intro

To enhance security, certain features now require *elevation* —a limited time window during which sensitive actions can be performed. You can begin an elevation by step-up authentication using a one-time passcode (OTP) generated by an authenticator app.

This article describes how to register Domo in your chosen authenticator app, how to start and end an elevation, and how admins can remove the requirement for elevation.

<Accordion title="Which actions require elevation?">
  * Changing another user's password.
  * Modifying authentication settings for the Domo instance.

  *In the future, other sensitive actions may also require elevation.*

</Accordion>

---

### Register Domo in Your Authenticator App

Starting an elevation requires entering a time-based one-time passcode (TOTP) generated by an authenticator app. Setting up your authenticator app is a one-time process. After setup, you can quickly generate a passcode whenever you need to begin an elevation.

<Accordion title="Which authenticator apps are supported?">
  Any authenticator app that supports time-based one-time passwords (TOTP) should be compatible with Domo. Examples include:

  * Microsoft Authenticator
  * Okta Verify

</Accordion>

### Start and End an Elevation

If you attempt to take an action that [requires an elevation](#require_elevation), you're prompted to start one. An elevation lasts for 15 minutes but can be ended early.

<Note>
  **Note:** If you haven’t yet done so, you’ll be prompted to [register Domo in your authenticator app](#register_domo) before you can start an elevation.
</Note>

When prompted to start an elevation, enter the passcode generated from your configured authenticator app.

<Frame><img alt="" src="/images/kb/ka0Vq000000CPM1-00N5w00000Ri7BU-0EMVq000006cGDC.jpg" style={{width: 530, height: 194}}/></Frame>
During an active elevation, a banner with a countdown timer displays. You can manually end the elevation by selecting **End Elevation** in the banner.

<Frame><img alt="" src="/images/kb/ka0Vq000000CPM1-00N5w00000Ri7BU-0EMVq000006cJcT.jpg" style={{width: 228, height: 85}}/></Frame>
### Remove Elevation Requirement

By default, elevation is required for the sensitive actions listed [above](#require_elevation). Required elevation is intended to increase security around these sensitive actions, so Domo recommends against removing it. However, admins can remove this requirement.

<Warning>
  **Important:** Admins must have the Admin system role or a custom role with the Manage All Company Settings grant enabled to change authentication settings. Learn more about [system roles](/s/article/360043438953) and [grants](/s/article/360043438973).
</Warning>

<Note>
  **Note:** By default, elevation is required to make changes to authentication settings, including the change to remove the requirement for elevation.
</Note>

Follow these steps to remove the requirement for elevation when making sensitive changes:

1. Navigate to **Admin** > **Authentication** > **Authentication**.
2. Under **Users**, uncheck the option labeled **Require OTP elevation for sensitive configuration changes**.
     
   <Frame><img alt="" src="/images/kb/ka0Vq000000CPM1-00N5w00000Ri7BU-0EMVq000006cHXS.jpg" style={{width: 448, height: 147}}/></Frame>
3. Save your changes.
