# New Features and Enhancements

## Activity Log Updates

We have converted the Activity Log to React. The following improvements come with this update:

* More intuitive event filtering with Event Type and Event broken out into separate dropdown filters.
* Date and Person filtering are now built into the column header rather than being separated into dropdown filter elements.
* A more compact user experience, making better use of space.

See [Activity Log](/s/article/360042934574) in the Knowledge Base.

## Removal of Auto Preview in Analyzer for Cloud Amplifier DataSets

Auto Preview is now disabled by default for Cloud Amplifier DataSets when you open Analyzer. This update reduces the number of queries produced and lowers costs. You can still turn on Auto Preview in the Analyzer menu.

## Workflows: Initialize Variables

You can now assign an initial value to a variable during the version-builder experience in Workflows. This feature eliminates the need to use start parameters or Code Engine functions to populate variables that require the same value at every start time.

Additional benefits include:

* **Simplified workflow design —** Assign initial values to variables directly during the build process.
* **Consistency —** Ensures that variables always contain the same initial value when initiating a workflow.
* **Improved efficiency —** Reduces unnecessary workflow steps.
* **Enhanced user experience —** Configure an initial value with a few clicks.
