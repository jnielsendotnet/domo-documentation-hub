## Elevation for Sensitive Changes

 With the Elevation for Sensitive Changes feature, actions like password changes now require an “elevation”:
 
 this isa time-bound period when sensitive actions can be taken. You can start an elevation by entering a one-time passcode (OTP) generated by an authenticator app. This update provides greater security for sensitive actions in your instance.
 
 Learn how to [register an authenticator app](https://domo-support.domo.com/s/article/000005831?language=en_US#register_domo)  and use the [elevation](https://domo-support.domo.com/s/article/000005831?language=en_US#start_end_elevation)  feature.

## Instance Templates

 Instance Templates allow you to create multiple instance feature templates by configuring different feature switches and behaviors in Domo.
 
 After configuring your Instance Templates, you can apply them when creating new instances. You can also apply templates to existing instances directly from the parent instance.
 
 The following features are available for Instance Templates:
 
* Customize Domo default settings for your instances to match your branding.
* Control what features the users in your managed instance have access to.

## v4 to v5 Dashboard Update

 As part of our regular technical maintenance cycle, we’re upgrading dashboards from Layouts v4 to Layouts v
5. This upgrade does not affect the core functionality of your dashboards, but it does enhance your editing experience to align with App Studio.
 
 You can expect the following updates:
 
* A more intuitive editing interface with click-and-drag handles.
* Layout aligners for easier content positioning.
* Minor shifts in the sizing of layout content. This may require some manual adjustments in existing dashboards.

 We recommend reviewing your dashboards after the update to ensure everything looks as expected.

## API Client Create

 API client creation and management can now be managed in your Domo instance in Admin Settings. Users with the existing Create API OAuth Clients grant, included in the Domo system roles, have access to the API Client Create feature.
 
 You can restrict the ability of users in your instance to create API clients by changing their user roles and grants as needed.
 
<Frame><img alt="" src="/images/kb/ka0Vq000000HWI1-00N5w00000Ri7BU-0EMVq000006bfAP.jpg" style={{width: 318, height: 398}}/></Frame>
<Note>
  **Note:** 
  The ability to create API clients is governed by the Create API OAuth Clients grant, which is included with all Domo system roles. This enhancement does not change your users’ ability to create API clients (does not change existing user roles or grants); however, since API clients can now be created in Domo, your users are more likely to see that capability. Please review your users’ roles and make any adjustments you deem necessary.

</Note>

# Report Builder for Scheduled Reports

 The Report Builder in App Studio allows you to build customizable email reports using your apps.
 
 Report Builder allows you to do the following:
 
* Include multiple app pages
* Drag, drop, swap, resize, and remove your content within the Report Builder
* Add additional content to enhance reports, including headers, borders, cards, and text
* Configuration options for customizing email headers and footers with logos and content
* Add a button that links to Domo content, including the app itself or a card/dashboard
* Add theme components, including card styles, background colors, and density, and import themes from any app to sync color palettes
* Add filters and schedule the report for users and groups

# SCIM APIs: User Enhancement and Groups

 SCIM–System for Cross-domain Identity Management–is a protocol that allows an identity provider (IdP) to push user and group creation, update, and deletion between software systems. While most SCIM User endpoints have been available in Domo previously, SCIM Group endpoints are also now available, allowing directory groups to be created and deleted, and group membership to reflect changes in the IdP, without requiring logging into Domo.
 
 Learn more about [SCIM in Domo](/s/article/000005241)  .
