# Nested Calculations

Users with the Edit Card grant enabled for their role can now access Nested Beast Mode calculations—these allow one calculation to reference another. Any changes made to a referenced calculation are automatically reflected in any calculations that reference it. Nesting is only permitted at one level—a calculation that references a previous calculation cannot be referenced by a subsequent calculation.

An accompanying DomoStats report — Beast Modes Used in Beast Modes — enables admins to track how nested calculations are being used in their instance.

To learn more, see [Nested Calculations](https://domo-support.domo.com/s/article/000005307?language=en_US).

#### Upsert

Upsert in Magic ETL allows you to update the rows in your output DataSet without reprocessing the full DataSet, based on a unique row-level key.

When selecting an update method for your DataFlow, you can define an upsert key in the configuration of your output DataSet. During the output step, Domo will check the unique keys resulting from the DataFlow processing - if those keys already exist in the output DataSet, they will be replaced with the updated rows of the same key.

For more information on Magic ETL, see our [Magic ETL documentation](https://domo-support.domo.com/s/topic/0TO5w000000ZanvGAC/magic-etl?language=en_US).

# Domo Cleanup for Workflows

The new Domo Cleanup for Workflows feature, powered by DomoStats, identifies unused content and surfaces it on a dashboard. This feature significantly streamlines BI content management and ensures a cleaner and more relevant environment.

To ensure that you are cleaning up the correct content, a QuickStart app is available in the Appstore to help you surface the content that needs to be archived.

# Publish "Read-only" Content

Publish “Read-Only” Content allows you to leverage the Domo Embed engine to publish read-only content to your subscribers. Read-only content does not allow subscriber changes, DataSets powering the content are not copied to the subscriber, and changes are pushed with no refresh required.

For more information on Publish, see [Publish Content to a Subscriber Organization](https://domo-support.domo.com/s/topic/0TO5w000000ZanbGAC/domo-everywhere?language=en_US).
