## Intro

Microsoft Azure Blob Storage is designed to easily and efficiently store any type of unstructured data, such as images, videos, audio, and documents. Use Domo's Azure Blob Storage SAS Token Writeback connector to export your data from a Domo DataSet into an Azure Blob container. To learn more about the Azure Blob Storage API, visit their page ([https://docs.microsoft.com/en-us/res...rvice-rest-api](https://docs.microsoft.com/en-us/rest/api/storageservices/blob-service-rest-api)).

You export data to Azure Blob Storage in the Data Center. This topic discusses the fields and menus that are specific to the Azure Blob Storage SAS Token Writeback connector user interface. General information for adding DataSets, setting update schedules, and editing DataSet information is discussed in [Adding a DataSet Using a Connector](/s/article/360042926274).

<Note>**Note:** The owner of a writeback dataset must also be an owner or co-owner of the input dataset.</Note>

## Prerequisites

To configure this connector, you will need the following:

- The Storage Account Name
- An Account access level SAS token

<Note>**Note:** This connector works only with the ** account access level SAS** token. So, if you receive an authentication error, please confirm if you are entering a container access level token instead.</Note>

This section enumerates the options in the **Credentials** and ** Details** panes in the Azure Blob Storage SAS Token Writeback Connector page. The components of the other panes in this page, ** Scheduling** and ** Name & Describe Your DataSet**, are universal across most connector types and are discussed in greater length in [Adding a DataSet Using a Connector](/s/article/360042926274).
