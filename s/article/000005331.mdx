### Intro

Depending on the complexity of your process, creating and configuring a workflow for the first time can require significant time and effort. You can always contact your Domo account team for support.

This article describes how to create a workflow from our library of pre-built templates or from a blank canvas.

After creating a workflow version, you must *validate*,*deploy*, and *start* it. Learn how below.

<Note>
**Glossary:** *Italicized* terms in this article have particular definitions within Workflows. Visit the Workflows [glossary](/s/article/000005108) to learn the definitions.
</Note>

<Tip>
**Learn more:** See [Manage Workflows](/s/article/000005179) to learn about workflow *versions*, the *executions* list, and tracking information.
</Tip>


---

### Create a Workflow from a Template

You can create a workflow from the template library or create a "template" from an existing workflow you have access to by creating a new *version* of it or performing a **Save As** action to reuse and customize it.

<Note>**Note:** You must have the Create Workflows grant to create a workflow. Learn more about [required grants](/s/article/000005179) for Workflows.</Note>

1. From your default Workflows view, select **+ New Workflow** > **Create from Template**to open the template library.
2. Choose a template from the library and enter a name in the **Model Name** field. (The max character limit is 75.) Your workflow must have a name to proceed.
   - If you change the pre-populated version number, the number must be in the <*<major.minor.patch>>* format.  
       

     
<Tip>**Tip:** It's good practice to use a name and description that helps you identify the workflow in the workflows list.
3. Select **Use This Template** to open the template in the Workflows [canvas](#design_your_workflow).</Tip>

   <Frame>![Screenshot](/images/kb/0EMVq000005xijh.jpg)</Frame>
4. Make any changes to the template configuration. Configuration options include:
   - Selecting a [shape](#add_shapes_to_your_canvas) and configuring it in the right configuration panel. Click an empty area of the canvas to close the configuration panel.
   - Select a flow (connect), or line, between the shapes to view the <img alt="add remove shape.jpg" src="/images/kb/0EMVq000002lgib.jpg" style={{width: 20, height: 43, display: 'inline', verticalAlign: 'start', margin: '0'}}/> **Add**/**Remove shape** controls and change shapes on the canvas.
   - Click-and-drag shapes and flows around the canvas to change the design of your workflow.  
       

     <Frame>![Screenshot](/images/kb/0EMVq000005s3yv.jpg)</Frame>
5. Select **Save**to keep your changes.

**Next step:** Learn how to [validate](#validate_a_workflow) your workflow.

### Create a Workflow from a Blank Canvas

<Note>**Note:** You must have the Create Workflows grant to create a workflow. Learn more about [required grants](/s/article/000005179) for Workflows.</Note>

1. From your default Workflows view, select **+ New Workflow** > **Blank Workflow**.
2. Enter a name for your workflow. (The max character limit for the name is 75.)
3. Select **Save** and open the [canvas](#design_your_workflow).
   - If you change the pre-populated version number, the number must be in the *<major.minor.patch>* format.
4. Add shapes and flows, or lines, to the canvas and configure them. Configuration options include:  
   - Selecting a [shape](#add_shapes_to_your_canvas) and configuring it in the right configuration panel. Click a blank space on the canvas to close the configuration panel.
   - Select a flow (connect), or line, between the shapes to view the <img alt="add remove shape.jpg" src="/images/kb/0EMVq000002lgib.jpg" style={{width: 20, height: 43, display: 'inline', verticalAlign: 'start', margin: '0'}}/> **Add**/**Remove shape**controls and change shapes on the canvas.

1. - Click-and-drag shapes and flows around the canvas to change the design of your workflow.
2. Select **Save** to keep your changes.

**Next step:** Learn how to [validate](#validate_a_workflow) your workflow.

# Save Your Workflow

You can keep the changes to your workflow by selecting **Save**.

<Note>**Note:** Before saving, you must configure every shape on the canvas. Otherwise, you'll receive an error message.</Note>

<Frame>![Screenshot](/images/kb/0EMVq000005s4qA.jpg)</Frame>

#### Auto Save

The workflow tool automatically saves the changes you make to the workflow canvas. If there’s a 10-second pause with no mouse movement or keyboard input, your edits are saved automatically.

You can check the save status by looking at the indicator in the top header. If there are unsaved changes, it shows “Unsaved Changes.” Once everything is saved—either manually or automatically—it updates to “All changes saved.”

<Note>**Note:** Auto save only applies to edits on the canvas. Changes made within configuration modals won’t be saved automatically. Always check the save indicator to confirm that your changes have been saved.</Note>

<Frame>![Screenshot](/images/kb/0EMVq00000DTW0D.jpg)</Frame>

#### Locking

When editing a workflow version, it is locked to prevent other users from making changes and possibly overwriting your work. If there’s no activity for 30 minutes, the lock releases, and the workflow switches to View Only mode.

To continue editing after the lock is released, simply click the “Edit Workflow” option in the three-dot menu.

#### Perform a Save As

You can perform a 'save as' for your workflow by selecting <img alt="" src="/images/kb/0EMVq000005s8Ab.jpg" style={{width: 30, height: 28, display: 'inline', verticalAlign: 'start', margin: '0'}}/> **More** (vertical three-dot menu) > **Save As**. If you have unsaved changes, you'll receive a warning message, but you can choose to 'save as' anyway and give the new version a number.

<Frame>![save as workflow.jpg](/images/kb/0EMVq000002mMbN.jpg)</Frame>



# Configure Workflow Start

Whether you create a new workflow from a template or a blank canvas, you must configure the workflow starting shape.

<Note>**Note:** Instead of using a manual start, you can trigger a workflow using a timer or from inside another Domo feature, such as App Studio. Triggers can only be configured after the workflow is [deployed](#deploy_a_workflow). Learn how to [configure a trigger](/s/article/000005179) for a deployed workflow version.</Note>

Select the default starting shape on the canvas to open the configuration panel. The title for a start shape defaults to "Start *Workflow Name*." You can change the title either by double-clicking inside the start shape to edit the text or by editing the title field in the configuration panel.   
In the configuration panel, you can also enter workflow description.

In the panel, you can add and configure any parameters the workflow needs as inputs. These can be entered by the workflow user via a form or given an initial or default value that is always passed when the workflow starts.

<Note>**Note:** If your workflow will loop, and you would rather not introduce a manual step for each execution or use a complex code function, you can assign an initial value to a variable, allowing the workflow to begin always using that value for the variable. Learn how to [assign an initial value to a variable](#add_a_variable) above.</Note>

<Frame>![Screenshot](/images/kb/0EMVq000005VAt3.jpg)</Frame>

   Parameters with a default value display with a blue dot in the parameters list.

**User Tasks**

A user task represents a human interaction. It can have multiple incoming flows but only one outgoing flow.

<Frame>![Screenshot](/images/kb/0EMVq00000DP0LV.jpg)</Frame>

- *Email —* Automatically sends an email to chosen recipients. Recipients can then respond using options embedded in the email body. Learn how to [configure an email](/s/article/000005797).

### Validate a Workflow

Validating your workflow identifies issues and incorrectly configured shapes. If there are issues, the system navigates you to a Validation view to correct them.

There are three kinds of validation issues:

- **Informational —** You don't need to correct informational issues before deploying your workflow. These issues are typically superficial and don't impact workflow performance. Informational issues are color-coded blue.
- **Warning —** Youdon't need to correct warnings before deploying a workflow, but we strongly recommend doing so. Warnings usually indicate that a best practice is not being followed and are color-coded yellow.
- **Error —**Errors must be corrected before you can deploy a workflow. Errors are color-coded red.

# Access Validation Mode

You can put a workflow version into validation mode by deploying it from either of two locations:

- From the canvas, choose **Action menu** > **Deploy**.
- From the [version list](/s/article/000005179), select <img alt="action menu.jpg" src="/images/kb/0EMVq000002Hc4Z.jpg" style={{width: 20, height: 25, display: 'inline', verticalAlign: 'start', margin: '0'}}/> **Action** **menu** (three vertical dots icon) > **Deploy**.

If the version has validation issues, a dialog displays with a border the color of the most severe issue (blue, yellow, or red). The dialog prompts you to either cancel the deployment or review the issues in validation mode. All errors must be resolved before you can deploy the workflow.

<Frame>![Screenshot 2024-07-09 at 1.44.50 PM.png](/images/kb/0EMVq000001giyZ.jpg)</Frame>

   
  
If only warnings or information issues are found, a dialog with an orange border displays. You can either review the warnings in validation mode or check the box to acknowledge their potential negative impact during runtime and select **Deploy** to proceed with the deployment.

<Frame>![review warnings dialog.jpg](/images/kb/0EM5w000006uOMl.jpg)</Frame>

From either dialog, selecting **Review Issues** opens the canvas in validation mode.

### Deploy a Workflow

Workflows must be deployed before they can run. A deployed *version* can be started manually or started by a trigger. Deploying a version puts it into [validation mode](#validate_a_workflow); after it's validated, it deploys.

If you have Deploy or Admin [permission](/s/article/000005179) to the workflow, you can select **Action menu** > **Deploy** for a version from the version list or from inside edit or view mode.

**Next step:** Learn how to [start](#start_a_workflow) your workflow.

### Start a Workflow

To start a workflow, you must have Execute or Admin [permission](/s/article/000005179) to it. Your workflow must be [validated](#validate_a_workflow) and [deployed](#deploy_a_workflow) before starting.

<Warning>**Important:** You can only start a deployed version if it is active. Deactivated versions can't be executed.</Warning>

You can start a workflow version from the version list page by selecting <img alt="" src="/images/kb/0EMVq000005uIoz.jpg" style={{width: 25, height: 22, display: 'inline', verticalAlign: 'start', margin: '0'}}/> **Start** (or from the canvas in edit or view-only mode by selecting <img alt="action menu.jpg" src="/images/kb/0EMVq0000026oDt.jpg" style={{width: 21, height: 26, display: 'inline', verticalAlign: 'start', margin: '0'}}/> **Action menu** > **Start**).

<Note>**Note:** Users with the Manage Workflows grant *and* Admin or Execute permission to a workflow can configure alternative triggers for it, such as from within an App Studio app. Learn more about [workflow triggers](/s/article/000005179).</Note>
