### Intro

Depending on the complexity of your process, creating and configuring a workflow for the first time can require significant time and effort. You can always contact your Domo account team for support.

This article describes how to create a workflow from our library of pre-built templates or from a blank canvas.

After creating a workflow version, you must *validate*, *deploy*, and *start* it. Learn how below.

<Note>
  **Glossary:** *Italicized* terms in this article have particular definitions within Workflows. Visit the Workflows [glossary](https://domo-support.domo.com/s/article/000005108?language=en_US#glossary) to learn the definitions.
</Note>

<Tip>
  **Learn more:** See [Manage Workflows](https://domo-support.domo.com/s/article/000005179?language=en_US) to learn about workflow *versions*, the *executions* list, and tracking information.
</Tip>

You can create new workflows as long as you have at least an Editor system role or a custom role with the Create Workflows grant enabled.

<Note>
  **Note:** If you can't access Workflows even with the appropriate grant, contact your Domo account team to enable the feature.

</Note>

### Create a Workflow from a Template

You can create a workflow from the template library or create a "template" from an existing workflow you have access to by creating a new *version* of it or performing a **Save As** action to reuse and customize it.

<Note>
  **Note:** You must have the Create Workflows grant to create a workflow. Learn more about [required grants](https://domo-support.domo.com/s/article/000005179?language=en_US#required_grants) for Workflows.
</Note>

1. From your default Workflows view, select **+ New Workflow** > **Create from Template** to open the template library.
2. Choose a template from the library and enter a name in the **Model Name** field. (The max character limit is 75.) Your workflow must have a name to proceed.
   * If you change the pre-populated version number, the number must be in the \< *\<major.minor.patch\>>* format.
       
     <Tip>
       **Tip:** It's good practice to use a name and description that helps you identify the workflow in the workflows list.

     </Tip>

3. Select **Use This Template** to open the template in the Workflows [canvas](#design_your_workflow).
     
   <Frame><img alt="" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005xijh.jpg" style={{width: 897, height: 434}}/></Frame>

4. Make any changes to the template configuration. Configuration options include:
   * Selecting a
     [shape](#add_shapes_to_your_canvas) and configuring it in the right configuration panel. Click an empty area of the canvas to close the configuration panel.
   * Select a flow (connect), or line, between the shapes to view the
     <InlineImage src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000002lgib.jpg" /> **Add** / **Remove shape** controls and change shapes on the canvas.
   * Click-and-drag shapes and flows around the canvas to change the design of your workflow.
       
     <Frame><img alt="" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005s3yv.jpg" style={{width: 896, height: 430}}/></Frame>

5. Select **Save** to keep your changes.

   <Tip>
     **Next step:** Learn how to [validate](#validate_a_workflow) your workflow.
   </Tip>

### Create a Workflow from a Blank Canvas

<Note>
  **Note:** You must have the Create Workflows grant to create a workflow. Learn more about [required grants](https://domo-support.domo.com/s/article/000005179?language=en_US#required_grants) for Workflows.
</Note>

1. From your default Workflows view, select **+ New Workflow** > **Blank Workflow**.
2. Enter a name for your workflow. (The max character limit for the name is 75.)
3. Select **Save** and open the [canvas](#design_your_workflow).
   * If you change the pre-populated version number, the number must be in the *\<major.minor.patch\>* format.
4. Add shapes and flows, or lines, to the canvas and configure them. Configuration options include:
     
   * Selecting a
     [shape](#add_shapes_to_your_canvas) and configuring it in the right configuration panel. Click a blank space on the canvas to close the configuration panel.
   * Select a flow (connect), or line, between the shapes to view the
     <InlineImage src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000002lgib.jpg" /> **Add** / **Remove shape** controls and change shapes on the canvas.

* Click-and-drag shapes and flows around the canvas to change the design of your workflow.

1. Select **Save** to keep your changes.

   <Tip>
     **Next step:** Learn how to [validate](#validate_a_workflow) your workflow.
   </Tip>

# Add a Variable

Follow these steps to add a variable to the **Workflow variables** list:

1. In the canvas left navigation, go to the **Workflow variables** tab.
     
   <Frame><img alt="Screenshot 2025-01-14 at 5.50.47 PM.png" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000004ohDx.jpg" style={{width: 400, height: 274}}/></Frame>

2. Select **Add Variable** to open the modal.
3. Enter a name in the **Variable Name** field. You cannot use spaces or special characters in the variable name.
4. Choose a data type from the **Data Type** dropdown. The data type may require a subtype or have additional options. Learn more about [data types](https://domo-support.domo.com/s/article/000005797?language=en_US#data_types).
     
   If you want to make it a list variable, check the box labeled **Make List**.
     
   <Frame><img alt="Screenshot 2025-01-14 at 5.52.15 PM.png" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000004og0A.jpg" style={{width: 650, height: 229}}/></Frame>

5. (Optional) If you want this variable to always have a certain value when the workflow executes, select **+ Add Initial Value**.
     
   You can then enter an initial value, depending on the data type. Learn more about [data types](https://domo-support.domo.com/s/article/000005797?language=en_US#data_types).
     
   <Frame><img alt="Screenshot 2025-01-14 at 5.58.13 PM.png" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000004ohkD.jpg" style={{width: 645, height: 261}}/></Frame>

# Configure Workflow Start

Whether you create a new workflow from a template or a blank canvas, you must configure the workflow starting shape.

<Note>
  **Note:** Instead of using a manual start, you can trigger a workflow using a timer or from inside another Domo feature, such as App Studio. Triggers can only be configured after the workflow is [deployed](#deploy_a_workflow). Learn how to [configure a trigger](https://domo-support.domo.com/s/article/000005179?language=en_US#manage_triggers) for a deployed workflow version.
</Note>

Select the default starting shape on the canvas to open the configuration panel. The title for a start shape defaults to "Start *Workflow Name*." You can change the title either by double-clicking inside the start shape to edit the text or by editing the title field in the configuration panel.
In the configuration panel, you can also enter workflow description.

In the panel, you can add and configure any parameters the workflow needs as inputs. These can be entered by the workflow user via a form or given an initial or default value that is always passed when the workflow starts.

<Note>
  **Note:** If your workflow will loop, and you would rather not introduce a manual step for each execution or use a complex code function, you can assign an initial value to a variable, allowing the workflow to begin always using that value for the variable. Learn how to [assign an initial value to a variable](#add_a_variable) above.
</Note>

<Frame><img alt="" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005VAt3.jpg" style={{width: 899, height: 377}}/></Frame>

#### Add Parameters

Parameters require or allow data values to be passed into the workflow when it executes. Follow these steps to add and configure your starting parameters:

1. Select your start shape to open the configuration panel, and select **Add Parameter** to open the new parameter modal.
     
   <Frame><img alt="" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005xrbi.jpg" style={{width: 250, height: 365}}/></Frame>

2. In the modal, enter a name for the parameter. Spaces and special characters are not allowed.
3. Choose a data type for the parameter. Learn about the available [data types](https://domo-support.domo.com/s/article/000005797?language=en_US#data_types).
     
   <Frame><img alt="" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005VCBh.jpg" style={{width: 500, height: 392}}/></Frame>

4. Use the **Required** toggle to indicate whether the parameter is required. By default, new parameters are marked as required.
5. (Conditional) If you want to assign a default value to a parameter, select **+ Add Default Value** after defining the name and data type. Default values can be changed until that workflow version is deployed.
     
   <Frame><img alt="" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005VD9N.jpg" style={{width: 602, height: 212}}/></Frame>

   Parameters with a default value display with a blue dot in the parameters list.
     
   <Frame><img alt="" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005sFC5.jpg" style={{width: 300, height: 93}}/></Frame>

6. Select **Add Parameter**.
   * (Optional) You can choose **More** (orange down arrow icon) > **Save and add another** if you are adding more than one parameter.
       
     <Frame><img alt="" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005V7Dg.jpg" style={{width: 499, height: 231}}/></Frame>

7. Repeat these steps for as many start parameters as needed.

The configured start parameters display in the configuration panel under **Parameters & Mapping**. A red asterisk beside the name indicates that the parameter is required for the workflow to start.
  
<Frame><img alt="" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005sDjm.jpg" style={{width: 250, height: 339}}/></Frame>

**Note:** A corresponding variable with the same name and data type is mapped to the parameter and added to the workflow variables list, accessible from the left navigation. You can map a parameter to a different variable, but it must be of the same data type.
  
<Frame><img alt="" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005sDob.jpg" style={{width: 250, height: 249}}/></Frame>

Make changes to a parameter by hovering over it and selecting <InlineImage src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005sEeD.jpg" /> **Edit**.

# Add Shapes to Your Canvas

Your workflow is a series of shapes—representing actions—connected by flows (or lines). After adding shapes to your canvas, you can drag them around to change the visual design of your workflow. Dragging a shape also drags any flows attached to it.

Add a shape to the canvas by selecting <InlineImage src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000002mjZZ.jpg" /> **Add action** at the end of any flow to display the action menu.
  
Select <InlineImage src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005sH93.jpg" /> **Add Flow** to add another connecting line.

<Frame><img alt="" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005sGeP.jpg" style={{width: 248, height: 197}}/></Frame>

<Frame><img alt="Screenshot 2024-09-18 at 1.27.45 PM.png" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000002mkCH.jpg" style={{width: 250, height: 696}}/></Frame>

In the action menu, possible actions are grouped together, as described below.

* **Flow Controls —** Flow controls manage the direction of the workflow. There are four available:
  + *Conditional Logic —* Applies IF/THEN statements to your workflow. Learn how to configure [conditional logic](https://domo-support.domo.com/s/article/000005797?language=en_US#conditional_logic).
  + *Parallel Logic —* Forces your workflow to perform independent steps in parallel. Learn how to configure [parallel logic](https://domo-support.domo.com/s/article/000005797?language=en_US#parallel_logic).
  + *Wait —* Delays the workflow for a specified interval before allowing it to continue. Wait actions can only have one outgoing flow.
      
    After adding a wait action, you can name it in the configuration panel. Use the **settings** dropdown in the parameter section to enter a custom value.
      
    <Frame><img alt="" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005sKRt.jpg" style={{width: 350, height: 193}}/></Frame>

  + *End —* Terminates the workflow. A workflow can have multiple End shapes if the design has places where one flow branches into two or more. Name and describe each End shape in the configuration panel.
      
    <Frame><img alt="Screenshot 2024-08-02 at 3.02.49 PM.png" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq0000023tLO.jpg" style={{width: 601, height: 244}}/></Frame>

* **User Tasks —** A user task represents a human interaction. It can have multiple incoming flows but only one outgoing flow. There are two kinds of user tasks:
  + *Email —* Automatically sends an email to chosen recipients. Recipients can then respond using options embedded in the email body. Learn how to [configure an email](https://domo-support.domo.com/s/article/000005797?language=en_US#email).
  + *General task —* Allows you to choose a queue and assign a needed form to a specific user. Learn how to [configure a general task](https://domo-support.domo.com/s/article/000005797?language=en_US#general_task).
* **Custom Functions —** A custom function automatically executes a pre-defined code package to perform a specific job before moving to the next step in the workflow. It can have multiple incoming flows but only one outgoing flow.
    
  Learn how to [configure](https://domo-support.domo.com/s/article/000005797?language=en_US#custom_function) and [test](https://domo-support.domo.com/s/article/000005797?language=en_US#test_a_function) a custom function.
* **AI Tools —** Pre-built AI functions to perform tasks in Domo. These include:
  + *Domo AI Service Layer —* Use services like Image to Text, Text Generation, and Text to SQL.
      
    <Frame><img alt="Screenshot 2025-02-06 at 5.43.46 PM.png" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005IkXl.jpg" style={{width: 242, height: 446}}/></Frame>

  + *Domo Jupyter —* Add and execute a Jupyter Notebook in your workflow. Learn more about [Jupyter Notebooks in Workflows](https://domo-support.domo.com/s/article/36004740075?language=en_US#executions_via_workflows).
      
    <Frame><img alt="Screenshot 2025-02-06 at 5.43.04 PM.png" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005IlnB.jpg" style={{width: 242, height: 444}}/></Frame>

* **Query Table —** Query and retrieve data from a Domo DataSet in your workflow. You can include variables in your query and preview it before sending.
    
  Learn how to add a [query table](https://domo-support.domo.com/s/article/000005797?language=en_US#query_table).

  <Warning>
    **Remove a shape:** To remove a shape from the canvas, select it to display the configuration panel. In the panel, select **More** (vertical three-dot icon) > **Delete**.

    <Frame><img alt="remove a shape.jpg" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000002tD9x.jpg" style={{width: 899, height: 302}}/></Frame>
  </Warning>

### Validate a Workflow

Validating your workflow identifies issues and incorrectly configures shapes. If there are issues, the system navigates you to a Validation view to correct them.

There are three kinds of validation issues:

* **Informational —** You don't need to correct informational issues before deploying your workflow. These issues are typically superficial and don't impact workflow performance. Informational issues are color-coded blue.
* **Warning —** You don't need to correct warnings before deploying a workflow, but we strongly recommend doing so. Warnings usually indicate that a best practice is not being followed and are color-coded yellow.
* **Error —** Errors must be corrected before you can deploy a workflow. Errors are color-coded red.

# Access Validation Mode

You can put a workflow version into validation mode by deploying it from either of two locations:

* From the canvas, choose **Action menu** > **Deploy**.
* From the
  [version list](https://domo-support.domo.com/s/article/000005179?language=en_US#manage_workflow_versions), select <InlineImage src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000002Hc4Z.jpg" /> **Action** **menu** (three vertical dots icon) > **Deploy**.

If the version has validation issues, a dialog displays with a border the color of the most severe issue (blue, yellow, or red). The dialog prompts you to either cancel the deployment or review the issues in validation mode. All errors must be resolved before you can deploy the workflow.

<Frame><img alt="Screenshot 2024-07-09 at 1.44.50 PM.png" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000001giyZ.jpg" style={{width: 501, height: 226}}/></Frame>

If only warnings or information issues are found, a dialog with an orange border displays. You can either review the warnings in validation mode or check the box to acknowledge their potential negative impact during runtime and select **Deploy** to proceed with the deployment.

<Frame><img alt="review warnings dialog.jpg" src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EM5w000006uOMl.jpg" style={{width: 500, height: 246}}/></Frame>

From either dialog, selecting **Review Issues** opens the canvas in validation mode.

### Deploy a Workflow

Workflows must be deployed before they can run. A deployed *version* can be started manually or started by a trigger. Deploying a version puts it into [validation mode](#validate_a_workflow); after it's validated, it deploys.

If you have Deploy or Admin [permission](https://domo-support.domo.com/s/article/000005179?language=en_US#manage_permissions) to the workflow, you can select **Action menu** > **Deploy** for a version from the version list or from inside edit or view mode.

<Tip>
  **Next step:** Learn how to [start](#start_a_workflow) your workflow.
</Tip>

### Start a Workflow

To start a workflow, you must have Execute or Admin [permission](https://domo-support.domo.com/s/article/000005179?language=en_US#manage_permissions) to it. Your workflow must be [validated](#validate_a_workflow) and [deployed](#deploy_a_workflow) before starting.

<Warning>
  **Important:** You can only start a deployed version if it is active. Deactivated versions can't be executed.
</Warning>

You can start a workflow version from the version list page by selecting <InlineImage src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq000005uIoz.jpg" /> **Start** (or from the canvas in edit or view-only mode by selecting <InlineImage src="/images/kb/ka0Vq000000Gp1l-00N5w00000Ri7BU-0EMVq0000026oDt.jpg" /> **Action menu** > **Start**).
