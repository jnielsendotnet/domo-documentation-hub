### Intro

You can customize Domo by adding, renaming, reordering, and hiding dashboards; and adding sub-dashboards to parent dashboards. You can have up to three levels of sub-dashboards.

To learn how to personalize the layout of cards on dashboards, see our [Card and Dashboard Management](https://domo-support.domo.com/s/topic/0TO5w000000ZanDGAS/card-and-dashboard-management) documentation. To learn how to share access to a dashboard, see [Sharing and Removing Access to Cards and Dashboards](/s/article/360042932994).

For general information about dashboards, see [Get Started with Dashboards](/s/article/360043428553).

For information about locking cards on a dashboard so that only the dashboard owner and users with an Admin default security role can reposition and resize them, see [Lock and Unlock Dashboard Content](/s/article/360042923814). Learn about [default security roles](/s/article/360043438953).

# Add a Dashboard

When you add a dashboard or sub-dashboard, it is unshared (or private) by default. Only dashboard owners and users with either an Admin default security role or a custom role with the Manage All Cards, Pages and Apps (App Studio) grant enabled can view private dashboards. You can share access to the dashboard with other users.

To learn about sharing access to content, see [Managing Custom Roles](/articles/en_US/Knowledge/360043438973).

1. [Access the **Manage Dashboards** modal.](#access_manage_dashboards_modal)

# Add a Sub-dashboard

You can add a sub-dashboards to any top-level dashboard in Domo.

<Note>
**Notes:**

- You *cannot* have a private dashboard that has a shared sub-dashboard. If you share a sub-dashboard, Domo shares its parent dashboard with the same users and groups as the shared sub-dashboard to keep dashboard hierarchy. However, content on the parent dashboard is not shared. If you want the recipient to have access to both dashboards, share the parent dashboard and then the sub-dashboard.
- You can have a shared parent dashboard with private and shared sub-dashboards.
- You can create up to three levels of sub-dashboards.
</Note>

Follow these steps to add a sub-dashboard:

1. [Access the **Manage Dashboards** modal.](#access_manage_dashboards_modal)
2. In the list of dashboards, locate the dashboard you want to add a sub-dashboard to and select <img alt="show_subpages.png" src="/images/kb/0EM5w000005vOG6.png" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/> **Expand**.    <Frame>![editable name.jpg](/images/kb/0EMVq000000kw2d.jpg)</Frame>
4. Enter a name for the new sub-dashboard. Names can contain up to 25 characters.

# Rename Dashboard/Sub-dashboard

You can rename any dashboard or sub-dashboard in Domo to which you have access inside the Manage Dashboards modal. Follow the steps below.

1. [Access the **Manage Dashboards** modal.](#access_manage_dashboards_modal)
2. Locate the name of the dashboard/sub-dashboard you want to rename and select it.  
     

   <Frame>![Screenshot 2024-04-19 at 11.36.34 AM.png](/images/kb/0EMVq000000mmOv.jpg)</Frame>
3. In the field, replace the current name with the new name. Names can contain up to 25 characters.
4. Select **Save**.
5. Select **Done** to close the modal.

# Reorder Dashboards/Sub-dashboards

In the Manage Dashboards modal, you can change the order of dashboards and sub-dashboards by dragging the handles beside the names. You can move a sub-dashboard above or below other sub-dashboards under the same top-level dashboard, but you cannot move it elsewhere in the hierarchy.

If you have an Admin default security role, you can change the hierarchical level of dashboards in the Admin Settings. See the Move Dashboards in the Hierarchy section below.

<Note>**Note:** Users with a Participant default security role cannot reorder sub-dashboards; they can only change the order of top-level dashboards. For more information about user access, see [Default Security Role Reference](/s/article/360043438953).</Note>

<Tip>**Tip:** You can reorder sub-dashboards of a locked dashboard in the Company Settings. For more information, see [Lock and Unlock Dashboard Content](/articles/en_US/Knowledge/360042923814).</Tip>

Follow these steps to move a dashboard or sub-dashboards somewhere else in Domo:

1. [Access the **Manage Dashboards** modal.](#access_manage_dashboards_modal)
2. Use the handle by the dashboard/sub-dashboard name to drag it to where you want it in the list.
3. Select **Done**.

<Note>**Note:** You can only create three levels of sub-dashboards.</Note>

# Change Dashboard Owner

You can change the dashboard owner if you have an Admin [default security role](/s/article/360043438953), a custom role with the (admin-level) Manage All Cards, Pages and Apps (App Studio) and Pages grant enabled, or if you are assigned as the dashboard owner. Change dashboard owners one dashboard at a time or change the owner of multiple dashboards at once in the Admin Settings.

<Note>**Note:** You can only take bulk actions on dashboards if you have an Admin default security role or have the Manage All Cards, Pages and Apps (App Studio) grant enabled. To learn about default security roles in Domo, see [Managing Custom Roles](/s/article/360043438973).</Note>

Follow these steps to change the owner of a single dashboard:

1. Navigate to the dashboard for which you want to change ownership.
2. Locate the current owner information and select **(change)**. <img alt="select change owner.jpg" src="/images/kb/0EM5w000006vr5t.jpg" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/> The ** Manage Ownership** modal opens.   
4. (Optional) Write a message to the new owner(s) in the message field.
5. (Optional) Check the box labeled **Send email notifications** if you want to notify the new owners by email of the change in ownership.
6. Select **Add**.

Ownership of the Page is transferred to the user or group you selected.

### Admin Tasks

Admins can perform the following tasks.

# Move Dashboards

Admins can move dashboards and change their position in the hierarchy. For example, they can change a top-level dashboard into a sub-dashboard by moving it under another top-level dashboard. Admins include those with an Admin default security role or a custom role with the Manage All Cards, Pages and Apps (App Studio) grant enabled. For more information about default security roles in Domo, see [Managing Roles](/s/article/360043428253).

You can move dashboards one by one or multiple dashboards at a time.

When you move a dashboard, you can make it either a parent or sub-dashboard. When you turn a dashboard into a sub-dashboard, you must select a new parent dashboard. You can use any other parent dashboard or sub-dashboard as the new parent dashboard.

#### Permissions

When you move a dashboard, you must determine access to the dashboard. <img alt="move dashboards permissions.jpg" src="/images/kb/0EM5w000006vqmN.jpg" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/> Use permissions to give access to the appropriate people. This table describes the available permissions:

| Option | Description |
| --- | --- |
| Use permissions for this dashboard | Current permissions for the dashboard remain the same.  Only people with current access to this dashboard will be able to access it in the new location. |
| Inherit permissions from top dashboard | *This option is only available if you change the dashboard into a sub-dashboard.*  Only people with permission to the sub-dashboard's new parent dashboard can access the dashboard after the move. |
| Merge permissions | *This option is only available if you change the dashboard into a sub-dashboard.*  Everyone with access to the dashboard before the move and everyone with access to the new parent dashboard can access the dashboard after the move. |
| Remove permissions | Removes all permissions from the dashboard. |

Follow these steps to move dashboards:

1. Access the Admin Settings.
2. Under **Content**, select ** Dashboards** to view a list of all dashboards in the instance.  
     
   You can use the Column Filters to filter the dashboards in the list. <img alt="dashboards column filters.jpg" src="/images/kb/0EM5w000006vqqB.jpg" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/> 3. Check the box for every dashboard you want to move.   
5. In the modal under **Choose location**, select ** Top level dashboard** or select ** Make subpage** if you want the dashboards to be the sub-dashboards of another top-level dashboard.
6. (Conditional) If you select **Make subpage**, select the top-level dashboard you want the new sub-dashboard to be under from the dropdown. The top results of your search appear automatically in a list as you enter characters and filter accordingly as you add or remove characters.
7. Under **Permissions**, select the desired permissions option for these dashboards. For more information about the available options, see the [Permissions](#permissions) section above.
8. Select **Move Dashboards**.  
   The changes you specified take effect.

# Delete Dashboards and Sub-dashboards

Depending on your security role, you can delete dashboards from Domo in the Admin Settings.

- If you have a Privileged or Editor [default security role](/s/article/360043438953) or a custom role with the Edit Pages grant enabled, you can delete any dashboards/sub- you own.
- If you have an Admin [default security role](/s/article/360043438953) or a custom role with the Manage All Cards, Pages and Apps (App Studio) grant enabled, you can delete any dashboard/sub- in Domo.

For more information about default security roles in Domo, see [Managing Custom Roles](/s/article/360043438973).

If you delete a dashboard and the dashboard has any subs, those subs are deleted. This does not delete any cards on the dashboard or sub-. You can find them on the **Cards** page of your Content Settings.  
  
Follow these steps to delete a dashboard/sub-dashboard:

1. Access the Admin Settings.
2. Under **Content**, select ** Dashboards** to open the dashboards list.
3. Check the box beside the dashboard/sub-dashboard you want to delete.
4. Select **Edit** > <img alt="delete 1.png" src="/images/kb/0EM5w000006vr1X.jpg" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/> ** Delete dashboard**.   
   A dialog displays to confirm dashboard deletion.   
     

   <Frame>![Screenshot 2024-04-16 at 2.39.49 PM.png](/images/kb/0EMVq000000l3Lp.jpg)</Frame>
5. Select **Delete Dashboards** or ** Cancel**.
