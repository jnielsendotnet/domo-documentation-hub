## Intro

This article contains instructions for deploying, requesting access to, and removing an app from your Domo instance.

---

## Deploy an App

In the Appstore, you can deploy an app to your organization's Domo instance. If any installations of the app are already deployed to your Domo instance, you have the option to request access to an existing version or to deploy a new version.

When you deploy an app, Domo first installs the app card(s) with sample data. Cards with sample data display with a label reading **Sample** to indicate they are not yet live.

<Frame><img alt="sample data.jpg" src="/images/kb/ka05w000001284M-00N5w00000Ri7BU-0EM5w000006uUr7.jpg" style={{width: 771.997, height: 708.993}}/></Frame>

After you deploy an app, look over the app card(s) to make sure this app is what you need. When you are ready, you can power up the cards in the app with live data by connecting to a third-party connector account (Dashboard apps) or DataSet with matching columns (all other apps). You can also assign another user to power up the app.

Follow these steps to deploy an app to Domo:

1. In the navigation header, select **Appstore**.
2. In the Appstore, search for and find the app you want to add. To learn more about searching in the Appstore, see
   [Appstore Overview](/s/article/360043438013).
3. Select the app to display its Details view.
4. In the Details view, review the requirements and notes for the app.
5. Select **Get**. Some apps may have different labels, for this option. A dialog displays, informing you that your app is being created. You can select **Notify Me** to continue working and receive a notification when the app is ready. You can also wait and be taken directly to the app when it is ready.

   <Warning>
     **Conditional:** If any versions of this app are already installed in your instance, you have the option to request access to an existing version or to add a new version. For information about requesting access to an existing version, see the [Request Access to an App](https://domo-support.domo.com/s/article/360042933274 "Requesting Access to an App") task. Otherwise, select **Add New** and proceed to the next step.

   </Warning>

<Note>
  **Note:** Anyone with a Participant default security role or anyone with a custom role that does not include the Use Appstore grant do not see the option to get the app. Instead, they see an option to **Request Demo**. Selecting this option displays a dialog in which you can reach out to your MajorDomo and request that the app be installed. For more information about default security roles, see [Managing Custom Roles](/s/article/360043438973).

</Note>

### Configure a Cardbuilder or Custom App

When you select **Connect** in the banner for a Cardbuilder or custom app, the **Find matches for your columns to connect** screen displays.

<Frame><img alt="Screenshot 2023-04-12 at 4.03.17 PM.png" src="/images/kb/ka05w000001284M-00N5w00000Ri7BU-0EM5w000006uUtS.jpg" style={{width: 1036.98, height: 452.995}}/></Frame>

1. In pane 1, select the DataSet you want to connect in the menu. The different columns in the DataSet display in that pane.
2. Drag the columns from pane 1 into the matching columns in pane
2. When a match is made, a green checkmark displays by the column. If there is no match, you cannot drag the DataSet into pane 2.

   <Frame><img alt="Screenshot 2023-04-12 at 4.10.27 PM.png" src="/images/kb/ka05w000001284M-00N5w00000Ri7BU-0EM5w000006uUth.jpg" style={{width: 1024.98, height: 451.997}}/></Frame>

3. After you match all of the columns in pane 2, select **Connect**. The cards in the DataSet now populate with live data from your DataSet.

You can update the columns and/or DataSet populating the app at any time by going to the dashboard, viewing the banner, and selecting **Change**.

## Request Access to an App

By default, when you install an app, only you, Admin-level users, and users with a custom security role that includes the Manage Appstore grant. If someone else wants access to an app that you installed, you can share access to the dashboard the same way you [share access to any dashboard](/s/article/360042932994). If you want access to an app that someone else installed, you can request access to the app dashboard from the Appstore.

For more information, see Controlling Access to Cards and Dashboards.

Follow these steps to request access to an app dashboard:

1. In the navigation header, select **Appstore **.****
2. Locate the app you want access to. You can view a list of apps in your instance by selecting
   <InlineImage src="/images/kb/ka05w000001284M-00N5w00000Ri7BU-0EM5w000006uUrl.jpg" /> **Installed Apps** in the sidebar and going to the **Your Installed Apps** and **Installed By Others** tabs.
3. When you locate the app, select it to see the Details view.
4. Select **Request Access**. A dialog displays with a request message that you can send as-is or customize.
5. (Optional) Customize the request message.
6. Select **Send Request**.

Your request is sent to the app owner via Buzz.

## Remove an Installed App

You can remove an app you have deployed to your Domo instance from the Appstore. This action removes instances of the app *only* from your own view of the instance. If other instances of the app have been installed in your organization's Domo instance by other people, these are not removed. When you remove an installed app, the version of the app in the Appstore is not affected.

<Note>
  **Note:** To remove an app, you must be the person who deployed the app or have an Admin default security role or a custom role with the Manage Appstore grant enabled. For more information about default security roles, see [Managing Custom Roles](/s/article/360043438973).
</Note>

1. In the navigation header, select **Appstore**.
2. In the sidebar, select
   <InlineImage src="/images/kb/ka05w000001284M-00N5w00000Ri7BU-0EM5w000006uUrl.jpg" /> **Installed Apps**.
     
   The **Installed Apps** page displays.
3. In the **Your Installed Apps** section, locate and select the app you want to remove.
     
   The app Details view displays.
4. Select **Uninstall**.
     
   A dialog displays warning that this action cannot be undone.

   <Frame><img alt="uninstall app.jpg" src="/images/kb/ka05w000001284M-00N5w00000Ri7BU-0EM5w000006uUrM.jpg" style={{width: 1051.99, height: 428.993}}/></Frame>

   <Frame><img alt="Screenshot 2023-04-12 at 11.26.05 AM.png" src="/images/kb/ka05w000001284M-00N5w00000Ri7BU-0EM5w000006uUrb.jpg" style={{width: 352.995, height: 261.997}}/></Frame>

5. Select **Uninstall** to remove the app or **Cancel** to go back.
