## New Features and Enhancements

### App Studio Editor Enhancements

Use App Studio to automatically generate charts and tables, using your data in just a few steps with no initial setup required.   

Build what you need using the following tools and options: 

*   Select your column, add calculations, adjust date ranges, and apply filters 
*   Choose from and build using four different table types: regular, editable, linked, and pivot tables 
*   Pick from dozens of chart options like bar, line, pie, heat map, world map, and scatter plots

![](/images/kb/app-studio-editor.png)
</Frame>



### Column PDP: Calculated Field Support

You can now use PDP-controlled columns in Beast Mode calculations and View calculated columns. When added, the masking policies are applied to the PDP-controlled columns, and the effects of column masking and the calculation formula are represented in the result.  

For example, I have a Last Name column that is masked to only show the first character, and the remaining characters are redacted. Then, I add a Beast mode calculation to the Last Name column that concatenates the First Name and Last Name columns. The Beast Mode calculation result appears as “George W*****”.  

<Frame>
![](/images/kb/col-pdp-calc-columns.jpg)
</Frame>

The Cloud Integration tables UI has been updated to include manual entry for database, schema, and table names, as well as the ability to connect tables in bulk. This update allows for improved communication of errors and statuses and table management for users with a large number of tables.

### Magic ETL Enhancements

#### "Add from Canvas" 

Magic ETL now includes the ability to add a tile directly to the canvas using a plus button that appears next to all non-terminal tiles on the canvas. This button opens a menu that enables you to search for the tile you want to add, or scroll through the list of tiles to choose from, with the most commonly used tiles listed at the top of the list. The feature improves the speed of DataFlow creation and overall workflow efficiency. 

<Frame>
![](/images/kb/add-from-canvas.png)
</Frame>



#### Power User Updates

We are releasing a suite of Magic ETL feature enhancements over the next few months that will improve the lives of technical Magic ETL users. The three new enhancements in this release include: 

*   The ability to configure a tile regardless of whether there are valid upstream tiles. 
*   The “Run to Here” functionality only runs the preview to the point where “Run to Here” is selected, allowing you to test streams of logic without needing to preview an entire DataFlow. 
*   An Inputs tab on each tile that lets you select the input to the tile without having to drag the line from the previous tile to it. 


<Frame>
![](/images/kb/power-user-updates.png)
</Frame>



#### Split Join and Split Filter

The new Split Join and Split Filter tiles provide more visibility into your data, easier troubleshooting, and pipeline development. 

**Split Join** — Manage multiple output streams from a single Join and easily identify rows that joined and those that did not. 

**Split Filter** — Capture rows that meet your filter criteria while viewing those that do not in a separate stream. 

Learn how to use the split join and split filter tiles in Magic ETL. 

<Frame>
![](/images/kb/split-join-config.png)
</Frame>



### Nav Configs

Nav Configs allow admins to create a custom navigation experience for their instance by adding Domo objects, including apps, a workspace home or folder, and dashboards, all to users' left navigation to access content.  

Admins can also create different configs for different groups or users and block which users can personalize their content. 

<Frame>
![](/images/kb/nav-configs.png)
</Frame>


### Pro Code Components: API Updates

The Pro Code components allow your custom apps to request access to browser peripheral APIs through Boolean properties in the manifest. These permissions enable advanced features like camera access for QA code scanning, geolocation for location-based analytics, and microphone for voice input. 

Available peripheral properties include: 

*   `allowGeolocation` 
*   `allowCamera` 
*   `allowMic` 
*   `allowBluetooth` 
*   `allowNFC`  

<Frame>
![](/images/kb/pro-code-apps.png)
</Frame>


### Workflows Enhancements

#### Annotations

The new annotations feature in Workflows lets you add comments directly to the workflow canvas. Annotations support rich text formatting and can be connected to one or multiple workflow steps.  

Key benefits of annotations include: 

*   Improved clarity
*   Enhanced collaboration
*   Faster onboarding
*   Better maintenance
*   Improved customer experience 

<Frame>
![](/images/kb/annotations-in-workflows.png)
</Frame>


#### Structured Outputs for AI Services

Structured outputs for the Text Generation, FileSets Image to Text, and Image to Text AI Services are now available in Workflows. With structured outputs, you can now define exactly which outputs you want right from the configuration panel in Workflows. 

Create a custom structured object with as many child properties as needed to capture precise values, mirroring the flexibility in Agent Tasks.  

<Frame>
![](/images/kb/structured-outputs-for-ai.png)
</Frame>



#### Updated Canvas Controls

We have made enhancements to the Workflow canvas to improve navigation and better align with other canvas-based tools in Domo. These enhancements aim to provide a more consistent and user-friendly workflow experience.  

Enhancements include: 

*   New canvas background 
*   Updated navigation control options for panning the canvas (with user preference settings) 
*   Better experience for adding and connecting steps 
*   Collapsible left panel

<Frame>
![](/images/kb/updated-canvas-controls.png)
</Frame>



#### Workflows Added as Tools to Agent Tasks

AI Agent Tasks can now use workflows as tools, allowing agents to leverage existing workflows alongside Code Engine functions or Jupyter scripts. Using workflows as tools allows the Agent task to gain access to a richer set of tools and extended functionalities, enabling the AI to process requests within a single workflow step. 

Benefits include: 

*   Increased flexibility
*   Enhanced decision-making 
*   Improved processing
*   Reusability

<Frame>
![](/images/kb/add-workflow-as-tool.png)
</Frame>

Worksheets include an accessible, drag-and-drop canvas for quick data analysis within Domo. It is closely integrated with Domo's data environment and supports seamless escalation when you're ready to move your worksheet to App Studio for sharing.

### Magic ETL Enhancements

#### Convert to SQL for Magic ETL Tiles

The Convert to SQL option in Magic ETL allows you to see the underlying SQL generated by the drag-and-drop actions in your DataFlows. You can also customize and edit the SQL queries in each tile to be used in external tools or for direct use in other environments.  

Being able to see the SQL code in your DataFlows allows you and your support teams to better diagnose problems, optimize queries, and improve overall performance.  

<Frame>
![](/images/kb/convert-to-sql.jpg)
</Frame>


#### Dark Mode

### Editable Table and Linked Tables

Dark mode is now available in the Magic ETL canvas. This new setting allows you to select a default mode for your canvas, including light, dark, or the option to inherit from the system setting, giving you control of how you view and use Magic ETL.   

<Frame>
![](/images/kb/dark-mode.png)
</Frame>



#### Edit Prior Versions

We have unlocked the ability to view an older version of a DataFlow without having to make the older version the current version. This feature allows you to open the older version and see all of the logic and configurations in it. You can also save the old version as a new version (making it the current version) or save it as a new DataFlow (creating a net new DataFlow outside of the current DataFlow’s context).  

<Frame>
![](/images/kb/load-from-other-dataflow-versions.png)
</Frame>
 


#### Personalized Default Settings

You can now persist user-level preferences across all your DataFlows in an instance. 
Preferences include: 

*   Grid Line variant
*   Appearance (light or dark mode) 
*   Enable Layout Upgrade (ability to configure panel placement)
*   Display "rows out" for preview executions 

<Frame>
![](/images/kb/persistent-user-level-settings.png)
</Frame>



#### Select Columns Update

The Select Columns tile has been re-designed to make it easier for you to use. 
Updates include: 

*   The ability to move single columns to the top, bottom, or anywhere in the list of selected columns 
*   The ability to move groups to columns to the top, bottom, or anywhere in the list of selected columns  
*   The ability to alphabetize contiguous selected columns 
*   The ability to search for specific columns  

<Frame>
![](/images/kb/select-columns-tile-enhancements.png)
</Frame>




#### Row Telemetry in Previews

Row counts for each action in a DataFlow are now available in Magic ETL. This update provides better visibility into how actions are impacting your data and makes it easier to troubleshoot and track data quality issues before creating the output DataSet.  

<Frame>
![](/images/kb/row-out-telemetry.png)
</Frame>



#### UI Customization

Panels in the new Magic ETL canvas interface have been converted into tools that can be moved to different parts of the screen. You can set layouts depending on how you like to work on different screens and in different scenarios; and save and persist your layouts across DataFlows. You can also choose your panel placement and select whether panels are pinned open or closed by default.  

<Frame>
![](/images/kb/customizable-window-layout.png)
</Frame>
