### Intro

The Activity log allows Domo administrators to see how their users are using Domo. They can see a near real-time record of activities such as user logins, dashboard views, groups created, and more.

Admins can view Activity Log entries in the Domo UI and export them to a CSV file or send them to a webhook. Activity Log data is also available in the [DomoStats](/s/article/360043433813) Activity Log report.

---

### Access Activity Log

<Warning>
  **Important:** You can only view the Activity log if you have an Admin system role or a custom role with the View Activity Logs grant enabled. Learn more about [roles and grants](/s/article/360043438973).

</Warning>

You can access the Activity log from the Admin Settings (**More** > **Admin**). Under **Governance**, select **Activity log**.

### Using the Log

The Activity log is updated in near real-time and includes activities logged within the rolling one-year period. You can explore the log with filters on the date range and the person who took the action. In addition, filters on type, event, and object can be applied to find specific activities. When you first load the Activity log, the data is filtered to a rolling 24-hour window.

<Note>
  **Refresh the log:** Selecting <img src="/images/kb/ka0Vq0000008Oht-00N5w00000Ri7BU-0EMVq000005bfFq.jpg" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/> **Refresh** reloads the log, displaying events logged since the page was last loaded.

</Note>

### DomoStats Report Available

<Frame><img alt="" src="/images/kb/ka0Vq0000000Rcv-00N5w00000Ri7BU-0EM5w000006vKuy.jpg" style={{width: 150, height: 150}}/></Frame>
For more extensive analysis of user activity, we recommend installing the DomoStats Activity Log report. This report loads a rolling 30-day window of activities the first time it executes; each subsequent execution of the report loads and appends only the activities logged since the last time the report was executed (up to the last 30 days' worth of activities).

The DomoStats Activity Log report allows building a long-running history of user activity in Domo. Learn more about the [DomoStats connector](/s/article/360043433813) and the [Activity Log report](https://domo-support.domo.com/s/article/360043433813?language=en_US#activity_log) in the Knowledge Base.

### Export to CSV

You can export the Activity log for your instance to a CSV file by selecting **Export CSV**.

<Frame><img alt="" src="/images/kb/ka0Vq0000008Oht-00N5w00000Ri7BU-0EMVq000005d0bZ.jpg" style={{width: 653, height: 399}}/></Frame>
The download process may take from a few seconds to several minutes, depending on the number of logs in your instance. When the download is successful, an **Export successful** message displays.

Then, select <img src="/images/kb/ka0Vq0000008Oht-00N5w00000Ri7BU-0EM5w000006uzIS.jpg" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/> **Download** to download the CSV file to your machine.

<Frame><img alt="" src="/images/kb/ka0Vq0000008Oht-00N5w00000Ri7BU-0EMVq000005bst0.jpg" style={{width: 445, height: 74}}/></Frame>
<Warning>
  **Important:** The log exports based on the filters you have configured at the time of export.

  For example, if your **Time** filter is set to **Past 24 hours**, the export file only contains data from the past 24 hours. If your export file does not contain all the data you expected, check your filter configuration and re-export if necessary.

</Warning>

### Send Events to a Webhook (Beta)

You can send log events to a webhook, sharing events with a third-party system and consolidating auditing across your enterprise.

<Note>
  **Note:** This feature is in beta and available on request. Contact your Domo account team to request enablement.

</Note>

You need the following items to configure this feature:

* The URL of a webhook that accepts an HTTP POST request containing a JSON body with the event details.
* A secret key. This is required for the configuration in Domo. If your webhook doesn't require a secret, you can enter any value in this field.
* The list of event types to send to the webhook.
* (Optional) An authorization header, if required by your webhook. For example, "Bearer ABC123DEF356."
* (Optional) An OAuth client ID/secret, if required by your webhook.

After obtaining these items, follow the configuration steps below.

1. [Access](#access_activity_log) the Activity log.
