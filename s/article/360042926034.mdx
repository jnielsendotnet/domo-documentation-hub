## Intro

The Approvals area of Domo allows you to you create request templates that anyone can use to submit requests. Requests can include items like a new budget proposal, a time-off request, or certification. Approvals are designed to allow you to incorporate all of your company’s unique approval processes. When creating a template, you determine the information that must be provided before submitting a request, and the designated individuals or groups review it.

<Frame><img alt="approvals.jpg" src="/images/kb/ka05w00000129wP-00N5w00000Ri7BU-0EM5w000006vSkE.jpg" style={{width: 335, height: 250}}/></Frame>
This article discusses how to configure a request template. For general information about the layout of the Approvals area, see [Submitting a Request for Approval](/s/article/360043430593 "Approval Center Layout").

Create Request Template

Follow these steps to create a request template:

1. In the navigation header, select **More** > **Approvals**. The **Approvals** area displays.
2. In the **Approvals** area, select **Manage templates**.

   <Frame><img alt="manage templates.jpg" src="/images/kb/ka05w00000129wP-00N5w00000Ri7BU-0EM5w000006vHyJ.jpg" style={{width: 653, height: 343.994}}/></Frame>
3. Select **+ Create Template**.

   <Frame><img alt="create template.jpg" src="/images/kb/ka05w00000129wP-00N5w00000Ri7BU-0EM5w000006vI8N.jpg" style={{width: 655, height: 323.988}}/></Frame>
   A disclaimer displays notifying you that all data collected through the template is captured in a DataSet and that you are responsible for complying with all applicable laws and regulations with respect to the collection, display, transfer, and use of such data and information.

   <Frame><img alt="Screenshot 2023-07-31 at 4.03.52 PM.png" src="/images/kb/ka05w00000129wP-00N5w00000Ri7BU-0EM5w000006vSl7.jpg" style={{width: 379, height: 247}}/></Frame>
4. Enter a unique title for your template and an optional icon.

   <Columns>
     <Frame><img alt="enter name.jpg" src="/images/kb/ka05w00000129wP-00N5w00000Ri7BU-0EM5w000006vSlR.jpg" style={{width: 655, height: 323.988}}/></Frame>
     <Frame><img alt="enter template icon.png" src="/images/kb/ka05w00000129wP-00N5w00000Ri7BU-0EM5w000006vSlc.jpg" style={{width: 655, height: 356}}/></Frame>
   </Columns>
5. (Optional) If you want to include specific instructions for users filling out this form, enter those instructions in the **Instructions** field.

   <Frame><img alt="instructions.jpg" src="/images/kb/ka05w00000129wP-00N5w00000Ri7BU-0EM5w000006vSll.jpg" style={{width: 655, height: 323.988}}/></Frame>
6. Select a tag from the **Template tag** menu.

   <Frame><img alt="assign tag.jpg" src="/images/kb/ka05w00000129wP-00N5w00000Ri7BU-0EM5w000006vSds.jpg" style={{width: 655, height: 356}}/></Frame>
7. Select **+** **Add Field** and create a field that users must fill out when submitting requests with this template. There are different types of answers available and you can decide whether an answer is required. Learn about the answer types below. You can add as many fields as you need.

   <Frame><img alt="add field.jpg" src="/images/kb/ka05w00000129wP-00N5w00000Ri7BU-0EM5w000006vSmF.jpg" style={{width: 656, height: 92}}/></Frame>
   <Note>
     **Note:** The information that a user provides in the first field is used as the title of the request.

   </Note>

8. For each additional field you want to include in this template, do the following:

   1. In the **Type** menu, select the appropriate field type for the desired response.
        
      The following table lists and describes all of the available field types:

      | Field Type   | Description                                                                                                                                                                                                                | Example                                                                                                    |
| ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| Short Answer | Allows users to enter a single line of text as a response. Also appropriate for times (as we currently do not have a field specific to times).                                                                             | <img alt="request_short_answer.png" src="/images/kb/ka05w00000129wK-00N5w00000Ri7BU-0EM5w000005wLrW.png"/> |
| Paragraph    | Allows users to enter a paragraph (multiple lines of text) as a response. Also appropriate for lists.                                                                                                                      | <img alt="request_paragraph.png" src="/images/kb/ka05w00000129wK-00N5w00000Ri7BU-0EM5w000005wLrv.png"/>    |

| Currency     | Allows users to enter currency values as a response as well as select from four different currency symbols. Note that users do not need to include currency symbols in their responses, as they are applied automatically. | <img alt="request_currency.png" src="/images/kb/ka05w00000129wK-00N5w00000Ri7BU-0EM5w000005wLre.png"/>     |

   2. (Optional) To move a field up or down in the order of fields in the request template, drag it by the handle
      <img src="/images/kb/ka05w00000129wP-00N5w00000Ri7BU-0EM5w000005vNlq.png" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/> to where you want it. You can also delete a field by selecting <img src="/images/kb/ka05w00000129wP-00N5w00000Ri7BU-0EM5w000006vSrA.jpg" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/> **Delete**.
9. Under **Approval Chain**, select the individuals or groups that need to review the requests submitted using this template. When the request is submitted, it is sent to the first person listed in the chain. When that person gives their approval, it is sent to the next. When all members of the chain have approved the request, the submitter is notified.

   1. (Conditional) If the request needs to be approved by more than one user, select **Add Approver Step** and select the new approving user.
   2. (Optional) If you do not want individuals modifying the approval chain when using this template, use the switch labeled **Lock Chain**. When a template has a locked chain, it means that the user submitting a request with this template cannot add/remove approvers or modify the order.
   3. (Optional) If desired, change the approver order by selecting the handle
      <img src="/images/kb/ka05w00000129wP-00N5w00000Ri7BU-0EM5w000005vNlq.png" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/> for a given approver and dragging the name up or down in the chain.
10. (Optional) Add observers to this request template. Observers have access to review the request and add comments, but cannot act on the request, such as approving or denying. When adding an observer to a template, the observer has access to every request submitted with this template.
11. When you are satisfied with the layout of your template, select **Save**.
