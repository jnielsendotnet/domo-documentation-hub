## Intro

Microsoft SharePoint Online enables an organization to easily create and manage custom team-focused and project-focused sites for collaboration. You can use Domo's Microsoft SharePoint Online REST Connector to get data from an organization-wide intranet portal on different aspects based on sites within Domo. To do this, you must first create an app in Microsoft Azure. For more information about Microsoft SharePoint Online, visit their website at [https://products.office.com/en-us/sharepoint/sharepoint-online-collaboration-software](https://products.office.com/en-us/sharepoint/sharepoint-online-collaboration-software "https://products.office.com/en-us/sharepoint/sharepoint-online-collaboration-software").

You create a Microsoft SharePoint Online connection in the Data Center. This topic discusses the fields and menus that are specific to the Microsoft SharePoint Online REST connector user interface. General information for adding DataSets, setting update schedules, and editing DataSet information is discussed in [Adding a DataSet Using a Connector](https://domo-support.domo.com/s/article/360042926274?language=en_US "Adding a DataSet Using a Connector").

## Prerequisites

To connect to Microsoft SharePoint Online in order to retrieve information, you must have the following:

* An installed Microsoft Azure app.
* The CLIENT ID and CLIENT SECRET. To generate an app and locate these credentials, see
  [Microsoft's documentation](https://docs.microsoft.com/en-us/azure/active-directory/develop/quickstart-register-app "Microsoft's documentation"). You may need the following information to register for the app:

  + **Azure Portal:** [https://portal.azure.com/](https://portal.azure.com/ "https://portal.azure.com/")
  + **Sharepoint App to Register:** SharePoint Online Rest API
  + **Domo Re-direct URL:** [https://oauth.domo.com/api/data/v1/oauth/providers/microsoft-sharepoint/exchange](https://oauth.domo.com/api/data/v1/oauth/providers/microsoft-sharepoint/exchange "https://oauth.domo.com/api/data/v1/oauth/providers/microsoft-sharepoint/exchange")

### Creating an Azure App for Enterprise OAuth

To create a Microsoft Azure app, follow the below steps, or see our [YouTube video](https://www.youtube.com/watch?v=mZOtUI-5IOQ "How to Set Up the Azure App Registration").

1. Log into [https://portal.azure.com](https://portal.azure.com/ "https://portal.azure.com").
2. Click on **App registrations**.
     
   <Frame><img alt="Capture1.PNG" src="/images/kb/ka05w000001236O-00N5w00000Ri7BU-0EM5w000005vNz7.png"/></Frame>

3. Click **New registration**.
4. Enter the application name and select the supported account type. Enter the Redirect URI for Web as
   [https://oauth.domo.com/api/data/v1/oauth/providers/microsoft-sharepoint/exchange](https://oauth.domo.com/api/data/v1/oauth/providers/microsoft-sharepoint/exchange "https://oauth.domo.com/api/data/v1/oauth/providers/microsoft-sharepoint/exchange").
5. Change the default tenant selection from single tenant to multi tenant. This is a requirement for the connector.
   
   <Frame><img alt="REgister.png" src="/images/kb/ka0Vq0000006d0H-00N5w00000Ri7BU-0EM5w000006vIDX.jpg" style={{width: 678, height: 523}}/></Frame>

6. Click **Register**.
7. Now, click **API Permissions**.
   
   <Frame><img alt="API_permission.png" src="/images/kb/ka0Vq0000006d0H-00N5w00000Ri7BU-0EM5w000005vO6o.png"/></Frame>

8. Click **+Add Permissions** button.
   
   <Frame><img alt="Capture8.PNG" src="/images/kb/ka05w00000123LH-00N5w00000Ri7BU-0EM5w000005vO61.png"/></Frame>

9. Select **Microsoft Graph > Delegated Permissions**.
10. Select the permissions as required.
11. After adding the permissions, all configured permissions will be listed as below.
      
   <Frame><img alt="Configure.png" src="/images/kb/ka0Vq0000006d0H-00N5w00000Ri7BU-0EM5w000006vIDc.jpg" style={{width: 899, height: 383}}/></Frame>

### To obtain the Sharepoint CLIENT ID:

1. Go to [https://portal.azure.com](https://portal.azure.com/ "https://portal.azure.com").
2. Select Microsoft Entra ID > App registrations.
3. Find and click on your web app.
4. The "application" ID is your CLIENT ID.

### To obtain the Sharepoint CLIENT SECRET:

1. Go to [https://portal.azure.com](https://portal.azure.com/ "https://portal.azure.com").
2. Select Microsoft Entra ID > App registrations.
3. Find and click on your web app.
4. Open **Certificates and secrets**.
5. Click **+New Value**.
6. Provide the value description and the expiry period for your secret. Click **Add**.
7. Your new value will appear. Copy the Value field and paste it in your Domo connector Value field.
   
   <Frame><img alt="Secret.png" src="/images/kb/ka0Vq000000A67N-00N5w00000Ri7BU-0EM5w000005vO71.png"/></Frame>

<Accordion title="Where can I find the client id and client secret?">

You need an installed Microsoft Azure app to get the client id and client secret.

**To obtain the client id:**

1. Visit the [Microsoft Azure Portal](https://portal.azure.com/ "Microsoft Azure Portal") and sign in using your Microsoft credentials.  
2. Select **Azure Active Directory > App registrations**.  
3. Find your app and click on it.  
4. Locate the "Application ID" from the web app. The Application ID is the client id.  

**To obtain the client secret:**

The client secret is generated when you create your Azure app. To generate a new client secret:  

1. Visit the [Microsoft Azure Portal](https://portal.azure.com/ "Microsoft Azure Portal") and sign in using your Microsoft credentials.  
2. Select **Azure Active Directory > App registrations**.  
3. Find your app and click on it.  
4. Open **Settings > Keys**.  
5. Create the client secret.

## Troubleshooting

* Make sure your authentication remains valid.
* Review the configuration to make sure that all required items have been selected.
* Review the Connector history for error messages.
* In rare cases, you may be requesting too much information and reaching API limitations or timeouts. If this is the case, you can review the history of the Connector run to see the error message and duration. If this is the case, you can reduce the number of accounts that are being pulled, choose a smaller number of metrics for the report that you are pulling, or reduce the timeframe that you are trying to pull.
