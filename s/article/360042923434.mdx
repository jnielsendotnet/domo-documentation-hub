## Intro

SAP Concur is a business expense and travel software platform. To learn more about SAP Concur's API, visit their page [https://developer.concur.com/api-reference/](https://developer.concur.com/api-reference/ "https://developer.concur.com/api-reference/").

You can connect to your SAP Concur account in the Data Center. This topic discusses the fields and menus that are specific to the SAP Concur connector user interface. To add DataSets, set update schedules, and edit DataSet information, see [Adding a DataSet Using a Data Connector](/s/article/360042926274 "Adding a DataSet Using a Data Connector").

## Prerequisites

To connect to your SAP Concur account and create a DataSet, you must have the following:

* Client ID
* Client Secret
* The username and password used to log into your SAP Concur account

To obtain your Client ID and Secret, contact your SAP Concur Partner Enablement Manager or Partner Account Manager. Once you have registered an application, you will receive a Client ID and Client Secret. For more information on getting started with SAP Concur's API, see [https://developer.concur.com/api-reference/authentication/getting-started.html](https://developer.concur.com/api-reference/authentication/getting-started.html "https://developer.concur.com/api-reference/authentication/getting-started.html")

## FAQ

<AccordionGroup>
<Accordion title="What credentials do I need to power up this connector?">

You need the username or email ID associated with your SAP Concur account.

</Accordion>

<Accordion title="How do I know that my SAP Concur credentials are secure?">

The login process uses the OAuth process, so your SAP Concur credentials are never seen or stored by Domo. This keeps your login secure. You can revoke Domo's access to your account at any time.

</Accordion>

<Accordion title="How frequently can the data be updated?">

As often as needed.

</Accordion>

<Accordion title="How does my user role affect the data returned in the SAP Concur connector?">

You can receive the company-wide expense reports only if your user has the 'Web Services Admin role'. Otherwise, you can only see items for your user.

</Accordion>

<Accordion title="Can I use the same account for generating different reports?">

Yes.

</Accordion>
</AccordionGroup>
