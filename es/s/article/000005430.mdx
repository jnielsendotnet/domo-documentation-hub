### Domo.AI

**Una nota sobre seguridad:** Domo reconoce que la IA debe regirse según las directrices internas de nuestros clientes.Los asistentes de IA envían a Domo metadatos sobre las fuentes de datos de Domo para OpenAI.

Por lo tanto, para utilizar asistentes de IA en Domo, los administradores deberán habilitar los modelos de OpenAI para utilizarlos en Configuración del administrador para la capa de servicio de IA.Además, los usuarios que no sean administradores de Domo necesitan un rol que tenga habilitado el permiso de "Utilizar servicios de IA" para poder aprovechar los asistentes de IA.

En los próximos meses, Domo presentará una opción de LLM interna para ejecutar asistentes de IA que se puedan utilizar sin tener que enviar información a OpenAI.

# Asistente de IA Beast Mode

El asistente de IA Beast Mode de Domo.IA puede ayudar a generar cálculos de Beast Mode dentro del editor de Beast Mode. Para utilizar el asistente de IA Beast Mode, debe tener habilitado el permiso de "Utilizar servicios de IA" para su rol.

Estos son algunos ejemplos de indicaciones que puede probar:

* "Devolver la diferencia entre la Fecha del pedido y la Fecha de envío en meses"
* "Crear un campo calculado donde si la prioridad del pedido es Crítica o Alta pasa a ser Importante, de lo contrario no es importante"
* "Agregar una semana a la columna Datos del pedido"
* "Calcular la suma acumulada de ventas totales por fecha de pedido"

Consulte [Asistente de IA Beast Mode](/s/article/000005304) para obtener más información.

Para saber más sobre los permisos, consulte [Administrar los roles personalizados](/s/article/360043438973).

Obtenga más información sobre [Domo.AI](https://ai.domo.com/) o explore la [documentación de Domo.AI](https://domo-support.domo.com/s/topic/0TO5w000000ZmGPGA0/domoai?language=en_US) en la Base de conocimientos.

# Editor SQL avanzado de vistas de DataSet con asistente de IA

El editor SQL avanzado del explorador de vistas le permite ver el SQL que alimenta cualquier vista de un DataSet. Puede editar el SQL, aplicarlo y ver cómo se actualiza su consulta.

Además, puede utilizar el asistente de IA para crear una consulta SQL sobre los datos que quiera incluir en su vista de DataSet. Escriba alguna indicación con un lenguaje sencillo y el asistente de IA devolverá una consulta SQL.

Estos son algunos ejemplos de indicaciones que puede probar:

* "Mostrarme todas las filas donde la prioridad del pedido es alta o crucial y el TCV es mayor que 10 000"
* "Limpiar mi columna Fecha de actualización para que esté en formato AAAA-MM-DD"
* "Crear una nueva columna calculada denominada Ganancia que tome las Ventas totales menos el Costo total"
* "Unirse al DataSet de detalles del cliente en el dominio del cliente e incluir todas las columnas de ambos DataSets"

Obtenga más información sobre el [Editor SQL avanzado con asistente de IA](https://domo-support.domo.com/s/article/360046074774?language=en_US#sql_editor_and_text-to-sql).

Obtenga más información sobre [Domo.AI](https://ai.domo.com/) o explore la [documentación de Domo.AI](https://domo-support.domo.com/s/topic/0TO5w000000ZmGPGA0/domoai?language=en_US) en la Base de conocimientos.

### Flujos de trabajo

Los flujos de trabajo aprovechan poderosamente otras características, incluidos el [Motor de códigos](#code_engine), las [Colas](#queues) y los [Formularios](#forms) para permitirle diseñar e implementar procesos de flujo de trabajo ejecutables con capacidades de código bajo. Puede iniciar un flujo de trabajo manualmente, a través de una Alerta, a través de un flujo de aprobación o con un temporizador. Utilice nuestra biblioteca de plantillas de flujos de trabajo para empezar rápidamente.

Para obtener más información, consulte [Descripción general de los flujos de trabajo](https://domo-support.domo.com/s/article/000005108?language=en_US) en la Base de conocimientos.

### Motor de códigos

El Motor de códigos es un entorno de tiempo de ejecución nativo de Domo que puede ejecutar código JavaScript o Python e interactuar con cualquier característica dentro o fuera de Domo. Puede utilizar la biblioteca de paquetes y funciones de Domo para integraciones y servicios comunes, o puede crear sus propias funciones para usarlas en su instancia de Domo. Las funciones del Motor de códigos se integran en los flujos de trabajo sin codificación.

Para obtener más información, consulte [Motor de códigos](https://domo-support.domo.com/s/article/000005173).

### Formularios

El nuevo editor de formularios le permite crear y publicar formularios para interacciones manuales personalizadas. Los formularios se integran a la perfección en los flujos de trabajo, de modo que puede crear y gestionar formularios reutilizables como parte de las tareas de usuario de un flujo de trabajo o enviar un formulario para iniciar un flujo de trabajo.

Para obtener más información, consulte [Formularios](https://domo-support.domo.com/s/article/000005171).

### Colas

Las colas le permiten organizar tareas manuales en diferentes colas y controlar quién puede acceder a ellas. Puede configurar una cola para mostrar tareas para una persona o un grupo. Cada tarea está asociada a un formulario creado por un flujo de trabajo. Puede filtrar las colas para ver las tareas por cesionario o estado, monitorear el volumen de la cola y personalizar los diseños.

Para obtener más información, consulte [Colas](https://domo-support.domo.com/s/article/000005172).

### Conectores JSON sin código y JSON sin código OAuth

Los conectores JSON sin código y JSON sin código OAuth le permiten conectar Domo a API REST que devuelven datos en un formato estándar de JSON. Ambos conectores admiten métodos de autorización estándar como autenticación básica, clave API, token portador y OAuth 2.0. Cada conector incluye tres áreas de configuración principales: Tipo de conexión, Tipo de solicitud y Configuración avanzada opcional.<br/>
Ambos conectores están disponibles para descarga en la [AppStore de Domo](https://www.domo.com/appstore/) [<InlineImage src="/images/kb/ka0Vq00000001FR-00N5w00000Ri7BU-0EMVq0000000XDh.jpg" />.](https://www.domo.com/appstore/)

Para obtener más información, consulte [Conector JSON sin código](/s/article/360043433473) y [Conector JSON sin código OAuth](/s/article/360052105454).

### Vista de gráfico en la página de detalles de Magic – Extracción, transformación y carga

La nueva vista de gráfico de Magic – Extracción, transformación y carga, así como el Resumen de la acción, le permiten ver exactamente dónde falló un DataFlow y revisar la información del error de la forma más eficiente. Al seleccionar un icono en el lienzo de vista previa, se señala la fila correspondiente en el Resumen de la acción, por lo que podrá solucionar los problemas de una forma más efectiva.

Esta función solo está disponible para DataFlows de Magic – Extracción, transformación y carga.

Obtenga más información sobre [Magic – Extracción, transformación y carga](https://domo-support.domo.com/s/topic/0TO5w000000ZanvGAC/magic-etl?language=en_US) en la Base de conocimientos.

### Diagramas

El nuevo diagrama siguiente y las mejoras de los diagramas ya están disponibles en Analyzer.

# Indicador de entre o en rango

El indicador de Entre y En rango le ayudan a ver los resultados que están entre los límites inferiores y superiores, mostrando si el resultado está dentro del rango objetivo, por lo que podrá identificar rápidamente excepciones a los valores esperados. Este diagrama está disponible en la sección de Indicadores de Analyzer.
  
El tamaño de la fuente, la anchura de las barras y los colores son valores personalizables.

# Etiquetas de diagrama de rosquilla

Para hacer que nuestro diagrama de rosquilla fuera más fácil de usar en las presentaciones, hemos añadido etiquetas de datos y texto emergente para realizar análisis interactivos. Las etiquetas de datos le permiten mostrar información muy útil con sus diagramas de rosquilla, incluso con un formato estático.

Para añadir etiquetas de datos a sus diagramas de rosquilla, vaya a la configuración de las Etiquetas de datos en Propiedades del diagrama de Analyzer.

# Bordes de tabla enriquecida

Las tablas mega y las tablas dinámicas ahora permiten mayor personalización de los bordes, incluyendo las opciones siguientes:

* Grosor y color del borde vertical
* Grosor y color del borde horizontal
* Bordes exteriores
* Bordes de los encabezados

Estos cambios no afectan a las tablas existentes. Las nuevas opciones son compatibles con la exportación a Excel.

# Mejoras de Domo.AI

Los conectores Amazon Bedrock ML y Databricks ML le permiten utilizar sus modelos de datos externos con la solución [Domo.AI](http://Domo.AI). Los conectores requieren diferentes claves del proveedor original para poder configurar su cuenta en Domo, lo que puede hacerse desde Configuración del administrador, en Domo. Combine sus modelos existentes con la potencia de Domo para visualizar sus datos, crear proyectos de IA y mucho más.

Para unirse a la fase beta de estos conectores, póngase en contacto con [ai@domo.com](mailto:ai@domo.com).

# App Studio

Ya está disponible la beta abierta de App Studio

Gracias a sus temas prediseñados y a la infinidad de maneras de personalizar el aspecto y el uso de sus aplicaciones, App Studio es una experiencia de nivel superior para la creación de tableros que le aporta el nivel de precisión y navegación que antes requería el marco de desarrollo de aplicaciones. App Studio está lista para convertirse en el destino definitivo para la presentación de datos y las narrativas convincentes, todo dentro de una única aplicación.
