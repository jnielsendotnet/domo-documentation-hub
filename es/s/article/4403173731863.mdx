## Nuevas características y mejoras

Las características y mejoras de esta versión incluyen lo siguiente:

### Cree su propio conector de reescritura

Con el IDE de conector, cree un conector de reescritura que enviará datos desde Domo hasta sus sistemas externos. Los conectores de reescritura aportan la eficacia para compartir datos entre sus sistemas empresariales que actualmente no se comunican entre sí. Además, ahora, si la biblioteca de Domo todavía no incluye lo que necesita, tiene a su disposición la biblioteca ampliada para realizar sus propias creaciones. Para obtener más información, visite [https://developer.domo.com/docs/writeback-connectors/build](https://developer.domo.com/docs/writeback-connectors/build "https://developer.domo.com/docs/writeback-connectors/build").

### Página principal de ciencia de datos

En la página principal de ciencia de datos se muestran de forma centralizada todas las herramientas eficaces de ciencia de datos, se proporciona acceso rápido a tales herramientas, se describen los productos y servicios, y es el lugar donde las herramientas se adaptan a la trayectoria de madurez de la ciencia de datos. Para obtener más información, vea Página principal de ciencia de datos.

### Mejoras de los SDK de Python y R

Los SDK de R y Python se han actualizado para usar la misma nomenclatura de las funciones para ambos lenguajes y para que ambos tengan las mismas funciones disponibles. Para obtener más información, visite [https://developer.domo.com/docs/domo-sdks/sdks](https://developer.domo.com/docs/domo-sdks/sdks "https://developer.domo.com/docs/domo-sdks/sdks").

### Vistas de filtro

Las vistas de filtro permiten personalizar un solo tablero para satisfacer las necesidades de todas las audiencias.

Con las vistas de filtro, puede hacer lo siguiente:

* Guardar vistas personalizadas que solo usted puede ver.
* Seleccionar y compartir vistas importantes para crear coherencia con una perspectiva nueva.
* Proporcionar cualquier número de historias de datos personalizadas para todas las personas con cualquier rol.
* Designar qué vista de filtro guardada es la "predeterminada" para un tablero.

Para obtener más información, consulte [Aplicación de vistas de filtro y filtros a nivel de página](/s/article/360042923914 "Aplicación de filtros a nivel de página (con vistas de filtro) (BETA)").

### Configuración de los filtros de página

Con la configuración de los filtros de página, ahora puede personalizar cómo sus usuarios interactúan con las tarjetas en su tablero. Puede desactivar **Permitir la adición de filtros nuevos** para que los usuarios solo puedan usar las interacciones con las tarjetas para filtrar. También puede desactivar los filtros de fecha si el tablero está creado para buscar en un período de tiempo específico.Para obtener más información, consulte [Aplicación de vistas de filtro y filtros a nivel de página](/s/article/360042923914 "Aplicación de filtros a nivel de página (con vistas de filtro) (BETA)").

### Opciones de seguridad de los informes programados

Con las nuevas opciones de seguridad de los informes programados, los administradores ahora pueden habilitar la opción **Limitar a usuarios de Domo**. Cuando se configura este ajuste, Domo bloquea los informes programados para cualquier dirección de correo electrónico externa; solo las direcciones de correo electrónico vinculadas a una cuenta existente recibirán informes. Para obtener más información, consulte [Programación de informes](/s/article/360043437773 "Programación de informes").

### Tableros de anchura automática

Con la opción Tablero de anchura automática, el contenido del tablero se ampliará para adaptarse a todo el espacio de la ventana disponible. Esto permitirá que se vean más filas y columnas en la vista del tablero. Esto se habilita en cada tablero para que pueda personalizar sus historias. Para obtener más información, consulte [Creación de historias de Domo](/s/article/360043428433 "Creación de historias de Domo").

### Nuevos tipos de diagramas de importación personalizados

Con los nuevos tipos de diagramas de importación personalizados, ahora puede crear diagramas personalizados a partir de muchos otros archivos, aparte de los archivos SVG.

Ahora, se admiten los formatos siguientes:

* **KML (lenguaje de marcado de keyhole):** formato de archivo utilizado para mostrar los datos geográficos en un navegador terrestre, como Google Earth.
* **GeoJSON:** formato estándar de código abierto diseñado para representar características geográficas sencillas, junto con sus atributos no espaciales.
* **TopoJSON:** extensión de GeoJSON que codifica la topología geoespacial y que suele ofrecer tamaños de archivo más pequeños.
* **Forma:** formato de archivo estándar ESRI. Incluye datos vectoriales geoespaciales (se importan en Google Earth y se exportan como KML).

Para obtener más información, consulte [Personalizar diagramas](/s/article/360042924454 "Personalizar diagramas").

### Colores divergentes

Los colores divergentes facilitan la creación y visualización rápida de distribuciones de rangos mediante la combinación de varios temas de color. Los colores divergentes le permiten seleccionar cualquier color de inicio y finalización. Además, puede seleccionar el número de rangos que desea usar. Los colores divergentes también le permiten controlar su punto medio. Puede seleccionar el color y el valor del punto medio o no incluir el punto medio. Por ejemplo, puede usar colores divergentes para facilitar la visualización de las tendencias activas y pasivas de los datos mediante un tema de color rojo combinado con un tema verde para mostrar un rango más completo de valores. Los colores divergentes aportan esta flexibilidad a las tarjetas Mapa y Mapa de calor. Para obtener más información, consulte [Mapa de calor](/s/article/360042924634 "Mapa de calor").

### Diagrama de Gantt de dependencias

El diagrama de Gantt de dependencias gestiona los cambios de fechas automáticamente y ofrece muchas opciones nuevas de visualización para comunicar los plazos de los proyectos, como:

* Calcular las fechas de finalización a partir de los datos sobre las dependencias y la duración.
* Mostrar las relaciones entre los datos de las dependencias.
* Visualizar los hitos.
* Mostrar los proyectos y subproyectos en una jerarquía.
* Permitir el seguimiento y la representación de los recursos en el diagrama.
* Cálculos de fechas inteligentes: contabilizar automáticamente los fines de semana o mostrarlos.

Para obtener más información, consulte [Diagrama de Gantt de dependencias](/s/article/1500003419382).

### Estado de Buzz

La característica Estado de Buzz le permite definir su estado en la Web o en el móvil como **Disponible**, **Ocupado**, **Ausente** o **Fuera de la oficina**. Cuando alguien le envíe un mensaje directo o lo @mencione en una conversación mientras su estado está definido como cualquier otra opción que no sea Disponible, esta persona recibirá un mensaje de advertencia para indicarle que usted no está disponible. También puede añadir mensajes personalizados a su mensaje de estado cuando no esté disponible. Un icono en su avatar indica su estado actual. Para obtener más información, consulte [Diseño de Buzz](/s/article/360043429973 "Diseño de Buzz").

### DataFlow de Adrenaline

Un DataFlow de Adrenaline es una herramienta de transformación de datos de alto rendimiento, a gran escala, interna y con tecnología de Adrenaline para DataSets masivos en Domo. Esto le permite impulsar las inversiones y las mejoras de rendimiento del motor de Adrenaline. Los DataFlows de Adrenaline son una interfaz de transformación basada en scripts SQL (similares a los DataFlows de MySQL y RedShift). Para obtener más información, consulte [Crear un DataFlow de Adrenaline](/s/article/360063698733 "Crear un DataFlow de Adrenaline").

### AutoML (aprendizaje automático)

AutoML de Domo le permite entrenar docenas de modelos de ML con tan solo unos clics, así como implementar en producción los modelos más eficaces. Vaya de los datos a los modelos para obtener resultados con más rapidez.

Domo facilita lo siguiente:

* Entrenar y adaptar automáticamente docenas de modelos de ML.
* Iniciar cientos de trabajos de entrenamiento en cualquier DataSet de Domo para encontrar el modelo que consiga el máximo rendimiento de la tarea.
* Implementar fácilmente el modelo en el DataSet con el nuevo icono de AutoML en Magic – Extracción, transformación y carga.
* Todo lo que se necesita para que AutoML sea eficaz es un problema bien definido y un DataSet limpio; y Domo se encargará del resto.

Para obtener más información, consulte [AutoML (aprendizaje automático)](/s/article/360048127854 "AutoML (aprendizaje automático)").

### Integration Studio

Integration Studio le permite vincular sistemas empresariales distintos que, de otra forma, no podrían compartir datos. Transforme datos o añada ciencia de datos e inteligencia artificial antes de compartir con un sistema empresarial distinto con el icono Reescritura de Magic – Extracción, transformación y carga. Para obtener más información, consulte [Integration Studio](/s/article/360045163773 "Integration Studio (Beta)").

### SMS de la aplicación Campañas

El complemento SMS de la aplicación Campañas le permite enviar mensajes SMS mediante programación desde Domo. Le permite ponerse en contacto automáticamente con quienes no son usuarios de Domo por diferentes vías en relación con elementos como: recordatorios de citas con los clientes, avisos de cambio de turnos del personal y el contratista, recordatorios de facturas impagadas, actualizaciones de envíos, alertas de reposición de productos, etc.

Use el complemento SMS de la aplicación Campañas para lo siguiente:

* Consultar los números de teléfonos móviles de un DataSet de Domo para rellenar la lista de destinatarios.
* Redactar el contenido del cuerpo del mensaje SMS manualmente o rellenarlo mediante programación a partir de los valores del DataSet de Domo.
* Enviar el mensaje SMS, programar su tiempo de ejecución o activarlo con Alertas de DataSet.

Para obtener más información, consulte [Aplicación Campañas de Domo](/s/article/360042933494 "Aplicación Campañas de Domo").
