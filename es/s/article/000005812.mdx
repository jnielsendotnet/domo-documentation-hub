# Actualizaciones del registro de actividad

Hemos convertido el registro de actividad a React. Con esta actualización se implementan las siguientes mejoras:

* Filtros de evento más intuitivos con Evento y Tipo de evento desglosados en diferentes filtros de menús desplegables.
* Los filtros de Fecha y Persona ahora son parte del encabezado de columna en lugar de estar separados en distintos elementos de filtro de menús desplegables.
* La experiencia de usuario es más compacta y el espacio está más aprovechado.

Vea el [registro de actividad](/s/article/360042934574) en la base de conocimiento.

# Eliminación de la vista previa automática en Analyzer para los DataSets de Cloud Amplifier

Ahora la vista previa automática para los DataSets de Cloud Amplifier está deshabilitada por defecto al abrir Analyzer. Esta actualización reduce el número de consultas que se producen y reduce los costes. Todavía puede activar la vista previa automática en el menú de Analyzer.

# Flujos de trabajo: inicializar variables

Ahora puede asignar un valor inicial a una variable durante la experiencia de creación de la versión en los flujos de trabajo. Esta característica elimina la necesidad de usar parámetros de inicio o funciones del motor de código para rellenar variables que requieren el mismo valor con cada tiempo de inicio.

Otros beneficios son los siguientes:

* **Diseño del flujo de trabajo simplificado:** Asigne valores iniciales a variables directamente durante el proceso de creación.
* **Consistencia:** Asegura que las variables siempre contengan el mismo valor inicial cuando un flujo de trabajo se inicia.
* **Eficiencia mejorada:** Reduce los pasos innecesarios del flujo de trabajo.
* **Experiencia de usuario mejorada:** Configure un valor inicial en tan solo unos clics.

<Frame><img alt="Screenshot 2025-01-15 at 8.09.52 AM.png" src="/images/kb/ka0Vq0000009NGj-00N5w00000Ri7BU-0EMVq000004pCmQ.jpg" style={{width: 896, height: 393}}/></Frame>
Aprenda a [añadir una variable a un flujo de trabajo](https://domo-support.domo.com/s/article/000005331?language=en_US#add_a_variable) y asignarle un valor inicial.
