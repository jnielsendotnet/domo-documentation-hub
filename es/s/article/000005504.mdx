# Diagrama alto-bajo horizontal

Con el diagrama alto-bajo horizontal añadido, como compañero del diagrama alto-bajo vertical, puede mostrar listas más largas y proporcionar más detalles sobre métricas con un aumento de volumen.

<Frame>![Screenshot 2024-04-25 at 11.05.20 AM.png](/images/kb/es/0EMVq000000pNXF.jpg)</Frame>

Tanto el tipo de diagrama alto-bajo horizontal como el vertical en Analyzer incluyen las siguientes opciones de estilo:

- Predeterminado
- Línea
- Barra
- Barra redondeada
- Círculo

La configuración del estilo de barra incluye la opción de ajustar el porcentaje de la altura de las barras del diagrama. También puede usar las columnas de su DataSet para establecer los valores alto y bajo de su diagrama.

Para saber más sobre los diagramas alto-bajo, consulte [Diagrama alto-bajo.](/es/s/article/360043429173)

### Mejoras de la plataforma

# Administración de la puerta de enlace de la API

Ahora los administradores pueden crear y gestionar clientes de la puerta de enlace de la API en Domo, y los registros de actividad incluyen más detalles sobre las acciones de los clientes.

La nueva asignación de Crear clientes de API OAuth controla la habilidad de crear clientes de la puerta de enlace de la API. Los administradores con esta asignación pueden deshabilitar la asignación para otros usuarios en su instancia de Domo.

Además de estas características, los administradores también pueden gestionar de manera programática los clientes de API; estas opciones incluyen:

- Aprovechar y controlar las asignaciones de funciones a usuarios. Para saber más sobre cómo crear clientes, visite el [portal para desarrolladores.](http://developer.domo.com/)

- Añada la fecha y la hora de la creación del cliente.

<Frame>![apiGateway Create New Client Portal.png](/images/kb/es/0EMVq000000rpHJ.jpg)</Frame>


# Aplicación Studio

#### Integración de publicaciones

La integración de la aplicación Studio con Domo Everywhere le permite crear una publicación directamente desde sus aplicaciones de Studio y publicar el contenido a sus suscriptores.

Un suscriptor se puede suscribir a las publicaciones individualmente, como invitado, o suscribirse automáticamente para recibir todas las publicaciones de una organización publicadora en la que confíe. 

Para saber más sobre la aplicación Studio, consulte [Aplicación Studio | Resumen.](/es/s/article/000005295) 

Para aprender a publicar contenido con Domo Everywhere, consulte [Publicar contenido en una instancia de suscriptor](/es/s/article/360045120554).

<Frame>![Publish App.png](/images/kb/es/0EMVq000000rqdB.jpg)</Frame>



#### Integración de Sandbox

Sandbox en la aplicación Studio le permite crear y probar aplicaciones de Studio en un entorno separado y promocionar contenido solo cuando esté listo para una audiencia más amplia. Al crear repositorios en Sandbox, puede elegir la nueva opción de aplicaciones de Studio y promocionar aplicaciones específicas para el propósito de promocionar contenido.

Para saber más sobre la aplicación Studio, consulte [Aplicación Studio | Resumen.](/es/s/article/000005295) 

Para saber más sobre Sanbox, consulte [Domo Sandbox](/es/s/article/4403367344023).

<Frame>![c6e84622-7f04-4378-bbc0-8100ec799958.png](/images/kb/es/0EMVq000000qXcb.jpg)</Frame>

Para saber más sobre DomoStats, consulte [DomoStats Connector.](/es/s/article/360043433813)

# Flujos de trabajo

#### Tipo de datos de la cuenta

Ahora los flujos de trabajo, motores de código y formularios son compatibles con un tipo de datos de la cuenta. Este nuevo tipo de datos le permite configurar y asignar datos de la cuenta en sus pasos del flujo de trabajo, funciones del motor de código y formularios.

Esto incluye una nueva característica de consulta que le permite encontrar y asignar fácilmente cuentas accesibles a un parámetro en un paso del flujo de trabajo.Los parámetros configurados con un tipo de datos de cuenta se pueden asignar a una variable de un tipo similar o a un valor codificado usando un campo de consulta.

Para saber más sobre los flujos de trabajo, consulte [Flujos de trabajo.](https://domo-support.domo.com/s/topic/0TO5w000000ZkAMGA0/workflows?language=en_US)

<Frame>![Screenshot 2024-05-01 at 10.47.30 AM.png](/images/kb/es/0EMVq000000rpKY.jpg)</Frame>

# Mejoras del Analyzer

#### Propiedades de diagrama contextuales

Las propiedades de diagrama contextuales te permiten hacer clic derecho en elementos del diagrama con Analyzer y ver la lista de propiedades del diagrama que se aplican a ese elemento. En la imagen que aparece a continuación, al hacer clic en el eje y, se mostrará la escala de categoría (Y) de la propiedad del programa.

<Frame>![Screenshot 2024-05-01 at 2.45.36 PM.png](/images/kb/es/0EMVq000000s4GL.jpg)</Frame>



#### Tipos de entrada de número

Todos los tipos de diagrama en Analyzer ahora incluyen una característica de validación de los atributos del diagrama que necesitan una entrada de número. Esta validación sirve para asegurar que está introduciendo valores numéricos limpios en el campo de atributos del diagrama.

En el ejemplo que aparece a continuación, el atributo del porcentaje de altura necesita una entrada numérica que se encuentre entre el cero y el cien; si el valor introducido está fuera de ese rango, el atributo se destacará en rojo y el diagrama no se representará.

<Frame>![Screenshot 2024-05-01 at 2.55.52 PM.png](/images/kb/es/0EMVq000000s4oD.jpg)</Frame>



#### Representación rápida

La nueva característica de representación rápida en Analyzer cambia las propiedades del diagrama para que ya no sea necesario obtener datos nuevos con cada carga, lo que reduce el tiempo de representación general. Para los diagramas que usan DataSets grandes, la representación rápida le permite actualizar varios elementos del diagrama sin tener que esperar a que el diagrama repetido se vuelva a cargar.



# Enmascaramiento de columnas

La característica de enmascaramiento de columnas en DataSets le permite ocultar información sensible, como PII y PHI. Puede usar el enmascaramiento de columnas para redactar valores enteros o esconder un cierto número de caracteres en un campo concreto.

Puede configurar el enmascaramiento de columnas en la sección de los PDP de la pantalla de detalles del DataSet.

Aprenda más sobre el [enmascaramiento de columnas](/es/s/article/360042934614) y los PDP en la base de conocimientos.

<Frame>![ColumnMasking (1).png](/images/kb/es/0EMVq000000xyD3.jpg)</Frame>
