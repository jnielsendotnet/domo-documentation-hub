# Nuevas características y mejoras

Las características y mejoras de esta versión incluyen lo siguiente:

## Administrador de Beast Mode

Las mejoras del Administrador de Beast Mode facilitan aún más la administración de sus Beast Modes.

### Archivar Beast Modes

Ahora puede archivar Beast Modes. De esta forma, se eliminan los Beast Modes de las tarjetas o los DataSets. Solo puede archivar un Beast Mode siempre que este no se use en una tarjeta. La característica de archivado también se puede utilizar de forma masiva. Puede seleccionar hasta 100 Beast Modes a la vez si desea utilizar el archivado masivo. Puede archivar una cantidad ilimitada de Beast Modes.

Después, puede desarchivar Beast Modes para restaurarlos a la tarjeta o el DataSet a los que pertenecen. Si se elimina una tarjeta o un DataSet, también se eliminará el Beast Mode. Para obtener más información, consulte Administrador de Beast Mode.

### Modo para nombre de Beast Mode duplicado

Dentro de Analyzer, hay un modo nuevo que aparece cuando hay un nombre de Beast Mode duplicado. Le permite cambiar el nombre del Beast Mode o eliminar los Beast Modes duplicados. Para obtener más información, consulte Adición de un cálculo en Beast Mode al diagrama.

## Programación mejorada del conector

El nuevo programador del conector presenta una IU simplificada y optimizada que es más intuitiva y muestra su zona horaria al realizar la programación.Los conectores se siguen programando en UTC, pero el nuevo programador muestra a qué hora se realiza la conversión a la zona horaria elegida. Para obtener más información, consulte Adición de un DataSet con un conector de datos.

## Aplicaciones de bloque de código

Los bloques de código permiten a los usuarios copiar y pegar varios tipos de código distintos que se pueden usar para crear soluciones para cuestiones complejas. Esto hace que la funcionalidad avanzada sea mucho más accesible. Para obtener más información, consulte Aplicaciones de bloque de código.

**Tipos de bloques de código disponibles:**

* Beast Modes
* SQL
* R
* Python
* Jupyter Notebook

## Componentes públicos de la aplicación de Domo

Ahora puede compartir componentes, como imágenes o código JavaScript, entre la aplicación de Domo personalizada y su sitio web público. También tiene la opción de crear su sitio web público directamente en Domo en lugar de tener que buscar otro proveedor de hospedaje. Para obtener más información, consulte Componentes públicos de la aplicación de Domo.

## Alternancia en conversaciones de Buzz

Con las nuevas flechas de alternancia en conversaciones de Buzz, puede cambiar fácilmente entre los mensajes de Buzz. Use las flechas para avanzar y retroceder por las conversaciones a medida que las lee. Para obtener más información, consulte Diseño de Buzz.

## Objetivos

Gracias a los objetivos de Domo, puede impulsar sus objetivos empresariales con datos mensurables, conocer el progreso con alertas y actualizaciones automáticas y crear una alineación descendente. Con los objetivos, también puede conocer los objetivos de todo el mundo y realizar un seguimiento de la evolución e interrelación de estos. Para obtener más información, consulte Creación y modificación de objetivos.

**Con los objetivos, puede:**

* Definir objetivos empresariales, departamentales y personales que todos puedan consultar.
* Medir los resultados con métricas alimentadas con datos en Domo.
* Adaptar los objetivos a la empresa, los departamentos y el equipo, para que todos puedan saber cómo su trabajo se ajusta a la situación general.
* Ver el progreso a medida que se actualiza en tiempo real con sus datos.
* Informar de las actualizaciones periódicas para que todos conozcan su progreso.
* Crear tableros eficaces para analizar el progreso de su empresa.
* Mostrar con facilidad qué objetivos individuales encajan con los objetivos empresariales.

## Datos federados

El modelo de datos federados de Domo le permite representar directamente tarjetas y otras interacciones a nivel de usuario en la plataforma de Domo mediante una base de datos hospedada fuera de la nube de Domo. En lugar de ejecutar la consulta de visualización en Domo, esta se ejecuta en la base de datos de destino y los resultados se envían a Domo para representar la visualización. Este modelo resulta especialmente importante para los usuarios que quieren mantener los datos en reposo en su infraestructura por motivos de seguridad o logísticos.

Para obtener más información, consulte Uso de la solución de datos federados.

## Característica beta

Vamos a lanzar una versión beta pública de Magic – Extracción, transformación y carga v2. Esta característica estará disponible automáticamente en su instancia.

### Magic – Extracción, transformación y carga v2 (Beta)

Magic – Extracción, transformación y carga v2 (Beta) presenta muchas mejoras de iconos y back-end, aparte de incorporar nuevos iconos. Al abrir la vista de edición de una instancia de Magic – Extracción, transformación y carga- Dataflow, aparece un botón de alternancia para habilitar Magic – Extracción, transformación y carga v2. Es importante tener en cuenta que ha cambiado la funcionalidad de algunos iconos. Para obtener más información, consulte Optimizaciones de Magic – Extracción, transformación y carga v2- Dataflow.

</Warning>
