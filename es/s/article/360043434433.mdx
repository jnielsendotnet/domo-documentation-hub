## Introducción

El conector de Facebook para Domo puede combinar información detallada de Facebook con datos de cualquier otro origen, como análisis financieros, software de gestión de relaciones con los clientes, etc., para dirigir campañas con una precisión sin precedentes. Use este conector para obtener datos sobre publicaciones, aficionados de páginas, referencias, información sobre cuentas y afines. Para obtener más información sobre la API de Facebook, visite su página ([https://developers.facebook.com](https://developers.facebook.com)).

Esta versión del conector de Facebook le permite recuperar los datos de Facebook de 30 días. La versión avanzada le permite recuperar datos de más de 30 días y también incluye más tipos de informes y parámetros de configuración. Para obtener más información sobre la versión avanzada del conector, consulte [Conector avanzado de Facebook](/s/article/360043434413).

El conector de Facebook es un conector de "aplicación en la nube", lo que significa que obtiene los datos almacenados en la nube. En Data Center, puede acceder a la página de este y de otros conectores de aplicación en la nube si hace clic en **Aplicación en la nube** en la barra de herramientas situada en la parte superior de la ventana.

Se puede conectar a la cuenta de Facebook desde Data Center. En este tema se tratan los campos y menús específicos de la interfaz de usuario del conector de Facebook. La información general sobre cómo agregar DataSets, configurar programaciones de actualizaciones y editar información sobre DataSets está disponible en [Adición de un DataSet con un conector de datos](/s/article/360042926274).

## Prácticas recomendadas

- Se pueden seleccionar varias cuentas para los informes. Esto resulta útil para organizaciones grandes con varias ubicaciones o divisiones.
- Los informes permiten seleccionar intervalos de fechas personalizados.
- Este conector solo ofrece 30 días de datos. Si desea obtener más datos, use el [conector avanzado de Facebook](/s/article/360043434413).

## Requisitos previos

Para conectar su cuenta de Facebook y crear un DataSet, debe disponer de lo siguiente:

- El nombre de usuario de Facebook o su número de teléfono
- La contraseña de Facebook

## Conexión a su cuenta de Facebook

En esta sección se enumeran las opciones de los paneles Credenciales y Detalles de la página del conector de Facebook. Los componentes de los otros paneles de esta página, **Programa** y **Asigne un nombre y una descripción a su DataSet,** son universales para todos los tipos de conectores y se tratan con más detalle en [Adición de un DataSet con un conector de datos](/s/article/360042926274).

### Panel Credenciales

El conector de Facebook para Domo usa OAuth para la conexión, por lo que no es necesario escribir credenciales en Domo. Haga clic en **Conectar** , o seleccione **Agregar cuenta** si tiene cuentas de Facebook existentes en Domo, para abrir la pantalla OAuth de Facebook en la que puede escribir el nombre de usuario y la contraseña de Facebook. Una vez haya especificado las credenciales válidas de Facebook, puede usar la misma cuenta cada vez que vaya a crear un nuevo DataSet de Facebook. Puede administrar cuentas de conectores en la pestaña **Cuentas** de Data Center. Para obtener más información sobre esta pestaña, consulte [Administración de cuentas de usuario para conectores](/s/article/360042926054).

**Nota:** Si ya ha iniciado sesión en Facebook al conectarse en Domo, se le autenticará automáticamente al hacer clic en **Agregar cuenta**. Si desea conectarse a una cuenta distinta de aquella en la que ha iniciado sesión, primero debe cerrar la sesión de Facebook.

## Preguntas frecuentes

#### ¿Cómo puedo saber si mis credenciales de Facebook son seguras?

El proceso de inicio de sesión usa OAuth, lo que permite que Domo nunca vea ni almacene las credenciales. Esto hace que el inicio de sesión sea seguro. Puede revocar el acceso de Domo a su cuenta en cualquier momento.

#### ¿Necesito una cuenta de un tipo determinado para configurar el conector?

Se puede usar cualquier cuenta de Facebook para configurar el conector. Sin embargo, debe ser administrador de las páginas de las que quiere obtener datos.

#### ¿Puedo obtener información de las páginas de las que no soy administrador?

No con este conector. Para obtener información sobre las páginas de las que no es administrador, debe usar el [conector avanzado de Facebook](/s/article/360043434413).

#### ¿Qué diferencia hay entre el conector de Facebook Ads y el conector avanzado de Facebook Ads?

El conector avanzado de Facebook ofrece más informes que el conector de Facebook. Los informes del conector de Facebook están limitados para interactuar directamente con páginas. Los informes sobre la información de usuarios, dominios, supervisión de aplicaciones y páginas públicas están disponibles en el conector avanzado.

El conector de Facebook también está limitado a devolver información de los 30 últimos días, mientras que el conector avanzado permite al usuario consultar datos en cualquier intervalo de fechas.

#### ¿Con qué frecuencia se actualizan mis datos?

La mayoría de la información sobre las páginas se actualiza cada 24 horas. Un número reducido de métricas de información detallada se actualiza cada 15 minutos aproximadamente. Se recomienda configurar habitualmente los DataSets para que se ejecuten cada 24 horas.

#### ¿Por qué las fechas de la interfaz de usuario de la información detallada de Facebook no siempre coinciden con las fechas de Domo?

A menudo, las personas validan sus DataSets comparando los resultados del conector con las estadísticas que pueden ver en la interfaz de usuario de información detallada de Facebook. Si lo hacen así, las fechas suelen tener una diferencia de 1 día. Por ejemplo, si en la interfaz de usuario de Facebook aparecen 1000 visitas a una página el 3 de marzo, el conector reflejará las 1000 visitas con fecha del 4 de marzo. Esto se debe a que la API devuelve datos durante un período de tiempo que finaliza en la marca de tiempo especificada. En este ejemplo, se registraron 1000 visitas durante el día que finalizó en la medianoche del 4 de marzo, es decir, el 3 de marzo. La interfaz de usuario de información detallada tiene esto en cuenta y registra las 1000 visitas como si se hubieran realizado el 3 de marzo.

#### ¿Existe algún límite de API que deba tener en cuenta?

Facebook limita el número de llamadas que se pueden realizar cada hora por cada usuario y página. Siempre que no ejecute el DataSet con demasiada frecuencia, el conector administra esta tarea automáticamente.

## Solución de problemas

- Asegúrese de que la autenticación sigue siendo válida.
- Revise la configuración para asegurarse de que todos los elementos necesarios están seleccionados.
- Revise el historial de mensajes de error del conector.
- En raras ocasiones, puede que se le solicite demasiada información y que alcance los límites o el tiempo de espera de la API. En su caso, puede revisar el historial de ejecuciones del conector para consultar la duración y el mensaje de error. De esta forma, puede reducir el número de cuentas de las que se extraen datos, elegir un número más reducido de métricas del informe del que extrae los datos o reducir el período de tiempo para la extracción.
- Tenga en cuenta que, al comparar las estadísticas de las páginas del DataSet de Facebook con las que se encuentran en la interfaz de usuario de la información detallada de Facebook, hay una diferencia de 1 día en las fechas. Por ejemplo, si en la interfaz de usuario de Facebook aparecen 1000 visitas de una página el 3 de marzo, el conector reflejará las 1000 visitas con fecha del 4 de marzo. Esto se debe a que la API devuelve datos durante un período de tiempo que *finaliza* en la marca de tiempo especificada. En este ejemplo, se registraron 1000 visitas durante el día que finalizó en la medianoche del 4 de marzo, es decir, el 3 de marzo. La interfaz de usuario de información detallada tiene esto en cuenta y registra las 1000 visitas como si se hubieran realizado el 3 de marzo.
