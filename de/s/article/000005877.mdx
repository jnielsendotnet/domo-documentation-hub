# Konnektoren: erweiterter Facebook-Anzeigen-Konnektor für mehrere Konten

# Erweiterung bei sensiblen Änderungen

Mit der Funktion „Erweiterung bei sensiblen Änderungen“ sind bei Aktionen wie Passwortänderungen nun eine Berechtigungserweiterung erforderlich. Dabei handelt es sich um einen zeitlich begrenzter Zeitraum, in dem sensible Aktionen durchgeführt werden können. Sie können eine Erweiterung starten, indem Sie einen einmaligen Passcode (OTP) eingeben (von einer Authentifizierungs-App generiert) Dieses Update bietet mehr Sicherheit bei sensiblen Aktionen in Ihrer Instanz.

Erfahren Sie, wie Sie eine [Authentifizierungs-App registrieren](https://domo-support.domo.com/s/article/000005831?language=en_US#register_domo) und die [Erweiterungsfunktion](https://domo-support.domo.com/s/article/000005831?language=en_US#start_end_elevation) verwenden.

# Instanzvorlagen

Mit Instanzvorlagen können Sie mehrere Feature-Vorlagen für Instanzen erstellen, indem Sie in Domo verschiedene Feature-Switches und -Verhaltensweisen konfigurieren.

Nachdem der Konfiguration der Instanzvorlagen können Sie diese beim Erstellen neuer Instanzen anwenden. Sie können Vorlagen auch direkt aus der übergeordneten Instanz auf vorhandene Instanzen anwenden.

Die folgenden Funktionen sind für Instanzvorlagen verfügbar:

* Anpassung der Domo-Standardeinstellungen für Ihre Instanzen, um Ihrem Branding zu entsprechen.
* Steuerung, auf welche Funktionen die Benutzer in Ihrer verwalteten Instanz zugreifen können.

# Update des Dashboards von v4 auf v5

Während unserer regelmäßigen technischen Wartungsarbeiten aktualisieren wir die Dashboards von Layouts v4 auf Layouts v5. Dieses Upgrade hat keine Auswirkungen auf die Kernfunktionen der Dashboards, verbessert jedoch die Bearbeitungsfunktionen, um sie an App Studio anzupassen.

Dies sind die Neuerungen:

* Eine intuitivere Bearbeitungsoberfläche mit Klick-und-Zieh-Handles.
* Layout-Ausrichter für eine einfachere Positionierung der Inhalte.
* Kleinere Änderungen bei der Größenanpassung der Layoutinhalte. Dafür sind möglicherweise einige manuelle Anpassungen in bestehenden Dashboards erforderlich.

Sie sollten die Dashboards nach dem Update überprüfen, um sicherzustellen, dass alle Inhalte wie erwartet angezeigt werden.

#### Testlauf von Arbeitsabläufen

Mit der Funktion „Testlauf von Arbeitsabläufen“ können Sie Ihren Arbeitsablauf vor der Bereitstellung ausführen. Auf diese Weise haben Sie die Möglichkeit, die Ausführungsdetails und Daten für jeden Testlauf zu überprüfen. Mit Testläufen ermitteln Sie ganz einfach vor der Bereitstellung alle erforderlichen Anpassungen zur Optimierung des Arbeitsablauf-Designs. Sie können das Arbeitsablauf-Modell so oft wie nötig wiederholen, um eine Optimierung des Arbeitsablaufs sicherzustellen.

# API-Client erstellen

Die Erstellung und Verwaltung von API-Clients kann nun in der Domo-Instanz über die Administratoreinstellungen verwaltet werden. Benutzer mit der bestehenden Berechtigung „API-OAuth-Clients erstellen“, die in den Domo-Systemberechtigungen enthalten ist, haben Zugriff auf die Funktion „API-Client erstellen“.

Wenn Sie Domo-Administrator sind, können Sie diese Funktion verwenden, um Benutzerrollen für Ihre Organisation entsprechend festzulegen. Sie haben auch die Möglichkeit, die Berechtigung der Benutzer in Ihrer Instanz zum Erstellen von API-Clients einzuschränken. Ändern Sie dazu deren Benutzerrollen und Berechtigungen nach Bedarf.

<Frame><img alt="" src="/images/kb/de/ka0Vq000000HWJd-00N5w00000Ri7BU-0EMVq000006bfAP.jpg" style={{width: 318, height: 398}}/></Frame>
<Note>
  **Hinweis:** Die Möglichkeit, API-Clients zu erstellen, wird durch die Berechtigung „API-OAuth-Clients erstellen“ gesteuert, die in allen Domo-Systemrollen enthalten ist. Diese Erweiterung wirkt sich nicht auf die Fähigkeit der Benutzer, API-Clients zu erstellen, aus (bestehende Benutzerrollen oder Berechtigungen bleiben unverändert). Da API-Clients jetzt jedoch in Domo erstellt werden können, wird diese Funktion Ihren Benutzern mit größerer Wahrscheinlichkeit angezeigt. Überprüfen Sie die Rollen Ihrer Benutzer und nehmen Sie gegebenenfalls erforderliche Anpassungen vor.

</Note>

# Bericht-Builder für geplante Berichte

Mit dem Bericht-Builder in App Studio erstellen Sie unter Verwendung Ihrer Apps anpassbare E-Mail-Berichte.

Und Folgendes ist mit dem Bericht-Builder möglich:

* Sie können mehrere App-Seiten hinzufügen.
* Sie können Inhalte im Bericht-Builder ziehen, ablegen, austauschen, entfernen sowie die Größe der Inhalte ändern.
* Sie können zusätzliche Inhalte hinzufügen, um Berichte zu verbessern, einschließlich Überschriften, Rahmen, Karten und Text.
* Es gibt verschiedene Konfigurationsoptionen zum Anpassen von Kopf- und -Fußzeilen in E-Mails mit Logos und Inhalten.
* Sie können eine Schaltfläche hinzufügen, die auf Inhalte von Domo verlinkt, einschließlich der App selbst oder einer Karte/einem Dashboard.
* Sie können Designkomponenten hinzufügen (darunter Kartenstile, Hintergrundfarben und Dichte) und Designs aus jeder beliebigen App importieren, um die Farbpaletten zu synchronisieren.
* Sie können Filter hinzufügen und den Bericht für Benutzer und Gruppen planen.

# SCIM-APIs: Verbesserungen bei Benutzern und Gruppen

Das Protokoll SCIM (System for Cross-domain Identity Management) ermöglicht es einem Identitätsanbieter (IdP), die Erstellung, Aktualisierung und Löschung von Benutzern und Gruppen zwischen Softwaresystemen per Push zu übertragen. Während SCIM bereits in Domo verfügbar war, wurden jetzt auch Gruppenendpunkte implementiert. Diese ermöglichen das Erstellen und Löschen von Verzeichnisgruppen sowie die Berücksichtigung von Änderungen im IdP in Gruppenmitgliedschaften, ohne dass eine Anmeldung bei Domo erforderlich ist.

Weitere Informationen zu [SCIM in Domo](/s/article/000005241).
