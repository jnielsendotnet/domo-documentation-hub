Ansicht der Seitenzusammenfassung

## Einführung

Sie können ein benutzerspezifisches Erlebnis für Benutzer und Gruppen in Ihrem Domo erstellen, indem Sie PDP-Richtlinien (Personalisierte Datenberechtigungen) für DataSets einrichten. Mit diesen Richtlinien können Sie Daten in DataSets für bestimmte Benutzer und Gruppen filtern. Wenn diese Benutzer und Gruppen KPI- und Sumo-Karten betrachten, auf die PDP-Richtlinien angewandt wurden, dann sehen sie nur die Daten, die sie betreffen.

Zum Beispiel haben Sie vielleicht eine Karte, die eine Landkarte mit Daten für zwei Regionen in den USA darstellt – Westen und Osten. Sie könnten zwei PDP-Richtlinien erstellen – eine für die Region „Westen“ und eine für die Region „Osten“. Für die Richtlinie „Westen“ würden Sie das Verkaufspersonal auswählen, das in der Region im Westen arbeitet, für die Richtlinie „Osten“ würden Sie das Verkaufspersonal auswählen, das in der Region im Osten arbeitet. Falls Larry, ein Verkäufer, der in Washington arbeitet, die Karte öffnet, würde er *nur* Daten für die Bundesstaaten in der Region im Westen sehen – alle Staaten in der Region im Osten würden ohne Daten erscheinen.

PDP-Richtlinien werden auch auf andere Tools in Domo angewandt, wie etwa auf Mitteilungen und DataFusions. Wenn Sie eine Mitteilung auf einer Karte abonniert haben, die von einem PDP-DataSet angetrieben wird, wird die Mitteilung entsprechend angepasst, um die festgelegten PDP-Richtlinien widerzuspiegeln. Wenn Sie eine DataFusion von einem oder mehreren Eingabe-DataSets erstellen, für die PDP-Richtlinien eingerichtet wurden, werden diese Richtlinien in das Ausgabe-DataSet integriert.

## Erstellen von PDP-Richtlinien

Sie erstellen und verwalten PDP-Richtlinien im Data Center.

**So erstellen Sie eine PDP-Richtlinie:**

1. Klicken <img alt="More_Menu.jpg" src="/images/kb/de/0EM5w000005wJQo.jpg" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/> **Sie auf > Data Center**.
2. Klicken Sie auf **DataSets** im Navigationsfeld auf der linken Seite.
3. Suchen und klicken Sie auf das DataSet, auf das Sie eine Richtlinie anwenden möchten.
4. Klicken Sie auf die Registerkarte **Personalisierte Datenberechtigungen**.  
   <Frame>
   ![](/images/kb/de/0EM5w000005wJQk.png)
   </Frame>
5. (Optional) Klicken Sie auf die Schaltfläche **Einfluss**, um eine Liste mit Karten und Mitteilungen zu öffnen, die von den PDP-Richtlinien auf diesem DataSet betroffen sein werden.   
   Obwohl dies nicht erforderlich ist, empfehlen wir unbedingt, dass Sie diese Liste prüfen, bevor Sie fortfahren. Weitere Informationen finden Sie unter [PDP-Ergebnisse anzeigen](/s/article/360042934614).
6. (Optional) Falls es noch zusätzliche Personen neben dem DataSet-Besitzer und den Administratorbenutzern gibt, die Zugriff auf all die Daten in diesem DataSet haben sollten, gewähren Sie dem entsprechenden Benutzer vollständige Berechtigung, indem Sie folgende Schritte ausführen:
   1. Klicken Sie auf <img alt="pdp_users_icon.png" src="/images/kb/de/0EM5w000005wJQt.png" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/> in der Zeile **Alle Zeilen**.
   2. Geben Sie im Dialogfenster **Gruppen und Personen hinzufügen** den Namen einer Person oder Gruppe ein, die vollständigen Zugriff auf die Daten haben soll. Wählen Sie einen Namen aus und fügen Sie ihn der Richtlinie hinzu.
   3. Klicken Sie auf **Anwenden**.
      <Warning>
      **Wichtig**: Falls es sich bei dem DataSet um ein Eingabe-DataSet in einem DataFlow handelt, empfehlen wir unbedingt, dass Sie dem DataFlow-Besitzer vollständigen Zugriff gewähren, da der DataFlow ansonsten abstürzen könnte, wenn PDP-Richtlinien festgelegt werden.
      </Warning>
7. Klicken Sie auf **Richtlinie hinzufügen**.
8. Geben Sie einen beschreibenden Namen für diese Richtlinie ein.
9. Klicken Sie auf **Daten hinzufügen**, um das Dialogfenster **Datenzugriff hinzufügen** zu öffnen.
10. Wählen Sie in der Dropdown-Liste **Spaltenname** die Spalte aus, die Sie filtern möchten.
11. Geben Sie im Namen **Zeilenwerte suchen/hinzufügen** den Namen einer Zeile ein, die Sie filtern möchten. Wählen Sie einen Zeilennamen und fügen Sie ihn der Richtlinie hinzu.
12. (Optional) Wiederholen Sie Schritt 8 so oft, wie es nötig ist, bis Sie alle Zeilen hinzugefügt haben, die Sie filtern möchten.
13. Klicken Sie auf **Anwenden**.
14. Klicken Sie auf <img alt="pdp_users_icon.png" src="/images/kb/de/0EM5w000005wJQt.png" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/>, um das Dialogfenster **Gruppen und Personen hinzufügen** zu öffnen.
15. Geben Sie in das Feld **Gruppen und Personen suchen/hinzufügen** den Namen einer Person oder der Gruppe ein, die Sie zur Richtlinie hinzufügen möchten. Wählen Sie einen Namen aus und fügen Sie ihn der Richtlinie hinzu.
16. (Optional) Wiederholen Sie Schritt 12 so oft, wie es nötig ist, bis Sie dieser Richtlinie alle gewünschten Gruppen und Benutzer hinzugefügt haben.
17. Klicken Sie auf **Speichern**.

Die Richtlinie ist jetzt erstellt. Sie wurde jedoch noch nicht auf dieses DataSet angewandt. Klicken Sie auf neben dem Text „PDP aktivieren“, um PDP für das DataSet zu aktivieren. Jetzt wird alles, was in Domo von diesem DataSet betrieben wird oder damit in Verbindung steht, von der Richtlinie beeinflusst und enthält einen Stempel von <img alt="pdp_stamp.png" src="/images/kb/de/0EM5w000005wJQv.png" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/>, um Benutzer darüber in Kenntnis zu setzen, dass sie aufgrund der PDP nicht alle verfügbaren Daten sehen.

Klicken Sie auf um diese Richtlinie zu deaktivieren.

Sie können mehrere Richtlinien auf dasselbe DataSet anwenden, indem Sie in der Registerkarte **Personalisierte Datengenehmigungen** auf **Richtlinie hinzufügen** für das DataSet klicken und die vorangegangenen Schritte wiederholen.

### Hinzufügen von Benutzern zu mehreren PDP-Richtlinien

Es kann vorkommen, dass Sie einen Benutzer oder eine Gruppe gleichzeitig zu mehreren DataSets hinzufügen möchten. Zum Beispiel haben Sie vielleicht einen Verkäufer namens „Ted“, der in der Region „Osten“ arbeitet, aber zusätzlich Zugriff auf die Daten von Kalifornien benötigt. Sie können dies erreichen, indem Sie eine *neue* Richtlinie hinzufügen, in der die Daten für Kalifornien auf Ted angewandt werden. Das Hinzufügen von „Kalifornien“ zu einer existierenden Richtlinie, in der Sie die östliche Region ausgewählt haben *wird nicht funktionieren* – Ihr DataSet erscheint leer. Das liegt daran, dass die Auswahl von mehreren Werten aus demselben DataSet eine AND-Anweisung anwendet und da „Kalifornien“ nie in der östlichen Region im DataSet gefunden wird, unterdrückt die Richtlinie die entsprechenden Daten. Getrennte Richtlinien im selben DataSet werden jedoch als OR-Anweisungen betrachtet, daher erscheinen die Daten von Ted sowohl für die Region Osten, wie auch für Kalifornien.

### PDP-Ergebnisse ansehen

Durch Klicken auf die Schaltfläche **Einfluss** können Sie ein Dialogfenster öffnen, das anzeigt, wie das Einrichten von PDP-Richtlinien für ein DataSet Einfluss auf DataFlows und Karten nimmt, die an das DataSet gebunden sind. Sämtliche betroffenen DataFlows finden sich unter der Registerkarte **Achtung**, sämtliche betroffenen Karten finden sich unter der Registerkarte **Personalisiert**. Wenn die Richtlinien keine DataFlows oder Karten beeinflussen, erscheinen keine Registerkarten.

Es wird unbedingt empfohlen, dass Sie die PDP-Ergebnisse ansehen, bevor Sie Richtlinien auf einem DataSet einrichten, besonders wenn es sich bei dem DataSet um ein Eingabe-DataSet in einem DataFlow handelt. Falls ein DataFlow durch das Einrichten von PDP-Richtlinien auf besagtem DataSet abstürzt, wird die Schaltfläche **Reparieren** bereitgestellt. Sie können auch den DataFlow-Besitzer in die Gruppe **Alle Zeilen** mit einschließen und somit sicherstellen, dass der DataFlow nicht abstürzt.

### Vorschau mit DataSets, die von PDP-Richtlinien beeinflusst werden

Alle PDP-Richtlinien bieten die Möglichkeit, eine Vorschau der Daten im DataSet abzurufen, die von dieser Richtlinie betroffen werden. Nachdem Sie eine Richtlinie erstellt haben, empfiehlt es sich, eine Vorschau der Daten für diese Richtlinie anzusehen, um sicherzustellen, dass die notwendigen Daten erscheinen.

**So zeigen Sie die Vorschau eines DataSets für eine PDP-Richtlinie an:**

1. Bewegen Sie die Maus in der Registerkarte **Personalisierte Datengenehmigungen** des DataSets, für das Sie eine Vorschau abrufen möchten, über die Zeile für die Richtlinie, für die Sie eine Vorschau wünschen.
2. Klicken Sie auf das Augensymbol <img alt="pdp_preview.png" src="/images/kb/de/0EM5w000005wJQm.png" style={{width: 20, height: 20, display: 'inline', verticalAlign: 'start', margin: '0'}}/>, um die Vorschau zu öffnen.
3. Klicken Sie auf **Fertig**, wenn Sie die Vorschau der Daten beendet haben.


## Löschen von PDP-Richtlinien

Sie können eine PDP-Richtlinien in der Registerkarte **Personalisierte Datengenehmigungen** für das DataSet mit der Richtlinie löschen Wenn Sie eine Richtlinie löschen, wird diese nicht länger auf Karten, Mitteilungen, DataFlows etc. angewendet.

**Wichtig**: Das Löschen aller Richtlinien für ein DataSet deaktiviert die PDP für das DataSet, sodass alle Benutzer mit Zugriff die Daten sehen können.

**So löschen Sie eine PDP-Richtlinie:**

1. Bewegen Sie die Maus in der Registerkarte **Personalisierte Datengenehmigungen** des DataSets, das die Richtlinien enthält, die Sie löschen möchten, über die Zeile der Richtlinie, die Sie zu löschen wünschen.
2. Klicken Sie auf das Abfalleimersymbol, um die Richtlinie zu löschen.
3. Klicken Sie zum Bestätigen auf **Löschen**. Falls dies die einzige Richtlinie ist, die auf das DataSet angewendet wird, erhalten Sie eine Warnung darüber, dass alle Benutzer mit Zugriff die Daten einsehen können, sobald die Richtlinie gelöscht wird.
4. Falls Sie dies wirklich tun möchten, klicken Sie erneut auf **Löschen**.
