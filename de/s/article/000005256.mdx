## Dojo wird zum Community-Forum

Im Rahmen der Veröffentlichung für März 2023- stellen wir das [Community-Forum](https://dojo.domo.com/main) vor und ersetzen damit das Dojo.

Das Community-Forum finden Sie unter Hilfe im Domo-Central. Mit diesem Rebranding aktualisieren wir die Benutzeroberfläche und -erfahrung, um das Finden der Inhalte, die Sie suchen, leichter zu machen. [Soziale Gruppen](http://dojo.domo.com/main/groups) und die jüngsten [Aufzeichnung von Events](http://dojo.domo.com/main/categories/recordings) sind jetzt im Community-Forum verfügbar.

## Neue Funktionen

Diese Funktionen sind Teil der Veröffentlichung für März 2023.

### Microsoft Office-Add-ins

Die Domo-Add-ins für Microsoft 365 ermöglichen die Integration deiner Microsoft Office-Produkte mit Domo. Anstatt Daten händisch zu übertragen, können diese nun effizient zwischen Excel, Outlook, PowerPoint, Word und Domo verschoben werden. So können Sie mit den Add-ins zum Beispiel Karten und DataSets in ein Word-Dokument oder Daten von einer Excel-Tabelle in Domo importieren.

Weitere Informationen finden Sie in der [Installationsanleitung für Domo-Add-ins für Microsoft 365](https://domo-support.domo.com/s/article/000005146) und in der [Benutzeranleitung für Domo-Add-ins für Microsoft 365-Desktop-Apps](https://domo-support.domo.com/s/article/000005143). Sie können auch einfach in Microsoft AppSource nach Domo suchen.

Enterprise-Kunden können die neuen Domo-Add-ins in ihrer Organisation bereitstellen. So können alle Nutzer auf die Add-ins direkt über deren Microsoft 365-Apps zugreifen, ohne sie vorher installieren zu müssen.

Weitere Informationen finden Sie in der [Installationsanleitung für Domo-Add-ins für Microsoft 365](https://domo-support.domo.com/s/article/000005145).

### Benutzerdefinierte Attribute und Listenattribute für Gruppen

Im Abschnitt **Governance** der **Administratoreinstellungen** können Sie nun Attribute verwenden, um dynamische Gruppenmitgliedschaften zu kontrollieren. Beispielsweise können Sie alle Nutzer einer bestimmten Abteilung oder eines bestimmten Standorts zu Mitgliedern einer Gruppe machen, die Zugang zu bestimmten Inhalten bekommt. Domo-Administratoren können die Attribute vom IdP ihrer Organisation importieren und so Nutzer automatisch zu den richtigen Gruppen zuzuordnen.

Mehr dazu finden Sie hier: [Administratoreinstellungen: Governance](https://domo-support.domo.com/s/article/000005147?language=en_US).

### Tabellenberechnungen

Tabellenberechnungen ermöglichen Ihnen, einen Wert in der Tabelle als Prozentsatz eines Spaltengesamtwertes darzustellen, als Teil einer laufenden Summe oder ihn mit anderen Werten in der Spalte zu vergleichen. All das funktioniert ohne die Verwendung von Beast Mode-Berechnungen oder DataFlows. Tabellenberechnungen sind für Megatabellen und Pivottabellen verfügbar.

Weitere Informationen zu Tabellenberechnungen finden Sie hier: [Tabellendiagramm](https://domo-support.domo.com/s/article/360043429573?language=en_US).

### Neue Diagrammtypen

#### Zirkuläres Sankey-Diagramm

Das zirkuläre Sankey-Diagramm baut auf dem ursprünglichem Sankey-Diagramm auf, indem es rekursive Konzepte erfasst, für die Abläufe benötigt werden, die zu den vorherigen Knoten zurückkehren. Mehr Informationen dazu finden Sie in unserem Artikel zum [zirkulären Sankey-Diagramm](https://domo-support.domo.com/s/article/000005155).

#### Slope-Diagramm

Ein Slope-Diagramm ist eine einfache Visualisierung einer Zeitspanne anhand von Linien, die auf den Start- und Endpunkten der Zeitspanne basiert. Dieses einfach zu lesende Diagramm eliminiert unübersichtliche Zwischenpunkte und erleichtert somit den Fokus auf die wesentlichen Einblicke. Weitere Informationen dazu finden Sie in unserem Artikel zum [Slope-Diagramm](https://domo-support.domo.com/s/article/000005154?language=en_US).

Varianzdiagramme ermöglichen das Hervorheben von Unterschieden zwischen zwei Datensätzen. Weitere Informationen dazu finden Sie in unserem [Varianzdiagramm](https://domo-support.domo.com/s/article/000005156) -Artikel.

## Neue Premiumfunktionen

### Domo Everywhere

#### Publikationen

* Automatische Abonnements ermöglichen es abonnierten Organisationen, alle Publikationen einer bestimmten Veröffentlichungsorganisation zu abonnieren und alle veröffentlichten Inhalte automatisch und ohne individuelle Zustimmung zu erhalten.
* In Verbindung mit den automatischen Abonnements kann eine abonnierte Organisation mithilfe neuer Standard-Freigaberegeln alle Publikationen einer bestimmen Veröffentlichungsorganisation automatisch an beliebig viele Nutzer oder Gruppen schicken.
* Eine neue Sortierfunktion ermöglichen es Ihnen, auf einen Blick zu erkennen, wann Sie eine Publikation zuletzt aktualisiert haben. Sie können auch sehen, ob eine Publikation einen Fehlerstatus hat und bei welchem Abonnenten.

Weitere Information erhalten Sie unter [Veröffentlichung von Inhalten an eine Abonnentenorganisation](http://domo-support.domo.com/s/article/360045120554?language=en_US) **.**

Programmatisches Wechseln von DataSets ist nun verfügbar. Weitere Informationen erhalten Sie unter [Wechseln von DataSets](http://developer.domo.com/docs/embed/dataset-switching) im Entwicklerportal.

Das Veröffentlichen von Domo Bricks wird jetzt von Domo Everywhere unterstützt.

## Beta-Funktionen

Diese Funktionen sind zum Zeitpunkt der Veröffentlichung für März 2023 in der Beta-Phase.

### Verarbeitung der Teilmenge

Verarbeitung der Teilmenge ist in der offenen Beta-Phase. Diese Funktion ist innerhalb der Magic ETL und ermöglicht es Ihnen, wenn nötig nur bestimmte Untergruppen Ihrer Daten anstatt des ganzen DataSets zu aktualisieren. Somit sparen Sie Zeit und erhöhen Ihre Effizienz. Sie können innerhalb Ihrer Dateien Partitionen erstellen und anschließend auswählen, welche Partitionen während der Verarbeiten aktualisiert werden sollen.

Weitere Informationen finden Sie unter [Verarbeitung von Teilmengen in Magic ETL](https://domo-support.domo.com/s/article/000005150?language=en_US).

Um an der Beta-Testung teilzunehmen, folgen Sie bitte diesem Link: [Verarbeitung von Teilmengen für Beta-Phase](http://domo.az1.qualtrics.com/jfe/form/SV_exOdxemYlvewU8S.).

### Arbeitsabläufe

Für Arbeitsabläufe werden wirkungsvoll weitere neue Produkte verwendet, unter anderem Code-Engine, Arbeits-Warteschlangen und Formulare (alle in der offenen Beta-Phase). So wird es Ihnen ermöglicht, ausführbare Arbeitsabläufe und -prozesse mit niederen Code-Kapazitäten zu erstellen und implementieren. Sie können einen Arbeitsablauf manuell, über Mitteilungen, einen Genehmigungsfluss oder eine Zeitschaltuhr starten. Mit Arbeitsabläufen können Sie Ihre Daten bearbeiten, um Prozess-Automatisierung, intelligente Daten-Pipelines, Power-App-Aktionen und manuelles Aufgaben-Management zu lösen, ohne dabei von Entwicklern abhängig zu sein.

Weitere Informationen erhalten Sie unter [Überblick über die Arbeitsabläufe](https://domo-support.domo.com/s/article/000005108?language=en_US).

### Code-Engine

Code-Engine ist eine Domo-native Laufzeit-Umgebung, die JavaScript- oder Python-Code ausführen und mit allen Funktionen innerhalb oder außerhalb von Domo interagieren kann. Domo bietet eine allgemeine Bibliothek an Paketen und Funktionen für gängige Integrationen und Dienste, aber Sie können auch Ihre eigenen Funktionen erstellen und diese in Ihrer Domo-Instanz verwenden. Sie können in Arbeitsabläufen auch eine Funktion von Code-Engine verwenden, ohne selbst einen Code schreiben zu müssen.

Weitere Informationen finden Sie unter Code-Engine.

### Formulare

Der neue Formular-Editor ermöglicht es Ihnen, Formulare für personalisierte manuelle Interaktionen zu erstellen und veröffentlichen. Sie können Formulare mit einem Arbeitsablauf integrieren, sodass ein Formular automatisch generiert und an einen Endnutzer weitergeleitet wird, wenn bei einem Arbeitsablauf eine manuelle Handlung notwendig ist. Sie können auch Daten mithilfe eines Formulars an einen Arbeitsablauf schicken, das es Ihnen erlaubt, auf die Informationen auf anderen Stufen zu verweisen.

Weitere Informationen finden Sie unter Formulare.

<Frame><img alt="release notes 8.jpeg" src="/images/kb/ka0Vq0000000rAn-00N5w00000Ri7BU-0EM5w000006uK0R.jpg" style={{width: 397, height: 637}}/></Frame>
### Arbeits-Warteschlangen

Arbeits-Warteschlangen ermöglicht es Ihnen, manuelle Aufgaben zu organisieren, in verschiedene Warteschlagen einzuteilen und zu kontrollieren, wer auf diese zugreifen kann. Sie können eine Warteschlange so einstellen, dass Aufgaben einer Person oder einer Gruppe angezeigt werden. Jede Aufgabe ist mit einem Formular verbunden, das von einem Arbeitsablauf generiert wurde. Sie können bei Warteschlangen Filter anwenden, um die Zuordnung oder den Status von Aufgaben einzusehen, den Umfang einer Warteschlange zu überprüfen und Layouts zu personalisieren.

Weitere Informationen finden Sie unter Arbeits-Warteschlangen.
